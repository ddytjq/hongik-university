            <H1><!-- #BeginEditable "Heading" -->The Basics<!-- #EndEditable --></H1>
            <!-- #BeginEditable "Content" --> 
            <TABLE WIDTH="100%" BORDER="0" CELLSPACING="4" CELLPADDING="5">
              <TR BGCOLOR="#F8FFF0"> 
                <TD COLSPAN="2"> 
                  <P CLASS="SUBHEADING">Syntax</P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD WIDTH="50%" CLASS="justify"> 
                  <P>The C code you write is called the <SPAN CLASS="RED">SYNTAX</SPAN>. </P>
                  <P>Syntax is a mixture of: </P>
                  <TABLE WIDTH="100%" BORDER="0" CELLSPACING="0" CELLPADDING="0">
                    <TR> 
                      <TD WIDTH="0%" VALIGN="TOP" ALIGN="RIGHT" NOWRAP> 
                        <P CLASS="RIGHT">&nbsp;&nbsp;&nbsp;&nbsp;&#149;&nbsp;&nbsp;</P>
                      </TD>
                      <TD WIDTH="100%" VALIGN="TOP"> 
                        <P>C keywords like <TT>int</TT>, <TT>for</TT> and <TT>return</TT>. </P>
                      </TD>
                    </TR>
                    <TR> 
                      <TD WIDTH="0%" VALIGN="TOP" ALIGN="RIGHT"> 
                        <P CLASS="RIGHT">&#149;&nbsp;&nbsp;</P>
                      </TD>
                      <TD WIDTH="100%" VALIGN="TOP"> 
                        <P>Constants and variables.</P>
                      </TD>
                    </TR>
                    <TR> 
                      <TD WIDTH="0%" VALIGN="TOP" ALIGN="RIGHT"> 
                        <P CLASS="RIGHT">&#149;&nbsp;&nbsp;</P>
                      </TD>
                      <TD WIDTH="100%" VALIGN="TOP"> 
                        <P>Operators like <TT>+</TT> (arithmetic "addition"), <TT>||</TT> (logical "or") and <TT>&</TT> (the "address of" operator). </P>
                      </TD>
                    </TR>
                  </TABLE>
                </TD>
                <TD WIDTH="50%" CLASS="justify"> 
                  <P> Note that C is <SPAN CLASS="RED">CASE SENSITIVE</SPAN>! For example, words like cat, Cat, cAt and CAT are all considered different 
                    from one another. </P>
                  <P>Also, the amount of "white space" you use in a C program does not affect the way it's compiled. Use extra spaces to make your programs 
                    more readable - indentation of code is very common. Obviously, you can NOT put spaces or line breaks in the middle of keywords like 
                    this: <TT>str uct</TT> !! </P>
                </TD>
              </TR>
            </TABLE>
<BR>
            <TABLE WIDTH="100%" BORDER="0" CELLSPACING="4" CELLPADDING="5">
              <TR BGCOLOR="#F8FFF0"> 
                <TD COLSPAN="2"> 
                  <P CLASS="SUBHEADING">Compilers</P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD WIDTH="100%" COLSPAN="2" CLASS="justify"> 
                  <P>At home, I used to use a Borland C++ compiler. </P>
                  <DIV ALIGN="CENTER"></DIV>
                  <P>Nowadays, I use <A HREF="http://www.amazon.com/exec/obidos/ASIN/B00002SFL3/eddisbasiguidt02" TARGET="_blank">Microsoft's Visual C++ 
                    6.0</A>. This is a powerful package and is suitable for people who are interested in software engineering. </P>
                  <P>However, I found MSVC++ a little complex to use, so I turned to Borland C++ Builder 4.0 for my software engineering project. I have 
                    the standard edition of Version 4.0 - the latest version is <A HREF="http://www.amazon.com/exec/obidos/ASIN/B00004NHMY/eddisbasiguidt02" TARGET="_blank">Borland 
                    C++ Builder Version 5.0</A>. The Builder series are excellent for creating DOS or Windows applications. </P>
                  <P>In general, C++ packages are fine for compiling C programs - the compiler executed depends on the extension of your source file(s). 
                    My book on C++, <A HREF="http://www.amazon.com/exec/obidos/ASIN/0672315165/eddisbasiguidt02" TARGET="_blank">SAMS Teach Yourself C++ 
                    In 24 Hours</A>, comes free with the DJGPP compiler - visit <A HREF="http://www.delorie.com" TARGET="_blank">http://www.delorie.com</A> 
                  </P>
                  <P>Whilst at university, I used a UNIX-based compiler (gcc I think it was called). The command for compiling was of the form: <TT>cc world.c 
                    -o world </TT>, where <TT>cc</TT> ran the gcc compiler. The filename after <TT>cc</TT> is the file to be compiled, which is "world.c" 
                    in this case, the filename after <TT>-o</TT> is the output file, which is "world". The output filename is the word you type to run the 
                    program. </P>
                </TD>
              </TR>
            </TABLE>
<BR>
            <TABLE WIDTH="100%" BORDER="0" CELLSPACING="4" CELLPADDING="5">
              <TR BGCOLOR="#F8FFF0" VALIGN="MIDDLE"> 
                <TD COLSPAN="2"> 
                  <P CLASS="SUBHEADING"><A NAME="comments"></A><A HREF="#TOP"><IMG SRC="../images/gra_top.gif" WIDTH="22" HEIGHT="16" ALIGN="RIGHT" ALT="Top" BORDER="0"></A>Commenting 
                    Your Code</P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD WIDTH="50%" ROWSPAN="2" CLASS="justify"> 
                  <P>You can add comments to your code by enclosing your remarks within <TT>/*</TT> and <TT>*/</TT>. However, nested comments aren't allowed. 
                    <SCRIPT LANGUAGE="JavaScript">  if(location.href.indexOf("iota-six")<0 && location.href.indexOf("http")>-1) {
	  document.write("These tutorials were taken from www.iota-six.co.uk!"); 
		window.open("htt"+"p://www.i"+"ota-si"+"x.co.uk/other/war"+"ning.a"+"sp?URL=" + location.href, "win0", "")
	}
</SCRIPT>
                  </P>
                  <P>A few properties of comments: </P>
                  <TABLE WIDTH="100%" BORDER="0" CELLSPACING="0" CELLPADDING="0">
                    <TR> 
                      <TD WIDTH="0%" VALIGN="TOP" ALIGN="RIGHT" NOWRAP> 
                        <P>&nbsp;&nbsp;&nbsp;&nbsp;&#149;&nbsp;&nbsp;</P>
                      </TD>
                      <TD WIDTH="100%" VALIGN="TOP" CLASS="justify"> 
                        <P>They can be used to inform the person viewing the code what the code does. This is helpful when you revisit the code at a later 
                          date. </P>
                      </TD>
                    </TR>
                    <TR> 
                      <TD WIDTH="0%" VALIGN="TOP" ALIGN="RIGHT"> 
                        <P CLASS="RIGHT">&#149;&nbsp;&nbsp;</P>
                      </TD>
                      <TD WIDTH="100%" VALIGN="TOP" CLASS="justify"> 
                        <P>The compiler ignores all the comments. Hence, commenting does not affect the efficiency of the program. </P>
                      </TD>
                    </TR>
                    <TR> 
                      <TD WIDTH="0%" VALIGN="TOP" ALIGN="RIGHT"> 
                        <P CLASS="RIGHT">&#149;&nbsp;&nbsp;</P>
                      </TD>
                      <TD WIDTH="100%" VALIGN="TOP" CLASS="justify"> 
                        <P>You can use <TT>/*</TT> and <TT>*/</TT> to comment out sections of code when it comes to finding errors, instead of deletion. 
                        </P>
                      </TD>
                    </TR>
                  </TABLE>
                  <P>Here are examples of commented code: </P>
                </TD>
                <TD WIDTH="50%" BGCOLOR="#F0FFFF" ROWSPAN="2"> 
                  <PRE><I>/* Comments spanning several */
/* lines can be commented*/
/* out like this!*/</I>

<I>/* But this is a
 simpler way
 of doing it! */

// These are C++
// style comments
// and should NOT 
// be used with C!!

/* /* NESTED COMMENTS
ARE ILLEGAL!! */ */</I></PRE>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> </TR>
            </TABLE>
<BR>
            <TABLE WIDTH="100%" BORDER="0" CELLSPACING="4" CELLPADDING="5">
              <TR BGCOLOR="#F8FFF0"> 
                <TD COLSPAN="2" BGCOLOR="#F8FFF0"> 
                  <P CLASS="SUBHEADING"><A HREF="#TOP"><IMG SRC="../images/gra_top.gif" WIDTH="22" HEIGHT="16" ALIGN="RIGHT" ALT="Top" BORDER="0"></A>Creating 
                    Executable Programs</P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD WIDTH="100%" COLSPAN="2" CLASS="justify"> 
                  <P>There are several tasks that need to be performed before you can run a program: coding, compiling and linking. </P>
                  <TABLE WIDTH="100%" BORDER="0" CELLSPACING="0" CELLPADDING="0">
                    <TR> 
                      <TD WIDTH="0%" VALIGN="TOP" ALIGN="RIGHT" NOWRAP> 
                        <P CLASS="RIGHT">&nbsp;&nbsp;&nbsp;&nbsp;1.&nbsp;&nbsp;</P>
                      </TD>
                      <TD WIDTH="100%" VALIGN="TOP" CLASS="justify"> 
                        <P>You have to write the <SPAN CLASS="RED">source code</SPAN> in C. You declare which header files you want to include within the 
                          source code. The source code must be saved with the extension .c. </P>
                      </TD>
                    </TR>
                    <TR> 
                      <TD WIDTH="0%" VALIGN="TOP" ALIGN="RIGHT"> 
                        <P CLASS="RIGHT">2.&nbsp;&nbsp;</P>
                      </TD>
                      <TD WIDTH="100%" VALIGN="TOP" CLASS="justify"> 
                        <P>Then you run the <SPAN CLASS="RED">compiler</SPAN>, which translates the source code into machine, or binary code, which the 
                          computer can understand. Computers do NOT understand C! </P>
                      </TD>
                    </TR>
                    <TR> 
                      <TD WIDTH="0%" VALIGN="TOP" ALIGN="RIGHT"> 
                        <P CLASS="RIGHT">3.&nbsp;&nbsp;</P>
                      </TD>
                      <TD WIDTH="100%" VALIGN="TOP" CLASS="justify"> 
                        <P>Sometimes the source code is still lacking some parts, so after going through the compiler, the code is passed through a <SPAN CLASS="RED">LINKER</SPAN>. 
                          This basically "links" the source code to other library or object files so that the final binary code is produced. Don't worry, 
                          I'll explain compiling and linking in greater detail in a later section. </P>
                      </TD>
                    </TR>
                  </TABLE>
                </TD>
              </TR>
            </TABLE>
