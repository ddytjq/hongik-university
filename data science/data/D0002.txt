    No, I code mostly in C++ now. I've found that the biggest development speed-up of Java is it's class library. Having a good class-library in C++ removes that advantage of Java, while opening the door for all the C++ goodies. I have also found that memory management (and resource management in general) in C++ is a lot less of a problem than I had thought earlier using the proper techniques. My disgust for C++ was mainly because had been taught C++ wrong.<br/><br/>I made clear what I was referring to was *supported* bindings. I've made plenty of JNI to not really need any unsupported bindings, I can &quot;easily&quot; make them myself.     Java applications/games are only portable to platforms for which a JRE has been implemented, I don't remember anyone claiming otherwise! <br/><br/>The benefit of Java is that it moves the burden of porting away from the application developer to a JRE provider. If there is enough user demand to run Java applications on a particular platform then it is likely that someone will port a JRE to that platform.
