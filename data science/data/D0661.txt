<td bgcolor="#d2cec4">
<SPAN Class="smbl">
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Handling ActionEvents aren't that big of a
deal. ActionEvents are what you will probably use most, because they are
incredibly easy to use. ActionEvents are most often used for catching simple
mouse clicks. If you are only using clicking, use ActionListener, not
MouseListener.<br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Setting up an ActionListener is not too
hard at all. The first thing you have to do is import the ActionListener class
by putting the following line in your code:<br><br><i>
import java.awt.event.*;</i><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Now that the ActionListener class is
imported, you can implement it. Basically, this means that the applet can be
used as an ActionListener (by objects within that applet). To implement the
ActionListener type the the following line into your applet after extends
Applet:<br><br><i>
implements ActionListener</i><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Do not compile yet or you will get an error
that says you need to declare the class as abstract. If you had this error, it
is because you didn't put public void actionPerformed(ActionEvent e){} into
your code. This is the method that is called whenever an event occurs. An
example is given below of an actionPerformed(ActionEvent e){} method that will
add one to a variables named x every time a button named hello is
clicked.<br><br><i><pre>
public void actionPerformed(ActionEvent e){
	if(e.getSource()==hello){
		x++;
	}	
}</i></pre><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Yummy. You may not understand what that if
statement is doing in there. It is there to decipher whether or not the
ActionEvent happened on the button named hello. If hello was clicked, then the
code in the braces is executed. This is the easiest and simplest way to have
clicks on different object do different things.<br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;One more thing you have to do, and then
you can compile. All the objects that you want to handle events have to have
the ActionListener added to them. If the actionPerformed() you want is defined
within the applet, type in init():<br><br><i>
objectnamehere.addActionListener(this);</i><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Most objects can add ActionListeners. The
only one that I am positive can't is Label. In summary, just remember that the
actual code that handles events in an ActionListener goes in the
actionPerformed() method. An example is shown above. Here is the applet:<br><br>

<applet code=actionevent.class width=100 height=50></applet><br><br>
<pre><small>
//-------------------------------//
//Copyright 2000 FreeJavaHelp.com//
//-------------------------------//

//the next line imports the Applet Class
import java.applet.Applet;
//the next line imports the gui components
import java.awt.*;
//the next line imports the event classes
import java.awt.event.*;
//the next line begins the program
public class actionevent extends Applet implements ActionListener{
    //creates a variable that will have letters added to it
    String x="y";
    //the next line creates a button that will be used to display the text
    Button countbutton=new Button(x);
    //public void init occurs when the applet is initialized
    public void init(){
        //the next line adds countlabel
        add(countbutton);
        //the next line adds an ActionListener to countlabel
        countbutton.addActionListener(this);
    }
    //actionPerfomed() is executed when an action is performed
    //usually a mouse click
    public void actionPerformed(ActionEvent e){
        x+="y";
        //the next line adds one to the text in countlabel
        countbutton.setLabel(x);
    }
}
</pre></small><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;If you have any questions about Event
Handling, we have a section devoted to it in our Forums. Check it out.
