&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;The capability to write text of some sort
is a very important one to a user. Whether it be for taking notes, writing a
report, or something else, text fields are one of the most important GUI
components. Luckily for us, Java makes it very simple to add text components
to our applets.<br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;In the AWT, there are two types of text
components. One of them is a TextField. A TextField is only one line, but
sometimes only one line is ideal, especially for web programming. The
constructors for  TextField are:<br><br><i>
TextField objectnamehere=new TextField();<br>
TextField objectnamehere=new TextField("text goes here");<br>
TextField objectnamehere=new TextField(columns);<br>
TextField objectnamehere=new TextField("text goes here",columns);</i><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;The first constructor creates a simple
TextField with no text already in it, and with no specified width. This does
not do much good, because most of the time you are going to want to have a
specified width. The second constructor is basically the same as the first,
except in between the quotes in the parentheses, you can add text that you
automatically want to appear in the TextField. Of course, you could also use a
String variable instead of the actual text there, but you would have to omit
the quotes.<br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Okay, now we get to the good stuff. The
third constructor allows you to set the width of the TextField. To do this,
enter the number of columns you want it to be in between the parentheses. The
number has to be an integer. The fourth constructor is basically the same as
the third, except it automatically adds text to the TextField. If you wanted
your TextField named main to have thirty columns and say helloworld, you would enter this line into your java file:<br><br><i>TextField main=new TextField("helloworld",30);</i><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;The other type of text component provided
by the AWT is the TextArea. TextArea is ideal for chat rooms, word processors,
or anything that requires a large amount of room for text. The major difference
between TextArea and TextField is that TextField is single lined, where as
TextArea can have as many rows as you like! The constructors for TextArea
are:<br><br><i>
TextArea objectnamehere=new TextArea(rows,columns);<br>
TextArea objectnamehere=new TextArea("text here",rows,columns);</i><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;These constructors aren't too difficult to
understand. For the first constructor in between the parentheses, first you put
the number of rows, then a comma, then the number of columns. The number of
rows and columns have to be integers. The second constructor is very similar to
the first, except it specifies text that will automatically be in the TextArea.
Put the text you want to be in the TextArea in between the quotes. For example,
if I wanted to make a TextArea named welcome with 20 rows, 40 columns, and the
text "welcome to the jungle", I would enter the following into the java
file:<br><br><i>TextArea welcome=new TextArea("welcome to the jungle",20,40);
</i><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Thats about it in terms of making text
components. Check out the applet, and we will talk a little bit about methods
after the applet:


<br><br>
<center>
<applet code=text.class width=350 height=300></applet></center><br><br>
<pre><small>
//-------------------------------//
//Copyright 2000 FreeJavaHelp.com//
//-------------------------------//

//this next line imports the Applet class so that this program can
//be an Applet
import java.applet.Applet;
//the next line imports the GUI components
import java.awt.*;

//the next line begins the program, which will be an Applet and be
//called text
public class text extends Applet{
	//the next line creates a simple text field
	TextField basic=new TextField();
	//the next line creates a text field with 50 columns
	TextField fifty=new TextField(50);
	//this creates a text field with the text Hello World in it
	TextField hello=new TextField("Hello World!");
	//the next lines creates a text field that says Goodbye
	//and has five columns
	TextField goodbye=new TextField("Goodbye",5);
	//the next line creates a  TextArea with 5 rows and 10 columns
	TextArea basictextarea=new TextArea(5,10);
	//the next line creates a TextArea with 3 rows, 12 columns
	//and the text Java Programming
	TextArea javap=new TextArea("Java Programming",3,12);
	//the next line creates a TextField with 20 columns that
	//will have an EchoCharacter
	TextField echo=new TextField(20);
	//the init method occurs when the Applet is initialized
	public void init(){
		//the next line adds the TextField called basic
		add(basic);
		//the next line adds the TextField with 50 columns
		add(fifty);
		//this line adds the TextField called hello
		add(hello);
		//the next line addes the TextField named goodbye
		add(goodbye);
		//the next line adds basictextarea
		add(basictextarea);
		//the next line adds the TextArea called javap
		add(javap);
		//the next line adds the TextField echo
		add(echo);
		//the next line sets the EchoCharacter of echo
		echo.setEchoCharacter('*');
	}
}



</pre></small>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;I tried to comment up the code well enough so that it would be easy to read. I certainly hope it was. Now we are going to move on to the most popular methods for the TextField, which are:<br><br><i>
objectnamehere.setText("text here");<br>
objectnamehere.getColumns();<br>
objectnamehere.getEchoChar;<br>
objectnamehere.setEchoCharacter('*');<br>
objectnamehere.getText();<br>
objectnamehere.setEditable();<br><br></i>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;The first method deletes all the text in a text field, and then enters what you tell it to. For example, if I had a TextField named pizza, I could do pizza.setText("Hello, would you like some pizza"); and the text in between the quotes would be added to the TextField. This is one method that needs to be committed to memory.<br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;The second method simply finds out how many columns are in the TextField. If in the constructor of a specific TextField, I set the number of columns to 20, when I would do the getColumns() method, a value of 20 would be returned.<br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;The third method finds out what character the Echo Characted is.What exactly an Echo Character is is a character, usually a *, that comes in place of the normal letters when you type them. This is ideal if you want to put a password in your program.In case you don't understand, check out the last TextField in the applet above. Type in it, and you will see * instead of the letters you are typing. This is an Echo Character. The fourth method sets the Echo Character. All you have to do is put the character you want to be the Echo Character in between the single quotes. If I had a TextField named basic, and wanted the Echo Character to be *, I would do:<br><br><i>basic.setEchoCharacter('*');</i><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;The fourth method is very useful in user interaction.What it does is get the text that is in the TextField. If I had a TextField named main, and wanted to store the text in a String variable called mytext, I would put the following in a java file:<br><br><i>
String mytext=main.getText();<br><br></i><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Sometimes you may not want the user to be
able to type in a TextField. For example, in a chatroom, the user shouldn't be
able to type in the TextField where the chatting is going on. To keep a user
from writing in a TextField, use the fifth method. For example, if I had a
TextField that was named uneditable, I could put the following in my java code
to make it so the user can't edit it:<br><br><i>
uneditable.setEditable(false);</i><br><br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;If
I wanted to make it editable, I could put the following in my java
code:<br><br><i>
uneditable.setEditable(true);</i><br><br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
Those are the most important methods for TextFields. TextArea has many of the
same methods as TextField. All of the methods above for TextFields, except
getEchoChar() and setEchoCharacter() can be used with TextAreas. Some other
methods that TextAreas have are:<br><br><i>
objectnamehere.append("text here");<br>
objectnamehere.getRows();<br>
objectnamehere.setRows(5);<br></i><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;The first method adds text to the end of
the text that is already in the TextArea. Its quite simple actually. If I had
"hello" in a TextArea already, and wanted to add "how are you", then I could
use the append method.<br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;The second method is used the same way as
the getColumns() method, except it gets the number of rows. The third method is
used the same way as the setColumns() method, except it sets the number of
rows.
