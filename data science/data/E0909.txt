<!--mstheme--></font></td><td valign="top" width="24"></td><!--msnavigation--><td valign="top"><!--mstheme--><font face="Arial, Arial, Helvetica"><h2><!--mstheme--><font face="Verdana, Arial, Helvetica" color="#006666">Order, Order!<font size="3"><br>
      Difficulty: 2 / 10</font><!--mstheme--></font></h2>
<p align="left"><i>Programmers learning C++ often come up with interesting
misconceptions of what can and can't be done in C++. In this example,
contributed by Jan Christiaan van Winkel, a student makes a basic mistake -- but
one that many compilers allow to pass with no warnings at all.</i></p>
<p align="center"><img border="0" src="../images/h-line.gif" width="248" height="2"></p>
<h3><!--mstheme--><font face="Verdana, Arial, Helvetica" color="#006666">Problem<!--mstheme--></font></h3>
<h4><!--mstheme--><font face="Verdana, Arial, Helvetica" color="#006666">JG Question<!--mstheme--></font></h4>
<p><b><font size="4">1.</font></b>  The following code was actually written by a
student taking a C++ course, and the compiler the student was using issued no
warnings about it. Indeed, several popular compilers issue no warnings for this
code. What's wrong with it, and why?</p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp; // Example 1<br>
&nbsp; //</font></p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp;
#include &lt;string&gt;</font></p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp;</font><font face="Courier New">
using namespace std;</font></p>
<p style="margin-top: 0; margin-bottom: 0">&nbsp;</p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp; class
A</font></p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp; {</font></p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp;
public:</font></p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp;&nbsp;&nbsp;
A( const string&amp; s ) { /* ... */ }</font></p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp;&nbsp;&nbsp;
string f() { return &quot;hello, world&quot;; }</font></p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp; };</font></p>
<p style="margin-top: 0; margin-bottom: 0">&nbsp;</p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp; class
B : public A</font></p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp; {</font></p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp;
public:</font></p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp;&nbsp;&nbsp;
B() : A( s = f() ) {}</font></p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp;
private:</font></p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp;&nbsp;&nbsp;
string s;</font></p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp; };</font></p>
<p style="margin-top: 0; margin-bottom: 0">&nbsp;</p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp; int
main()</font></p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp; {</font></p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp;&nbsp;&nbsp;
B b;</font></p>
<p style="margin-top: 0; margin-bottom: 0"><font face="Courier New">&nbsp; }</font></p>
<h4><!--mstheme--><font face="Verdana, Arial, Helvetica" color="#006666">Guru Question<!--mstheme--></font></h4>
<p><b><font size="4">2.</font></b>  When you create a C++ object of class type,
in what order are its various parts initialized? Be as specific and complete as
you can.</p>
<p align="center"><img border="0" src="../images/h-line.gif" width="248" height="2"></p>
<h3><!--mstheme--><font face="Verdana, Arial, Helvetica" color="#006666"><a name="Solution"></a>Solution<!--mstheme--></font></h3>

<p><font color="#999933"><b><font size="4">1.</font></b>   [...]&nbsp; What's wrong with
[this code], and why?</font></p>
<p><font face="Courier New" color="#999933">&nbsp; // ...<br>
<br>
&nbsp;&nbsp;&nbsp; B() : A( s = f() ) {}<br>
<br>
&nbsp; // ...</font></p>
<p>This line harbors a couple of related problems, both associated with object
lifetime and the use of objects before they exist. Note that the expression
&quot;s = f()&quot; appears as the argument to the A base subobject constructor,
and hence will be executed before the A base subobject (or, for that matter, any
part of the B object) is constructed.</p>
<p>First, this line of code tries to use the A base subobject before it exists.
This particular compiler did not flag the (ab)use of A::f(), in that the member
function f() is being called on an A subobject that hasn't yet been constructed.
Granted, the compiler is not required to diagnose such an error, but this is the
kind of thing standards folks call &quot;a quality of implementation issue&quot;
-- something that a compiler is not required to do, but that better compilers
could be nice enough to do.</p>

<p>Second, this line then goes on and merrily tries to use the s member
subobject before it exists, namely by calling the member function operator=() is
being called on a string member subobject that hasn't yet been constructed.</p>

<p><font color="#999933"><b><font size="4">2.</font></b>  When you create a C++ object of class type,
in what order are its various parts initialized? Be as specific and complete as
you can.</font></p>

<p>The following set of rules is applied recursively:</p>

<!--mstheme--></font><!--msthemelist--><table border="0" cellpadding="0" cellspacing="0" width="100%">
  <!--msthemelist--><tr><td valign="baseline" width="42"><img src="../_themes/gotw/aglabul1.gif" width="15" height="15" hspace="13" alt="bullet"></td><td valign="top" width="100%"><!--mstheme--><font face="Arial, Arial, Helvetica">
    <p style="margin-top: 0; margin-bottom: 6">First, the most derived class's
    constructor calls the constructors of the virtual base class subobjects.
    Virtual base classes are initialized in depth-first, left-to-right order.<!--mstheme--></font><!--msthemelist--></td></tr>
  <!--msthemelist--><tr><td valign="baseline" width="42"><img src="../_themes/gotw/aglabul1.gif" width="15" height="15" hspace="13" alt="bullet"></td><td valign="top" width="100%"><!--mstheme--><font face="Arial, Arial, Helvetica">
    <p style="margin-top: 0; margin-bottom: 6">Next, direct base class
    subobjects are constructed in the order they are declared in the class
    definition.<!--mstheme--></font><!--msthemelist--></td></tr>
  <!--msthemelist--><tr><td valign="baseline" width="42"><img src="../_themes/gotw/aglabul1.gif" width="15" height="15" hspace="13" alt="bullet"></td><td valign="top" width="100%"><!--mstheme--><font face="Arial, Arial, Helvetica">
    <p style="margin-top: 0; margin-bottom: 6">Next, (nonstatic) member
    subobjects are constructed, in the order they were declared in the class
    definition.<!--mstheme--></font><!--msthemelist--></td></tr>
  <!--msthemelist--><tr><td valign="baseline" width="42"><img src="../_themes/gotw/aglabul1.gif" width="15" height="15" hspace="13" alt="bullet"></td><td valign="top" width="100%"><!--mstheme--><font face="Arial, Arial, Helvetica">
    <p style="margin-top: 0; margin-bottom: 6">Finally, the body of the
    constructor is executed.<!--mstheme--></font><!--msthemelist--></td></tr>
<!--msthemelist--></table><!--mstheme--><font face="Arial, Arial, Helvetica">
<p>For example, consider the following code. Whether the inheritance is public,
protected, or private doesn't affect initialization order, so I'm showing all
inheritance as public.</p>

<p><font face="Courier New">&nbsp; // Example 2<br>
&nbsp; //<br>
&nbsp; class B1 { };<br>
&nbsp; class V1 : public B1 { };<br>
&nbsp; class D1 : virtual public V1 { };<br>
<br>
&nbsp; class B2 { };<br>
&nbsp; class B3 { };<br>
&nbsp; class V2 : public B1, public B2 { };<br>
&nbsp; class D2 : public B3, virtual public V2 { };<br>
<br>
&nbsp; class M1 { };<br>
&nbsp; class M2 { };<br>
<br>
&nbsp; class X : public D1, public D2 { M1 m1_; M2 m2_; };
</font></p>

<p>The inheritance hierarchy looks like this:</p>

<p><font face="Courier New">&nbsp;&nbsp;B1&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;B1&nbsp;&nbsp;
B2<br>
&nbsp;&nbsp;&nbsp;|&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;|&nbsp;&nbsp; /<br>
&nbsp;&nbsp;&nbsp;| &nbsp;&nbsp;&nbsp; &nbsp;|&nbsp; /<br>
&nbsp;&nbsp; |&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; | /<br>
&nbsp; V1&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; V2&nbsp;&nbsp;&nbsp; B3<br>
&nbsp;&nbsp; |&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; |&nbsp;&nbsp; /<br>
&nbsp;&nbsp; |v&nbsp;&nbsp;&nbsp;&nbsp; v|&nbsp; /<br>
&nbsp;&nbsp; |&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; | /<br>
&nbsp; D1&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; D2<br>
&nbsp;&nbsp;&nbsp; \&nbsp;&nbsp;&nbsp;&nbsp; /<br>
&nbsp;&nbsp;&nbsp;&nbsp; \&nbsp;&nbsp; /<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; \ /<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; X</font></p>

<p>The initialization order for a X object in Example 2 is as follows, where
each constructor call shown represents the execution of the body of that
constructor:</p>

<p><font face="Courier New">&nbsp; first, construct the virtual bases:<br>
&nbsp;&nbsp;&nbsp; construct V1:<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; B1::B1()<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; V1::V1()<br>
&nbsp;&nbsp;&nbsp; construct V2:<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; B1::B1()<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; B2::B2()<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; V2::V2()</font></p>

<p><font face="Courier New">&nbsp; next, construct the nonvirtual bases:<br>
&nbsp;&nbsp;&nbsp; construct D1:<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; D1::D1()<br>
&nbsp;&nbsp;&nbsp; construct D2:<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; B3::B3()<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; D2::D2()</font></p>

<p><font face="Courier New">&nbsp; next, construct the members:<br>
&nbsp;&nbsp;&nbsp; M1::M1()<br>
&nbsp;&nbsp;&nbsp; M2::M2()</font></p>

<p><font face="Courier New">&nbsp; finally, construct X itself:<br>
&nbsp;&nbsp;&nbsp; X::X()</font></p>

<p>This should make it clear why in Example 1 it's illegal to call either A::f()
or the s member subobject's construct.</p>

<h4><!--mstheme--><font face="Verdana, Arial, Helvetica" color="#006666">A(nother) Word About Inheritance<!--mstheme--></font></h4>

<p>Of course, although the main point of this issue of GotW was to understand
the order in which objects are constructed (and, in reverse order, destroyed),
it doesn't hurt to repeated a tangentially related guideline:</p>

<blockquote>
  <p><b>Guideline:</b> Avoid overusing inheritance.</p>

</blockquote>
<p>Except for friendship, inheritance is the strongest relationship that can be
expressed in C++, and should be only be used when it's really necessary. For
more details, see also:</p>
