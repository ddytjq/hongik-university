&nbsp;
<P>

  The chapters in this book center around specific example applications. 
The applications demonstrate advanced programming techniques, 
fully implemented and ready to run (in VB5).
  <P>

  <TABLE WIDTH="100%" CELLPADDING="0" CELLSPACING="0">
    <TR><TD BGCOLOR="#7FCFFF" ALIGN="Center"><B>Part One: Application Basics</B></TD></TR>
  </TABLE>
  <P>

  <DL>
<DT><B>1. ExpenseReporter</B></DT>
<A HREF="avbt/expense_reporter.gif">(show me)</A>
<DD>
The ExpenseReporter application allows the user to fill out 
and print a simple expense report. This example demonstrates:
<UL>
<LI>Scrolling areas
<LI>Dynamic controls and enhanced navigation
<LI>SDI and MDI interfaces
<LI>Loading and saving data
<LI>Recent file list
</UL>
</DD>

<DT><B>2. ExpenseReporter (continued)</B></DT>
<DD>
Chapter 2 continues the discussion of the ExpenseReporter application. It demonstrates:
<UL>
<LI>Validating data and forms
<LI>High resolution printing
<LI>Print preview at different scales
<LI>Help
<LI>Context-sensitive help
<LI>About dialogs
<LI>Splash screens
</UL>
</DD>

<DT><B>3. AppointmentBook</B></DT>
<A HREF="avbt/appointment_book.gif">(show me)</A>
<DD>
AppointmentBook is a meeting and appointment planning calendar that demonstrates:
<UL>
<LI>Using bitmasks
<LI>MVC paradigm for complex interactions
<LI>Creating rotated fonts
<LI>Alarms
<LI>Nonrectangular forms
<LI>HTML help
</UL>
</DD>
</DL>
  <P>

  <TABLE WIDTH="100%" CELLPADDING="0" CELLSPACING="0">
    <TR><TD BGCOLOR="#7FCFFF" ALIGN="Center"><B>Part Two: Add-Ins</B></TD></TR>
  </TABLE>
  <P>

  <FONT SIZE="-1">Note that Microsoft changes the way Add-Ins work so much from release to release
that the details described here for installing an Add-In won't work unless you have VB5.</FONT>

<DL>
<DT><B>4. PropertySetter</B></DT>
<DD>
PropertySetter 
<A HREF="avbt/property_setter.gif">(show me)</A>
is an add-in that allows a developer to quickly examine and 
modify the properties of many controls on a form. It demonstrates:
<UL>
<LI>Creating, testing, and installing add-ins
<LI>Modifying control properties in an add-in
</UL>
</DD>

<DT><B>5. Aligner</B></DT>
<DD>
The Aligner 
<A HREF="avbt/aligner.gif">(show me)</A>
add-in allows a developer to easily arrange controls in 
rows, columns, or rows and columns. It demonstrates:
<UL>
<LI>Arranging controls
</UL>
</DD>

<DT><B>6. Scroller</B></DT>
<DD>
The Scroller 
<A HREF="avbt/scroller.gif">(show me)</A>
add-in allows a developer to easily create a scrolling area. 
The designer simply selects the controls that should be scrolled, and invokes the 
Scroller add-in. Scroller does the rest. This add-in demonstrates:
<UL>
<LI>Creating add-in submenus
<LI>Creating scrolling areas
</UL>
</DD>

<DT><B>7. AddInMaker</B></DT>
<DD>
AddInMaker 
<A HREF="avbt/addin_maker.gif">(show me)</A>
is an add-in that creates other add-ins. Using AddInMaker a developer
can build complex add-ins in a matter of minutes. Creating an add-in to make other 
add-ins provides a good review of add-in creation.
</DD>
</DL>
  <P>

  <TABLE WIDTH="100%" CELLPADDING="0" CELLSPACING="0">
    <TR><TD BGCOLOR="#7FCFFF" ALIGN="Center"><B>Part Three: Databasics</B></TD></TR>
  </TABLE>
  <P>

  <DL>
<DT><B>8. PeopleWatcher</B></DT>
<DD>
PeopleWatcher
<A HREF="avbt/people_watcher.gif">(show me)</A>
is a simple corporate personnel system that demonstrates:
<UL>
<LI>Using databases
<LI>Using the Data control
<LI>Using Data Access Objects (DAO)
<LI>Customizing user interfaces for users with different privileges
</UL>
</DD>

<DT><B>9. Query</B></DT>
<DD>
The Query
<A HREF="avbt/query.gif">(show me)</A>
application is an ad hoc database reporting tool.
It allows the user to enter and execute SQL database 
manipulation statements. This example demonstrates:
<UL>
<LI>Creating databases
<LI>Composing SQL commands
<LI>Processing SQL statements
</UL>
</DD>

<DT><B>10. PeopleWatcher Remote</B></DT>
<DD>
PeopleWatcher Remote is a version of PeopleWatcher that runs
remotely on a network. By centralizing the database, this application
makes database management much easier in a multi-user setting. This version demonstrates:
<UL>
<LI>Accessing remote databases
<LI>Improving network performance
</UL>
</DD>
</DL>
  <P>

  <TABLE WIDTH="100%" CELLPADDING="0" CELLSPACING="0">
    <TR><TD BGCOLOR="#7FCFFF" ALIGN="Center"><B>Part Four: Client/Server Applications</B></TD></TR>
  </TABLE>
  <P>

  <DL>
<DT><B>11. TimeSync</B></DT>
<DD>
TimeSync
<A HREF="avbt/time_sync.gif">(show me)</A>
is a client/server application that allows the user to synchronize 
the system clocks on two computers. This application demonstrates:
<UL>
<LI>Testing client/server applications within the client
<LI>Testing client/server applications locally
<LI>Testing client/server applications remotely
<LI>Trouble shooting client/server applications
</UL>
</DD>

<DT><B>12. QueryServer</B></DT>
<DD>
The QueryServer
<A HREF="avbt/query_server.gif">(show me)</A>
application provides remote database reporting and
ad hoc query services. This application demonstrates:
<UL>
<LI>Building a report list
<LI>Building a librarian to list reports
<LI>Creating query servers to generate reports
</UL>
</DD>

<DT><B>13. AsyncServer</B></DT>
<DD>
AsyncServer provides remote reporting and ad hoc 
query services asynchronously. The user can request
several reports and, if they are served by different computers, 
all of the reports can execute at the same time. This version demonstrates:
<UL>
<LI>Building asynchronous servers in Visual Basic
<LI>Requesting reports
<LI>Generating reports
<LI>Displaying results
</UL>
</DD>
</DL>
  <P>

  <TABLE WIDTH="100%" CELLPADDING="0" CELLSPACING="0">
    <TR><TD BGCOLOR="#7FCFFF" ALIGN="Center"><B>Part Five: ActiveX</B></TD></TR>
  </TABLE>
  <P>

  <DL>
<DT><B>14. The Alarm Control</B></DT>
<DD>
Visual Basic's Timer control can schedule events at most roughly 60 
seconds in advance. The Alarm control allows an application to wait 
for a events much farther in the future. This control demonstrates:
<UL>
<LI>Control creation basics
<LI>Reading and writing property values
<LI>Delegating properties
<LI>Raising events
</UL>
</DD>

<DT><B>15. Bar Gauge</B></DT>
<DD>
The BarGauge
<A HREF="avbt/bar_gauge.gif">(show me)</A>
control allows the user to graphically view and 
modify a numeric value. This control demonstrates:
<UL>
<LI>Drawing the BarGauge
<LI>Handling mouse events
<LI>Handling keyboard events
<LI>Enhancing the BarGauge
</UL>
</DD>

<DT><B>16. Histogram</B></DT>
<DD>
The Histogram
<A HREF="avbt/histogram.gif">(show me)</A>
control uses BarGauges to display several data values. 
It provides its own scrolling area if necessary to display more data than
will fit at one time. This control demonstrates:
<UL>
<LI>Control arrays in custom controls
<LI>Scrolling data
<LI>Delegating to a control array
<LI>Supporting indexed properties
<LI>Raising indexed events
</UL>
</DD>
</DL>
  <P>

  <TABLE WIDTH="100%" CELLPADDING="0" CELLSPACING="0">
    <TR><TD BGCOLOR="#7FCFFF" ALIGN="Center"><B>Part Six: ActiveX on the Web</B></TD></TR>
  </TABLE>
  <P>

  <DL>
<DT><B>17. ActiveX Controls on the Web</B></DT>
<DD>
This chapter tells how Web pages can take advantage of ActiveX controls. It explains:
<UL>
<LI>Ensuring safe use
<LI>Displaying ActiveX controls
<LI>Learning class IDs
<LI>Selecting colors
<LI>Loading data asynchronously
</UL>
</DD>

<DT><B>18. WebSurvey</B></DT>
<DD>
The WebSurvey application presents a Web user with a survey form.
When the user fills out the form, the results are sent to a Visual Basic
program that stores them in a file. This example demonstrates:
<UL>
<LI>Building a survey form
<LI>Sending data across the Web
<LI>Receiving data across the Web
<LI>Packing and unpacking data
</UL>
</DD>

<DT><B>19. SiteMapper</B></DT>
<DD>
SiteMapper explores all of the Web pages it can reach from an initial location.
It creates a list of the pages it visited and the graphic files they include. This example demonstrates:
<UL>
<LI>Retrieving Web documents
<LI>Parsing Web URLs
<LI>Mapping Web sites
</UL>
</DD>
</DL>
  <P>
