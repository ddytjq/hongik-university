<p>Developers using Visual C++ expect a high-performance tool for creating high-performance applications. Control is also important to developers using Visual C++ ; they want to be able to code at as low a level as necessary to get the performance and functionality they need, so broad API support is a must. When performance and control are combined, developers have a truly flexible tool at their disposal.</p>

<h2>Faster Optimizing Compiler</h2>

<p>The optimizing compiler has been a key strength in past versions of Microsoft Visual C++; significant advances in code size and speed were delivered in version 5.0. Visual C++ 6.0 has further improved the performance of the optimizing compiler:</p>
<ul>
<li><b>Compiles are now completed more quickly.</b> Compiler optimizations are retained while compiler throughput on projects with large precompiled headers is significantly faster: as much as 30 percent on debug projects and 15 percent on non-debug projects.</li>
<li><b>Compiled code has a smaller footprint.</b> When developers take advantage of MFC-MFC is currently used by 76 percent of Visual C++ users -the compiler generates smaller code than before due to the increased granularity of MFC. In static linking scenarios, this can mean a decrease in code size of up to 35 percent, with routine real-world reductions of 5-10 percent. Moreover, because DLLs are smaller, load time is improved.</li>
<li><b>Compiled code executes faster.</b>  New optimizing keywords have been added to those already recognized by the compiler. These keywords can be used to ensure applications access the optimizations they need given their target platforms and systems. Further speed enhancements are available through support for Pentium and Pentium Pro operation codes.  That is, the Visual C++ 6.0 compiler understands inline assembly for these processors. Finally, continued attention has been given to tuning existing optimizations to squeeze the best performance out of a developer’s C++ code.</li>
<li><b>Compiled executables are more robust.</b> Multithreaded applications are more robust thanks to the addition of thread-safe shutdown and configurable time-out before exit capabilities.</li>
<li><b>New and improved compiler warnings.</b> Warnings have been added and upgraded to help catch some common programming errors and flag typical problem spots, including:</li>
<ul> 
<li>Empty if statements (new)</li>
<li>Incorrect or missing assignments (new)</li>
<li>Unreferenced variables (improved)</li>
<li>Invalid type specifiers (improved)</li>
<li>Redefintion (improved)</li>
<li>Templates (improved)</li>
</ul></ul>
<h2>Delay Load Imports</h2>

<p>The Delay Load Imports option of the Visual C++ 6.0 linker can create code to automatically manage delay loading of DLLs, which is particularly useful when an application links to DLLs but doesn’t necessarily use them. With Delay Load Importing, DLLs are not loaded until they are required to continue execution. This obviates the requirement for custom code to specifically load a library prior to its use. Moreover, it speeds application start times when certain DLLs are used infrequently.</p>

<h2>Support for Standards</h2>

<p>As C++ has evolved from C, it has become the de facto standard for professional programming in systems ranging from embedded products to mainframes. It has also undergone a great deal of refinement and standardization.</p> 

<h3>Language Standards</h3>

<p>C++ is now an ANSI/ISO standard. Microsoft committed two engineers to contribute to the ISO/ANSI standards effort. These engineers provided a significant amount of input during the draft process and gave feedback to other engineers working on Visual C++ at Microsoft. Microsoft supports the C++ standard through its implementation (in Visual C++) of the C++ language and the Standard Template Library (STL). Microsoft Visual C++ support for ANSI/ISO C++ includes the newest keywords and types, including bool, explicit, false, mutable, true, and typename. Array new and delete have also been added.</p>

<h3>Platform, Internet and other Standards</h3>

<p>Microsoft platform standards are another important standards area for Visual C++ customers. In fact, the majority of Visual C++ customers surveyed felt that support for Microsoft standards was "important" or "very important."  Visual C++ includes support for:</p>
<ul>
<li>Microsoft standards: ActiveX, COM/DCOM, Active Server Pages, MFC, OLE DB, ADO</li>
<li>Internet standards: HTTP, FTP, XML, HTML, Dynamic HTML, Gopher</li> 
<li>Data standards: SQL, ODBC</li> 
</ul>
