<H2><A NAME="quickstart"></A> <A NAME="s4">4. Quick Start steps </A></H2>

<P>To implement a project in object oriented PHP do -
<UL>
<LI> First you need connection to database sql servers - use one of these:
<UL>
<LI> <B>ADODB</B> (Active Data Objects Data Base) 
<A HREF="http://php.weblogs.com/ADODB">http://php.weblogs.com/ADODB</A></LI>
<LI> <B>Metabase</B> (Database independent access and management)
<A HREF="http://phpclasses.upperdesign.com/browse.html/package/20">http://phpclasses.upperdesign.com/browse.html/package/20</A></LI>
</UL>

<P>
</LI>
<LI> Second you need generic PHP classes to create forms, tables and other
HTML objects. Get these from 
<B>PHP classes</B> at 
<A HREF="http://phpclasses.upperdesign.com">http://phpclasses.upperdesign.com</A>.
See also the Top downloaded classes from
<A HREF="http://phpclasses.upperdesign.com/browse.html/top">Top PHP classes</A>.
And see Form classes, template classes at 
the site 
<A HREF="http://www.milkywaygalaxy.freeservers.com">corporate Intranet/Internet</A>

<P>
</LI>
<LI> Third, design and create your own PHP classes by inheriting from the
generic 
<A HREF="http://phpclasses.upperdesign.com">PHP classes</A>.
<P>
</LI>
<LI> Fifth, use IDE tools from 
<A HREF="PHP-HOWTO-14.html#PHP IDE">PHP IDE</A>

<P>
</LI>
<LI> Most of your PHP code should be in classes for easy code maintainence and re-usability.</LI>
</UL>
<P>See also 
<A HREF="http://phpclasses.upperdesign.com/browse.html/author/35463">Aldev's PHP classes</A><P>
