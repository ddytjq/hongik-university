        <p><b>Feb-08-2002: CSocket class source code is available</b></p>
        <p>Go to <a href="../winsock-api/csocket-class/csocket-class-01.asp">
        this page</a> to download the FTP Client sample application that uses 
        the CSocket class instead of the Winsock Control.</p>
        <p><b>Jan-08-2002: Sample Application is modified</b></p>
        <p>Thanks a lot to everyone who provided bug reports and improvements 
        for this project. The application is more stable now, but it is not 
        perfect though. Below is a list of some fixes:</p>
        <ul type="square">
          <li>The program recognizes messages and comments sent by an FTP server 
          through the control connection.<br>
&nbsp;</li>
          <li>Sometimes you could get the error message from an FTP server &quot;425 
          Can't open data connection&quot;. The problem is that this sample sent the 
          PORT command with number of port which was already in use by this 
          application or another one running on the local system. The problem is 
          fixed, the program checks a port for availability before to send the 
          PORT command.<br>
&nbsp;</li>
          <li>When the program is downloading a file, it firstly creates a new 
          local file and then establishes the data connection, so no more lost 
          data on fast Internet connections. That was a big mistake to design 
          that thing in another way.<br>
&nbsp;</li>
          <li>The file and directory names with blank space characters are 
          welcome now. Thanks to Philippe GERARD for his function SplitSpecial.<br>
&nbsp;</li>
          <li>The CTimeOut class was improved and works better now.<br>
&nbsp;</li>
          <li>The REST command is not sent now on any download or upload 
          operation when you don't need to restart a data transfer.<br>
&nbsp;</li>
          <li>Files are uploading properly now.</li>
        </ul>
        <p>Actually, I can't remember all the changes I have made, so just 
        download the new archive and see yourself how it works. I hope you'll 
        love it. Unfortunately, two big problems still exist:
        </p>
        <ul type="square">
          <li>There is no standard format of the FTP file and directory 
          listings, so this task is up to you. Some guys are working on 
          producing the RFC document that should bring some order in that chaos, 
          but this work still is in an experimental stage. The CFtpConnection 
          class can recognize a couple of well known formats used by FTP servers 
          running on UNIX and Windows/DOS systems.<br>
&nbsp;</li>
          <li>The second problem is about deployment of the apps and components 
          built with the CFtpConnection class. Thanks a lot to
          <a href="http://www.vbip.com/comment.asp?94">Tim Hill</a> and
          <a href="http://www.vbip.com/comment.asp?102">Nick Thompson</a> for 
          the trick with the .srg file. But it still is not clear to us: &quot;Is it 
          legal to register the Winsock Control in such a way?&quot;. Moreover, some 
          people report that they cannot properly deploy the application even 
          using the trick mentioned above. So, the only way I see to resolve 
          this problem is to develop a replacement of the Winsock Control :). Is 
          it possible with Visual Basic? I think so; we'll see.</li>
        </ul>
        <p>The Comments section on this page is not suitable for any kind of 
        discussion. So, please, if you have any questions, tips, and ideas, 
        don't use that website's feature. Instead, go to the forum where I have 
        set up the <a href="http://www.vbip.com/forum/group.asp?Group=18">
        discussion group devoted to this sample application</a>.</p>
        <p align="center"><img border="0" src="images/ftp_explor_screen-02.gif"></p>
        <p><b>Mar-06-2000: Sample Application is modified<br>
        </b>Many thanks to Vincent Lozada for his improvements. Read his message
        below, in the Comments section.&nbsp;
        </p>
        <p>It's just an example of how to implement FTP with Winsock Control and
        Visual Basic. If you are looking for a stable component it is not for
        you, but if you want to develop one, you are in the right place. This
        sample uses the CFtpConnection class, a wrapper class that encapsulates the Winsock Control
        calls and the FTP syntax; you can look at the <a href="winsock_ftpclass_ref.asp"> quick reference</a> to learn
        more about that class module.&nbsp;
        </p>
        <p>I suppose the CFtpConnection class will be useful for you in the
        current stage, otherwise I hope it is a good template for your own
        component. Anyway, I think it is a good chunk of source code :).
        </p>
        <p>To understand how this class was developed you should read another
        page at this website: <a href="winsock_ftp_01.asp"> &quot;Secrets of FTP (Practical introduction to
        FTP)&quot;</a>. If you are looking for the info about how to use the Winsock
        Control, go to the <a href="index.asp"> Winsock Control index page</a> and start at
        <a href="winsock_control.asp"> the first article</a>. Since you know how the Winsock Control and the FTP work, you
        can begin to develop your own component or application.	
        </p>
        <p>Why Winsock? Because, if you want to get all the features of the FTP
        without third-party software, you have not a choice. I think, neither
        the Internet Transfer Control nor the WinInet API can not bring the
        complete functionality of the FTP to your Visual Basic application.	
        </p>
        <p>Unfortunately, I have not much time to test this app and the CFtpConnection class. I hope you will&nbsp; help me to make it better.
        Please, post your feedback, comments and questions below.	
        </p>
