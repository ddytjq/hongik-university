<CENTER><H3><FONT COLOR="#AOAO99">
DELEGATES and C++
</FONT></H3></CENTER><HR>

<!-- Author and contact details -->

This article was contributed by
<A HREF="mailto:fabien.lebourg@eole-soft.com">Fabien Lebourg</A>.


<!-- For which environment is this code specific??? -->
<p><u>Environment:</u> [eg VC6 SP4, W98 W2K NT ...]

<p>
<!-- The 'p' starts a paragraph of normal text -->

<B>Presentation:</B> <BR><BR>
This article shows an implementation of delegates. What is a delegate? An object
that acts like a multi-function pointer with a subscription system. It really
simplifies the use of static or 'object' member function pointers for callback
notifications and event handling. While implementation is a bit complicated, the
use is very intuitive:
<BR>

<!-- start a block of source code -->
<B>Quick Sample:</B>
<PRE>
class CFoo {
public:

   <span class="codeComment">// declaration of member callback</span>
   void Foo( int );

};

<span class="codeComment">// declaration of static callback</span>
void Foo( int i );


void main( ) {

   CFoo obj;

   ...

<span class="codeComment">   // Declare a delegate for functions
   // with one argument of type int</span>
   CDelegate1< int > delegate;

<span class="codeComment">   // Add Foo static function</span>
   delegate.Add( Foo );
<span class="codeComment">   // Add Foo member function based on foo object</span>
   delegate.Add( obj, CFoo::Foo );

   ...

<span class="codeComment">   // Execute the callbacks</span>
   delegate( 0 );

   ...

<span class="codeComment">   // Remove Foo static function</span>
   delegate.Remove( Foo );
<span class="codeComment">   // Remove Foo member function</span>
   delegate.Remove( obj, CFoo::Foo );

}</PRE>

<p>
<B> How to?<BR><BR></B>
You just have to add the headers to your project and include the "Delegates.h" header
in your C++ code (in stdafx.h for example) to gain access to these classes: <BR><BR>
<table border="1" bordercolor="#000000" style="border: thin" align="center">
    <tr>
      <td></td>
      <td><b>No Multi-Threading</b></td>
      <td><b>Multi-Threading (with MFC)</b></td>
    </tr>
    <tr>
      <td><b>No argument</b></td>
      <td><div align="center">CDelegate0</div></td>
      <td><div align="center">CMTDelegate0</div></td>
    </tr>
    <tr>
      <td><b>1 argument</b></td>
      <td><div align="center">CDelegate1&lt; ARG &gt; </div></td>
      <td><div align="center">CMTDelegate1&lt; ARG &gt; </div></td>
    </tr>
    <tr>
      <td><b>2 arguments</b></td>
      <td><div align="center">CDelegate2&lt; ARG1, ARG2 &gt; </div></td>
      <td><div align="center">CMTDelegate2&lt; ARG2, ARG2 &gt; </div></td>
    </tr>
</table>
</p>
<BR>
<BR>

<p>
<B>Notes:<BR><BR></B>
a) STL is used for multi-platform implementation. <BR>
b) C++ does not authorize variable length template argument lists and that's
why I made different implementations for delegates with zero, one or two arguments.
If someone knows how to do in another way, just let me know.<BR>
c) C++ only defines scope resolution operator for base classes. We have to
specify the conversion in this particular case:<BR>

<PRE>
class CFoo
{
   ...
   void Foo( int ) { }
   ...
   void Init( CDelegate1<int>& delegate )
   {
<span class="codeComment">      // ERROR: CFoo:: operator does not exist here</span>
      delegate.Add( *this, CFoo::Foo );

<span class="codeComment">      // CORRECT: type cast is available</span>
      delegate.Add( *this, (void (CFoo::* )( int )) Foo );
   }
   ...
};</PRE>

To solve this little "read/write complication" we can define a typedef in our class declaration:<BR>

<PRE>class CFoo
{
   ...
   typedef MEMBER_FUNCTION_CALL_TYPEDEF1( CFoo, int ) OwnHandler;
   ...
};</PRE>
<BR>
So we could have:<BR>
<PRE>delegate.Add( *this, (OwnHandler) Foo );</PRE>
<BR>
Actually the 'base' macros are: <BR>
<pre>MEMBER_FUNCTION_CALL_TYPEDEF0 for 0 arg member function
MEMBER_FUNCTION_CALL_TYPEDEF1 for 1 arg member function
MEMBER_FUNCTION_CALL_TYPEDEF2 for 2 args member function</pre>
<BR>
You could also define a macro with MEMBER_FUNCTION_CALLx in your cpp file...
<BR><BR>
<B>Conclusion:<BR><BR></B>
I hope this code will help you in your programming. I'm working at EOLE SA and this method
helped me in my work because it's lighter and less restrictive than interface definition/derivation
style of notifications (connection points).<BR>
Thanks for your attention. Please report any bugs!<BR>
