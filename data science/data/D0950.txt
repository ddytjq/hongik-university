<h2>
Lesson: Overview of Applets
</h2>
<blockquote>
This lesson discusses the parts of an applet.
If you haven't yet compiled an applet and included it in an HTML page,
you might want to do so now.
Step by step instructions are in 
<a target="_top" href="../../getStarted/applet/index.html">The Anatomy of an Applet</a><a target="_top" href="../../getStarted/applet/index.html"><img src="../../images/tutorialIcon.gif" width=11 height=11 border=0 align="ABSMIDDLE" alt="(in the Writing Applets trail)"></a>.
<p>

Every applet is implemented by creating a subclass of the <code>Applet</code> class.
The following figure shows the inheritance hierarchy of the <code>Applet</code> class.
This hierarchy determines much of what an applet can do and how,
as you'll see on the next few pages.

<p><center><IMG SRC="../../figures/applet/1classes.gif" WIDTH="158" HEIGHT="172" ALIGN="BOTTOM" NATURALSIZEFLAG="3" ALT=""></center></p>
</blockquote>
<h3>A Simple Applet</h3>
<blockquote>
Below is the source code for an applet called <code>Simple</code>.
The <code>Simple</code> applet displays a descriptive string
whenever it encounters a major milestone in its life,
such as when the user first visits the page the applet's on.
The pages that follow
use the <code>Simple</code> applet and build upon it
to illustrate concepts that are common to many applets.
If you find yourself baffled by the Java source code,
you might want to go to 
<a target="_top" href="/docs/books/tutorial/java/index.html">Learning the Java Language</a><a target="_top" href="/docs/books/tutorial/java/index.html"><img src="../../images/tutorialIcon.gif" width=11 height=11 border=0 align="ABSMIDDLE" alt="(in the Writing Applets trail)"></a> to learn about the language.
<p>

<blockquote><pre>
/*
 * 1.0 code.
 */

import java.applet.Applet;
import java.awt.Graphics;

public class Simple extends Applet {

    StringBuffer buffer;

    public void init() {
	buffer = new StringBuffer();
        addItem(&quot;initializing... &quot;);
    }

    public void start() {
        addItem(&quot;starting... &quot;);
    }

    public void stop() {
        addItem(&quot;stopping... &quot;);
    }

    public void destroy() {
        addItem(&quot;preparing for unloading...&quot;);
    }

    void addItem(String newWord) {
        System.out.println(newWord);
        buffer.append(newWord);
        repaint();
    }

    public void paint(Graphics g) {
	//Draw a Rectangle around the applet's display area.
        g.drawRect(0, 0, size().width - 1, size().height - 1);

	//Draw the current string inside the rectangle.
        g.drawString(buffer.toString(), 5, 15);
    }
}
</pre></blockquote>


</blockquote>
<h3><a href="lifeCycle.html">The Life Cycle of an Applet</a>
</h3>
<blockquote>

You can use the <code>Simple</code> applet to learn
about the milestones in every applet's life.
</blockquote>

<h3><a href="appletMethods.html">Methods for Milestones</a>
</h3>
<blockquote>

The <code>Applet</code> class provides a framework for applet execution,
defining methods that the system calls
when milestones -- major events in an applet's life cycle -- occur.
Most applets override some or all of these methods
to respond appropriately to milestones.
</blockquote>

<h3><a href="componentMethods.html">Methods for Drawing and Event Handling</a>
</h3>
<blockquote>
Applets inherit the drawing and event handling methods
of the AWT <code>Component</code> class.
(AWT stands for Abstract Windowing Toolkit; 
applets and applications use its classes
to produce user interfaces.)
<em>Drawing</em> refers to anything related to 
representing an applet on-screen --
drawing images,
presenting user interface components such as buttons,
or using graphics primitives.
<em>Event handling</em> refers to detecting and processing
user input such as mouse clicks and key presses,
as well as more abstract events
such as saving files and iconifying windows.
</blockquote>

<h3><a href="containerMethods.html">Methods for Adding UI Components</a>
</h3>
<blockquote>
Applets inherit from the AWT <code>Container</code> class.
This means that they are designed to hold <code>Components</code> --
user interface objects
such as buttons, labels, pop-up lists, and scrollbars.
Like other <code>Containers</code>, applets use layout managers
to control the positioning of <code>Components</code>.
</blockquote>

<h3><a href="security.html">What Applets Can and Can't Do</a>
</h3>
<blockquote>
For security reasons, 
applets that are loaded over the network
have several restrictions.
One is that an applet can't ordinarily read or write files
on the computer that it's executing on.
Another is that an applet can't make network connections
except to the host that it came from.
Despite these restrictions,
applets can do some things 
that you might not expect.
For example, applets can invoke 
the public methods of other applets on the same page.
</blockquote>
