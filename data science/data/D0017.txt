<h2>Run the microbenchmarks</h2>

<b>Note:</b> now that Doug Bell's more comprehensive
<a href="http://www.javaworld.com/javaworld/jw-04-1997/jw-04-optimize.html">Benchmark
applet</a> is out, I'm no longer recording times for these microbenchmarks.
You can get a lot more information from Doug's applet -- but it takes a while
longer :-).
<p>

This applet runs the last version of the <a href="benchmarks.html">Java
Microbenchmarks</a>.  Make sure that your machine is idle before starting the
applet; it will take about 45 seconds to run.  Each of the microbenchmarks is
run three times for one second at a time, and the maximum number of operations
completed within a second is used to calculate the time per operation.

<p>
<center>
<applet code="MicrobenchApplet.class" width=500 height=355>
[Your browser doesn't run Java]
</applet>
</center>
<p>

For an analysis of the results for a range of Java implementations on a single
machine, see <a href="benchmarks.html">Java Microbenchmarks</a>.

<p> The tables below show results in microseconds contributed by <a
href="#contributors">other users</a> for v1.1 of the Java microbenchmarks (see
also <a href="microbenchold.html">older results</a> for previous versions).
Lower times are better.  I've separated the results into x86 and non-x86
architectures, grouped results for similar platforms, and put them in rough
order of speed.  Browser names are abbreviated:
<ul>
  <li>NN = <a href="http://www.netscape.com/comprod/products/navigator/">Netscape
      Navigator</a>
  <li>IE = <a href="http://www.microsoft.com/ie/">Internet Explorer</a>.
  <li>JDK = Java Development Kit
  <li>MRJ = Macintosh Runtime for Java
  <li>CW = Metrowerks Java (CodeWarrior)
</ul>

I've highlighted the names of browsers that use a just-in-time compiler.  <p>
