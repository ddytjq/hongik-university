      <h1>The Archive</h1>
      <p align="center">&nbsp;&nbsp;&nbsp; This area has files 
        and source code for programs I make.&nbsp; If you have any questions about 
        programming in , Pascal(DOS), or C/C++(DOS or Windows), feel free to <a href="contact.htm">contact me</a>.</p>
      <p align="center"><strong>Keep in mind that most programs 
        here require CWSDPMI to run in pure DOS mode. Windows does not require 
        it, however. You can </strong><a
href="ftp://ftp.simtel.net/pub/simtelnet/gnu/djgpp/v2misc/csdpmi4b.zip"><strong>download</strong></a><strong> it from this 
        link to ftp.Simtel.net or the </strong><a
href="http://www.delorie.com/djgpp/"><strong>DJGPP</strong></a><strong> page.</strong></p>
      <p align="center">All items here have been tested with DJGPP. 
        Most, if not all should work on any ANSI standard compiler (if it doesn't 
        use Allegro.) Programs saying &quot;Alleg req&quot; need DJGPP and Allegro 
        to <strong>COMPILE</strong>, but not to run the EXE.</p>

      <h2>Libraries</h2>
      <p align="center"><a href="zips/jrnd.zip" name="jrnd"><font
size="5">jrnd.zip</font></a><font size="5"> -- header file required to compile 
        most of my programs. v2.1. Last updated: 3/8/00</font></p>
      <p align="center">It's been over a year since I've last updated this library, 
        and now with the new portable Allegro I have been finding it more and 
        more difficult to use the ANSI random functions as they yield different 
        results and in different ranges. So now jrnd has gotten a lot better in 
        version 2.1. The most notable change in the system is that it comes with 
        its own public domain RNG, so it will yield identical results on any ANSI 
        compiler on any platform. Also jrnd is now C compatable. If you are running 
        C++ you can take advantage of the overloaded functions but now C users 
        can still use it through alternately named calls (since there are no overloaded 
        funcs in C). The last change is that jrnd.h and jrnd2.h have been merged 
        together, so you can get random numbers in 32-bit integer, floating point, 
        or fixed point formats all from the same include.</p>
      <p align="center"><strong>Version 2.1 fixes major bugs in version 2.0 when 
        using with Allegro and other includes and code, and now has .cpp version 
        of rndalgor to add to your project if you prefer not to use jrndglob.h.</strong></p>
      <p align="center"><a href="#contents"><strong>Back to top</strong></a></p>
      <p align="center"><a href="zips/linklist.zip" name="linklist_H"><font
size="5">Linklist.zip</font></a><font size="5"> -- Templated library for use with 
        large data. v2.1 Updated: 3/8/00</font></p>
      <p align="center">This class primarily uses pointers, but 
        it also has alternate functions for returning by reference. Has comments 
        for nearly EVERY LINE, good for learning a lot of c++. Overloaded functions, 
        operators, reference, returning by reference, templates, pointers, memory 
        allocations, inline optimized functions.</p>
      <p align="center"><strong>Latest Changes:<br>
        </strong>Added Save and Load functionality to save or load your place 
        in the list.</p>
      <p align="center"><strong>I want to caution you that I made 
        this lib two years ago, and although it does work perfectly and very fast, 
        it is not very robust -- If you do something wrong then the list will 
        not pick it up, like when using Save or Load, or the _* functions. I recommend 
        using the std::list C++ class now if you can.</strong></p>
      <p align="center"><a href="#contents"><strong>Back to top</strong></a></p>
      <p align="center"><a href="zips/qgui.zip" name="QuickGUI"><font
size="5">QuickGUI</font></a><font size="5"> -- Easy to create dialogs. v1. Updated: 
        11/8/99</font></p>
      <p align="center">This very small library works over the 
        BGUI and Allegro libraries to help with creating a more interactive GUI 
        program in Allegro. This was made for the MUUDPIC GUI chat interface, 
        but is releasable seperately.<strong> Includes .a file compiled on 11/8/99 
        with latest DJGPP, BGUI, and Allegro on those dates.</strong></p>
      <p align="center"><strong>Features:</strong><br>
        self-updating GUI textbox, just use the insertion operator. (TextBox &lt;&lt; 
        &quot;stuff&quot;;)<br>
        Super quick input using the insertion operator. (InputBox &gt;&gt; data;)<br>
        A simple wrapper for the alert dialog when you just want that one-lined 
        quick OK box.</p>
      <p align="center"><strong>NOTE:</strong> Rhide project files 
        are included but if you use them change the directories properly. Right 
        now there is no documentation, but everything DOES work right and well. 
        Look at the example file and .h file to see how to use the lib. This library 
        requires <a
href="http://www.concentric.net/~Skis/" target="_top">BGUI</a> and <a
href="http://www.talula.demon.co.uk/allegro/" target="_top">Allegro</a> libraries to work.</p>
      <p align="center"><a href="#contents"><strong>Back to top</strong></a></p>
      <p align="center"><a name="JPEG"><font size="5">JPEG library</font></a><font
size="5"> by Eric Vannier. 2/3/98. v1.03</font></p>

      <p align="center"><a href="zips/libjpeg.tar.gz">Linux Version</a>

      <h2>Networking</h2>
      <p align="center"><a href="zips/chat.zip" name="Chat"><font
size="5">Chat.zip</font></a><font size="5"> -- Chat on the internet. Last Updated: 
        7/19/99</font></p>
      <p align="center"><strong>Note: This program is going to 
        be pretty much scrapped in the next few days because it might has a bug 
        in it (it may have been the VXD?) and I forgot to include the VXD needed! 
        (lame me). However, the source is still good to look at for beginning 
        BSD sockets. MUUDPIC will basically replace this program and outperform 
        it in every way possible.</strong></p>
      <p align="center">This zip includes both a chat server, chatserv, 
        and a chat client, chatcli. In order to use these across the internet, 
        you need to know your IP address, which can be found be using winipcfg.exe 
        in Windows 95/98. This program will NOT work in Windows NT because of 
        the differences in the Winsock interface. If you have Windows 98, or a 
        later version of Windows 95 (that has Winsock 2 installed), copy the csock.vxd 
        file into the Windows/System directory. This is a virtual device driver 
        that is run ONLY when this program runs, so it won't take up memory.</p>
      <p align="center">To use, one computer runs chatserv, and 
        the other chatcli &lt;IP&gt; where &lt;IP&gt; is the server's IP address. 
        To test it over one computer, open two windows, chatserv in one and chatcli 
        127.0.0.1 in the other. The only major bug is that it is VERY slow, and 
        that is due to a serious bug in the libsocket library that I test, but 
        do not program.</p>
      <p align="center">This includes the source files since it 
        is primarly for a learning tool for BSD sockets.</p>

      <h2>Profiling</h2>
      <p align="center"><a href="files/fixtest.cpp" name="fixtest"><font
size="5">Fixtest.cpp</font></a><font size="5"> -- Allegro's fixed vars with important 
        results! v1. 12/9/98.</font></p>
      <p align="center">This test program shows very interesting 
        results concerning Allegro's fixed math routines, and their speed in C++. 
        Seems the fix class in C++ is useless speed-wise as compared to floating 
        point!!! But the C-style fixed typedef is MUCH MUCH faster!!! This compares 
        floats to doubles to Allegro's fixed in various ways. The results are 
        in the comments at the top of the code so click the link to see them and 
        the code below it.</p>
