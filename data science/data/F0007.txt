<p>About Geneart :</p>
<p>Geneart allows you to create beautiful images by step-by-step improvement of calculated images during an evolutionary process where you are the force of selection.</p>
<p>Use the upper right button to create a totally new species. Use the button showing the "biohazard" symbol ti create five new mutations of the parent in the center. Click on a child on one of the five outer images to make it new parent. Click on the parent image to enlarge. Click on the backward arrow to undo up to nine steps. Click on the "Disk" button for archive operations. Click on one of the three buttons on the left side to "lock" certain genes (linear, sine, bitshift, from top to bottom).</p>
<p>To generate images, proceed as follows : click on the "double-helix" - creation button until a vaguely appealing image occures. Make this image parent, click the "biohazard" - mutation button until a vaguely better image occurs in one of the children. Make this child parent, then proceed.</p>

<h3>Algorithms</h3>
<a HREF="../../downloads/paths.zip" name="paths">Paths</a> by <a href="nls@inode.at">Wolfgang Kienreich</a><br>
<i>A pathfinder demonstation using popular algorithms.</i><br>
<strong>Wolfgang's additional note :</strong><br>
I'm afraid a little error crept into my pathfinding sample. The public function "DistanceEstimate" in the class "CSquare" contains the following line ...<br>
DestinationDistanceEstimate = Int(Sqr(nDX ^ 2 + nDY ^ 2))<br>
It has to be corrected to the following for the AStar algorithm to work properly...<br>
DestinationDistanceEstimate = Int(Sqr(nDX ^ 2 + nDY ^ 2)*10)<br>
Explanation: AStar uses the heuristic estimate of the remaining distance from a given square to the destination square to process "interesting" squares first. This is a method guaranteed to find the shortest path possible, as long as the heuristic estimate is *always* longer than the shortest path that realy exists. Unfortunately, this is not the case if the original formula is used. In addition, because of clipping all the decimals, the original function did also not calculate well for close distances (one or two squares).<br>
If you publish my sample, please include this explanation in the description and express my apologies to the downloaders.<br>
