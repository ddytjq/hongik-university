    <td valign="top" width="100%"><font size="2" face="Arial"><strong>OCX Helper: Solution to
    &quot;Object Server not correctly registered&quot; (Error 336) in VB4</strong><br>
    <br>
    Have you been looking for a solution to that annoying &quot;Object server not correctly
    registered&quot; (Error 336) error message when selecting Custom Controls from the Visual
    Basic 4.0 Tools menu? Well here's an answer...<br>
    <br>
    Each error message VB4 displays corresponds to a particular control whose TypeLib key is
    either empty or missing under it's parent CLSID (class identifier) key in the Registry.<br>
    <br>
    OCX Helper was created to find the offending control entries for you. It doesn't fix them,
    that's your job. But it does include Help file that offers very simple steps to help you
    correct these entries. And it's freeware!<br>
    <br>
    Version 1.01 is a maintenance release. OCX Helper v1.01 now runs correctly on Windows NT4.<br>
