<p class=ChapterTitle><span style='mso-ansi-language:EN-US'>Simple Recordsets -
Reading Data <o:p></o:p></span></p>

<p class=MsoNormal>So you've decided that ASP-ADO is the solution for your
situation, and you've learned how to set it up. Now you're ready to begin using
ASP-ADO to access and manipulate data within web pages. This chapter discusses
the simplest situation – the simple task of <span class=ImportantWords><span
style='font-size:9.0pt;mso-bidi-font-size:10.0pt;font-family:Times;color:black'>reading</span></span>
the data. Although we are only reading the data, we can still use it in several
ways. Not only can we write it to the page; we can also use the data in
variables, expressions and as the arguments for other functions. </p>

<p class=MsoNormal><![if !supportEmptyParas]>&nbsp;<![endif]><o:p></o:p></p>

<p class=MsoNormal>Then we look at some of the techniques and problems with
presenting the data that we read in an HMTL table. Last, even though VBScript
implements only parts of Object Oriented Programming, we'll include a few
paragraphs to provide an overview of the aspects of OOP that are available.</p>

<h1><a name="_Toc445024323"><span style='mso-ansi-language:EN-US'>Preparing to
use Simple Recordsets</span></a><span style='mso-ansi-language:EN-US'><o:p></o:p></span></h1>

<p class=MsoNormal>One of my sailing coaches emphasized that yacht <!--[if supportFields]><span
style='mso-element:field-begin'></span><span style="mso-spacerun:
yes"> </span>XE &quot;recordsets:preparation&quot; <![endif]--><!--[if supportFields]><span
style='mso-element:field-end'></span><![endif]-->races are won or lost long
before the day of the race. Regattas are won during the months of preparation
and training prior to the starting gun. Fortunately, ADO does not require many
weeks of practicing tacks and jibes in the cold rain; you just have to consider
the following three issues:</p>

<p class=MsoNormal><![if !supportEmptyParas]>&nbsp;<![endif]><o:p></o:p></p>

<p class=Bullets>Creating a DSN</p>

<p class=Bullets>Having the correct User Identification, Password and
Permissions</p>

<p class=Bullets>Understanding the structure of the database</p>

<h2><a name="_Toc445024324"><span style='mso-ansi-language:EN-US'>Step 1 -
Create a DSN</span></a><span style='mso-ansi-language:EN-US'> <o:p></o:p></span></h2>

<p class=MsoNormal>In Chapter 2 we talked about creating an <!--[if supportFields]><span
style='mso-element:field-begin'></span><span style="mso-spacerun:
yes"> </span>XE &quot;recordsets:preparation:DSN&quot; <![endif]--><!--[if supportFields]><span
style='mso-element:field-end'></span><![endif]--><!--[if supportFields]><span
style='mso-element:field-begin'></span><span style="mso-spacerun:
yes"> </span>XE &quot;DSN:creating&quot; <![endif]--><!--[if supportFields]><span
style='mso-element:field-end'></span><![endif]-->ODBC Data Source which is
referred to by a DSN – this is typically performed by a systems administrator
or other operator with access to the server. To review: the DSN contains the
information necessary to make a physical connection into the database. This
information may include database name, path, server name, User ID, Password, Driver
name and other parameters..<span style="mso-spacerun: yes">  </span>These
pieces of information are wrapped up in a DSN, which is given a name. You need
to get the name of your DSN from your systems administrator prior to using
ASP-ADO.</p>

<p class=MsoNormal><![if !supportEmptyParas]>&nbsp;<![endif]><o:p></o:p></p>

<p class=MsoNormal>If you are working on PWS you have access to the settings of
your Windows OS and can set your own DSN (also discussed in Chapter 2), by
selecting <span class=ScreenText><span style='font-size:9.5pt;mso-bidi-font-size:
10.0pt;font-family:Helvetica'>Start | Control Panel | 32 bit ODBC</span></span>
and following the steps of the wizard. In my development work I keep two copies
of the database – one on my development machine using PWS and one on the
server. </p>

<p class=MsoNormal><![if !supportEmptyParas]>&nbsp;<![endif]><o:p></o:p></p>

<p class=Background><span lang=EN-GB>In Chapter 6, we will talk about how to
make a connection without a DSN.</span></p>

<h2><a name="_Toc445024325"><span style='mso-ansi-language:EN-US'>Step 2 - User
Identification and Password</span></a><span style='mso-ansi-language:EN-US'><o:p></o:p></span></h2>

<p class=MsoNormal>I'm frequently <!--[if supportFields]><span
style='mso-element:field-begin'></span><span style="mso-spacerun:
yes"> </span>XE &quot;recordsets:preparation:user ID and password&quot; <![endif]--><!--[if supportFields]><span
style='mso-element:field-end'></span><![endif]--><!--[if supportFields]><span

style='mso-element:field-begin'></span><span style="mso-spacerun:

style='mso-element:field-end'></span><![endif]-->approached by people seeking
help with ADO, only to find that the problem is not with their ADO code, but

security requirements of the database. Generally this means that you present a
User Identification (<span class=CodeinText><span style='font-size:9.0pt;

Security can be established at various levels, but you will need to get this
information from the owner or systems administrator of the database. </p>

<h2><a name="_Toc445024326"><span style='mso-ansi-language:EN-US'>Step 3 -
Understand the Structure of the Database</span></a><span style='mso-ansi-language:
EN-US'><o:p></o:p></span></h2>

<p class=MsoNormal>Another obvious (but frequently overlooked)<!--[if supportFields]><span
style='mso-element:field-begin'></span> XE
&quot;recordsets:preparation:database structure&quot; <![endif]--><!--[if supportFields]><span
style='mso-element:field-end'></span><![endif]--><!--[if supportFields]><span
style='mso-element:field-begin'></span><span style="mso-spacerun:
yes"> </span>XE &quot;databases:ADO errors&quot; <![endif]--><!--[if supportFields]><span
style='mso-element:field-end'></span><![endif]--><span style="mso-spacerun:
yes"> </span>point of preparation is understanding the structure of the database.
ADO will always produce errors if your commands don't use the exact spelling of
the tables, queries, views and fields. A more subtly but equally deadly error
arises when commands to the database conflict with its relationships between
tables. In Chapter 5 we'll look at a technique for finding out the names of the
fields, but you still have to know the names of the tables. The structures of
the tables used in this book are provided in Appendix A.</p>

<p class=MsoNormal><![if !supportEmptyParas]>&nbsp;<![endif]><o:p></o:p></p>

<h4><span lang=EN-GB>Common Errors in</span><!--[if supportFields]><span
lang=EN-GB><span style='mso-element:field-begin'></span> XE
&quot;recordsets:preparation:common errors&quot; </span><![endif]--><!--[if supportFields]><span
lang=EN-GB><span style='mso-element:field-end'></span></span><![endif]--><span
lang=EN-GB><span style="mso-spacerun: yes"> </span>preparation:</span></h4>

<p class=MsoNormal><![if !supportEmptyParas]>&nbsp;<![endif]><o:p></o:p></p>

<p class=Bullets>Incorrect spelling of DSN</p>

<p class=Bullets>DSN no longer exits, or name has been changed</p>

<p class=Bullets>The DSN will not work if the file is moved to another location
after the DSN is created (applies to Access and Excel files).</p>

<p class=Bullets>DSN is of User or File type rather than a System DSN</p>

<p class=Bullets>UserID or password is misspelled or incorrect</p>

<p class=Bullets>Names of fields or tables are incorrectly known by plural or
singular (e.g. Author instead of Authors)</p>

<p class=Bullets>Type (number/text/date) of fields is not known correctly</p>

<h1><a name="_Toc445024327"><span style='mso-ansi-language:EN-US'>Syntax for
Simple Recordsets</span></a><span style='mso-ansi-language:EN-US'><o:p></o:p></span></h1>

<p class=MsoNormal>Once you have properly prepared for using ADO, creating the
simplest recordsets only requires three lines of code. Here's a first example:

</span>' code for members</span></p>

</span>' code for non-members</span></p>

<p class=MsoNormal><![if !supportEmptyParas]>&nbsp;<![endif]><o:p></o:p></p>

<p class=MsoNormal>In our last case we switch to retrieving data from a
different field. The member field was established (at the time the database was
designed) as of type True/False. Therefore it will return a value of true or
false, which can be directly used as a whole expression. If the database
contains true the code for members will be run.</p>

<p class=Bullets>Errors in upper/lower case for data stores that are case

<p class=Bullets>Quotes around numerical values</p>
