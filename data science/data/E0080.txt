<table><tr><td align=left><img border=0 src="pictures/buttons/cppsm.gif" alt="[online C++ tutorial]"></td></tr></table><center><h2>Glossary</h2></center>
<dl>
<dt> <b> <a name="algorithm">algorithm</a> </b> </dt>
<dd>  A series of steps that systematically outline how 
	to accomplish a goal.
 </dd>
		
<dt> <b> <a name="assembler">assembler</a> </b> </dt>
<dd>  a piece of software that translates a specific 
	<a href="#assembly_language">assembly language</a> into 
	<a href="#machine_language">machine language</a> 
	for a specific processor.
 </dd>
		
<dt> <b> <a name="assembly_language">assembly language</a> </b> </dt>
<dd>  a set of instructions that a programmer
	can use to create a program for use on a specific 
	processor. Assembly language is translated into
	machine language by a piece of software called an
	assembler
 </dd>
		
<dt> <b> <a name="bit">bit</a> </b> </dt>
<dd>  The smallest possible unit of memory.  A bit can store only
	two different values - a "0" or a "1".  If a bit could
	store only one possible value, it would not store any information
	at all, because each bit would look identical.  However, a
	bit storing one of two possible values can be combined with
	lots of other bits to store huge quantities of information.
	The term for a group of eight bits, a common occurrence,
	is called a <a href="#byte">byte</a>. 
 </dd>
		
<dt> <b> <a name="byte">byte</a> </b> </dt>
<dd>  A small unit of memory.
	A byte is composed of eight <a href="#bit">bits</a>.  
	One bit
	can store two different combinations (0 or 1), two bits can store
	four different combinations (00, 01, 10, 11), three bits can store 
	eight
	different combinations (000, 001, 010, 011, 100, 101, 110, 111);
	following the pattern,
	eight bits, or one byte, can store 256 different values.
 </dd>
		
<dt> <b> <a name="casting">casting</a> </b> </dt>
<dd>  The act of viewing (and often converting) the information 
	contained in a variable of one <a href="#type">type</a> as if that 
	information was of another type.
 </dd>
		
<dt> <b> <a name="class">class</a> </b> </dt>
<dd>  ...
 </dd>
		
<dt> <b> <a name="comment">comment</a> </b> </dt>
<dd>  These are put into source code by programmers in order 
	to better explain what the code does. When source code is
	<a href="#compiled">compiled</a> into 
	<a href="#assembly_language">assembly language</a>, all comments 
	are removed. Comments
	only describe code, there is no action associated with them.
	Since comments are removed by the compiler, they can be
	written in plain English (or any other natural language
	you choose!).
 </dd>
		
<dt> <b> <a name="compiled">compiled</a> </b> </dt>
<dd>  Once third-generation languages have been translated
	into assembly code and then to machine code, they are said
	to have been compiled. Loosely, this could also be used to
	refer to any single intermediate step.
 </dd>
		
<dt> <b> <a name="compiler">compiler</a> </b> </dt>
<dd>  a piece of software that takes third-generation
	language code and translates it into a specific 
	<a href="#assembly_language">assembly
	code</a>. Compilers can be quite complicated pieces of software. 
 </dd>
		
<dt> <b> <a name="constructor">constructor</a> </b> </dt>
<dd>  ...
 </dd>
		
<dt> <b> <a name="cout">cout</a> </b> </dt>
<dd>  In the C++ language, this is the name given to the standard
	output stream. When a buffer of characters is sent to this 
	stream, they will appear as output on the terminal the program
	was ran from.
 </dd>
		
<dt> <b> <a name="debug">debug</a> </b> </dt>
<dd>  the act of removing errors, logical mistakes, or 
	flow of control problems in a program.
 </dd>
		
<dt> <b> <a name="decimal">decimal</a> </b> </dt>
<dd>  a number represented as a whole number and some fractional
	part as well. Decimal numbers either take up more memory than 
	integers, or are accurate to less places, or both.
 </dd>
		
<dt> <b> <a name="execute">execute</a> </b> </dt>
<dd>  The act of starting up and running a certain program.
	This involves loading the program from disk into main memory,
	reserving memory space for the program to work with, and
	sequentially executing the steps of the program as outlined 
	by the programmer
 </dd>
		
<dt> <b> <a name="function">function</a> </b> </dt>
<dd>  ... </dd>
		
<dt> <b> <a name="garbage">garbage</a> </b> </dt>
<dd>  an all-encompassing term used to denote variable values 
	that are undefined, or bad input into a program.
 </dd>
		
<dt> <b> <a name="gcc">gcc</a> </b> </dt>
<dd>  this is the name of the gnu C++ <a href="#compiler">compiler</a>. 
	This free compiler is
	frequently used on machines running the UNIX operating 
	system.  The latest version of gcc can be
	<a href="ftp://prep.ai.mit.edu/pub/gnu/">downloaded
	via ftp</a>.
 </dd>
		
<dt> <b> <a name="instruction_set">instruction set</a> </b> </dt>
<dd>  a set of commands that a certain <a href="#CPU">CPU</a> 
	understands.
	These are very basic instructions that are wired into the 
	logic of the CPU.
 </dd>
		
<dt> <b> <a name="integer">integer</a> </b> </dt>
<dd>  a whole number value. In programming languages, integers
	also have a maximum value, depending on the language definition
	or the architecture of the machine that the code is compiled
	on
 </dd>
		
<dt> <b> <a name="keyword">keyword</a> </b> </dt>
<dd>  a word that has some sort of predefined meaning in the
	context of a programming language
 </dd>
		
<dt> <b> <a name="machine_language">machine language</a> </b> </dt>
<dd>  a processor specific set of binary codes
	that correspond to actions to be taken by the processor of a 
	machine.
 </dd>
		
<dt> <b> <a name="main">main</a> </b> </dt>
<dd>  In C++, this is the first function that is called when a 
	program is executed. When a program returns from main, 
	execution has been completed.
 </dd>
		
<dt> <b> <a name="object">object</a> </b> </dt>
<dd>  ...
 </dd>
		
<dt> <b> <a name="operator">operator</a> </b> </dt>
<dd>  Acts on one or more subexpressions to produce a value. 
 </dd>
		
<dt> <b> <a name="parameter">parameter</a> </b> </dt>
<dd>  a piece of data that is passed into a program,
	subroutine, or function call.
 </dd>
		
<dt> <b> <a name="path">path</a> </b> </dt>
<dd>  A path is used to specify where certain files can be found
	in a directory tree. Sometimes path is used to mean where 
	an application looks to find certain files, such as "make
	sure stdio.h is in your path, or the compiler won't find it."
 </dd>
		
<dt> <b> <a name="portable">portable</a> </b> </dt>
<dd>  a description of how easy it is to take
	code written on one machine or system and
	correctly compile it on another
	machine or system.  This process is called "porting".
 </dd>
		
<dt> <b> <a name="preprocessor_directive">preprocessor directive</a> </b> </dt>
<dd>  a command placed within a source code 
	listing, that directs the compiler to do something before
	the rest of the source code is parsed and compiled.
 </dd>
		
<dt> <b> <a name="private">private</a> </b> </dt>
<dd>  ...
 </dd>
		
<dt> <b> <a name="protected">protected</a> </b> </dt>
<dd>  ...
 </dd>
		
<dt> <b> <a name="pseudo-code">pseudo-code</a> </b> </dt>
<dd>  a high-level abstraction of code, usually used to 
	outline the general steps in an <a href="#algorithm">algorithm</a>
	without having to
	write actual code (usually done for the reader's 
	or programmer's benefit).
 </dd>
		
<dt> <b> <a name="public">public</a> </b> </dt>
<dd>  ...
 </dd>
		
<dt> <b> <a name="public_interface">public interface</a> </b> </dt>
<dd>  ...
 </dd>
		
<dt> <b> <a name="return_type">return type</a> </b> </dt>
<dd>  specifies the type of data that is returned from a
	function call. 
 </dd>
		
<dt> <b> <a name="type">type</a> </b> </dt>
<dd>  describes what sort of information a <a href="#variable">variable</a>
	stores, as
	well as how much space that information takes up.
 </dd>
		
<dt> <b> <a name="UNIX">UNIX</a> </b> </dt>
<dd>  A multi-processing, multi-user, family of operating systems 
	that run on a variety of architechtures. A shareware version
	of this operating system, Linux, runs on the IBM compatible 
	PC.
 </dd>
		
<dt> <b> <a name="variable">variable</a> </b> </dt>
<dd>  a name, given by a programmer, to represent a piece of 
	data within a certain program. This name is then used
	to refer to that piece of data.
 </dd>
		
<dt> <b> <a name="Windows">Windows</a> </b> </dt>
<dd>  A family of operating systems created by Microsoft, that
	run on Intel based IBM compatible machines. 
 </dd>
		
</dl>
