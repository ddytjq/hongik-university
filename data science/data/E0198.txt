<p>Visual C++ 6.0 includes several features specifically intended to enhance developer productivity where it matters most&#151;without sacrificing flexibility, performance, or control. Significant innovations in the integrated development environment (IDE), the debugger, and the many wizards in Visual C++:</p>
<ul>
<li><b>Make it easier than ever to write C++ code.</b> Enable the IDE to assist the developer in writing C++ code by providing statement completion, function and member information, and easy API access. Provide the most up-to-date graphical view of the code for easier navigation.</li>
<li><b>Provide powerful, flexible debugging tools that save significant development time.</b> Make debugging as intuitive and flexible as possible by allowing developers to change code while debugging and see the changes reflected in that same debugging session. Simplify application status information and provide detailed information on what modules are loaded.</li>
<li><b>Quickly generate functional, full-featured applications.</b> Provide developers with the widest range of project options and then make creation of these projects as simple as possible without sacrificing control. Reduce development time by generating much of the code automatically, based on the chosen options.</li>
<li><b>Capitalize on the richness and functionality of existing applications.</b> Simplify the encapsulation and extension of existing applications with an absolute minimum requirement for new code, while allowing end users to utilize familiar techniques in familiar environments.</li>
<li><b>Support multiple monitors and efficient automation.</b> Allow developers to configure the tool as they see fit and provide a simple mechanism to automate common, repetitive tasks.</li>
</ul>

<h2>Performance and Control</h2>

<p>
Users of Visual C++ cite the optimizing compiler, fast compile and link times, and small, fast executables as three of their top five reasons to use Visual C++.  Clearly, the performance of generated applications and of the tool itself are important to developers using Visual C++. To exploit the performance Visual C++ can provide to their applications, developers want the ability to code directly to Windows APIs.  Making available a large number of APIs and supporting standards is thus very important. When developers combine control and performance they are able to produce the fastest and most scalable code for distributed, enterprise scenarios. In these areas, Visual C++ 6.0 focuses on:</p>
<ul>
<li><b>Enhancing compiler throughput and flexibility.</b> Decrease compile time to shorten the overall turnaround cycle for a given project. Support custom scenarios by compiling and building as flexibly as possible.</li> 
<li><b>Generating the fastest compiled code.</b> Continue to optimize the compiler to provide developers with the fastest and smallest code by supporting additional optimizations and special features of key microprocessors.</li>
<li><b>Improving application performance with more control over DLL loading.</b> Enable applications to load only the code they need when they need it for better application performance.</li>
<li><b>Supporting important standards and APIs.</b> Make APIs more discoverable and usable in the environment. Support key language and operating systems standards, as well as hardware, network, legacy, database, and Internet standards to meet the diverse needs of enterprise developers.</li>
</ul>

<h2>Latest Platform, Internet, and Component Development Support</h2>

<p> 
The majority of customers using Visual C++ identify support for the latest Microsoft user interface innovations as a key factor in deciding to use Visual C++.  Support must be up to date for professional developers to meet their customers' requirements and to avoid sacrificing usability and functionality. These customers seek frameworks that can encapsulate user interface innovations and provide consistent usage models. Because many of these customers rely on multitier Internet or intranet solutions as key aspects of their businesses, tool support for component development and the latest Internet and Web technologies are essential. Visual C++ 6.0 relies on libraries and a powerful component object model to provide:</p>
<ul>
<li><b>Access to tested and proven UI elements and controls.</b> Support the latest controls and simplify their use with graphical placement and manipulation. Make these user interface innovations part of the platform application framework to ensure they are accessible.</li>
<li><b>Support for the latest Web technologies.</b> Evolve Web support to include the latest standards like Dynamic HTML. Support the creation of applications and content that have a Web-based (browser) usage model.</li>
<li><b>Support for Web server development.</b> Allow developers to write code to customize Internet Information Server and include up-to-date, native support for the core Internet protocols.</li>
<li><b>Support for combining controls.</b> Reduce coding requirements by allowing controls to be easily combined and reused. Make this functionality available for ActiveX and Windows controls.</li>
<li><b>Simplified component development and expedited code reuse.</b>
Extend COM support and allow for wizard-based creation of a wide range of COM objects.</li>
</ul>

<h2>Enterprise Application Development</h2>

<p>As the use of information technology changes to meet new business needs, the very definition of &quot;enterprise applications&quot; is also evolving. While enterprise infrastructures and applications vary widely across organizations, common attributes of &quot;enterprise&quot; applications include:</p>
<ul>
<li>They are component based for flexibility and scalability.</li>
<li>They require life cycle support, including design, development, management, and analysis.</li>
<li>They require more sophisticated database architectures.
<li>They are developed by teams.</li> 
<li>They are distributed in nature and integrated with disparate back-end systems.</li>
</ul>

<p>Developers using Visual C++ who acquire Visual Studio, Enterprise Edition receive additional enterprise development features to address these needs including a complete suite for component-based development, enterprise database features, enhanced lifecycle productivity inclusive of application design, performance analysis, and deployment tools, team development support, and development versions of the Microsoft BackOffice family of applications servers.</p>

<p>Visual C++ 6.0 also provides additional new features for building enterprise applications. These features provide the most comprehensive control of execution and scalability, as well as support for revolutionary platform technologies and flexible data programming. In fact, Visual C++ provides the most thorough support for Microsoft's Universal Data Access strategy&#151;including ADO and OLE DB&#151;to provide efficient access to all types of data on the widest variety of systems.</p>

<p>The productivity advances in Visual C++ 6.0 dramatically increase turnaround time while making coding C++ much faster and more accessible&#151;without generating unnecessary code or making assumptions about what the developer is trying to do. Of course, control at the lowest level is present if developers choose to access it. Visual C++ includes numerous productivity enhancements in the IDE, in standards and technology accessibility, and in hardware support.</p>

<h2>IntelliSense Technology</h2>

<p>Five IntelliSense features in Visual C++ 6.0 speed development time by monitoring what is typed and by assisting with code syntax, parameter information, APIs, and more:</p>
<ul>
<li><b>Complete word</b> automatically completes the rest of a variable or function name once it has identified enough characters to do so.</li> 
<li><b>Auto list members</b> opens a drop-down menu in the editor that contains a list of valid member variables or functions for the selected class or structure. When this feature is used on a blank line, a global list of methods and members is displayed, including system API functions, C++ classes, and variables. Selecting from the list inserts the member into the code.</li>
<li><b>Auto parameter info</b> displays the complete prototype, including a parameter list, for a function when the open parenthesis is typed immediately after the function name. The parameter in bold indicates the next parameter required as you type the function. With overloaded functions, the parameter list gives the user a choice of which function to use.</li>
<li><b>Auto type info</b> displays the complete declaration for any identifier in a pop-up window when the mouse cursor is positioned over the identifier.</li> 
<li><b>Code comments</b> displays comments in a ToolTip window for the functions selected in the members drop-down list (see above). By default, any comments in the source code that use the &quot;//&quot;, &quot;/*&quot;, and &quot;*/&quot; delimiters are displayed.</li>
</ul>

<h2>Dynamic ClassView Updating</h2>

<p>Dynamic ClassView updating increases developer productivity and has the added benefit of relaying the most up-to-date information to the programmer. The ClassView tab of the Project Workspace window displays the classes and their members for all projects on the workspace. When a new class definition, variable, or member function is added to a source file, ClassView is automatically updated to reflect the existence of the new item. No rebuild is required. The opposite scenario also applies: when an addition or deletion is made in ClassView, the source code is automatically updated to reflect the change. Navigation is also improved because developers can now jump directly from a class that implements a dialog box to the dialog editor. Finally, classes are more easily modified with the member function deletion available through the ClassView pop-up menu and the WizardBar.</p>

<p>Both the new IntelliSense technology and Dynamic ClassView updating in Visual C++ 6.0 rely on the new Dynamic Parsing capability of the Visual C++ IDE. Dynamic Parsing parses code as it is typed to update the IDE automatically. More than simply providing C++ code snippets like some other tools, Dynamic Parsing allows Visual C++ to intelligently predict, assist, and focus the coding process. Developers can easily turn off the features enabled by Dynamic Parsing.</p>

<h2>Powerful and Flexible Debugging Tools</h2>

<p>Users of Visual C++ cite the integrated debugger as the feature most important to them in Visual C++ and the feature most likely to bring them to Visual C++ for their next development project.  With Visual C++ 6.0, the debugger has been updated to provide developers with even greater productivity and detailed information about the state of their system and their code.</p>

<h3>Edit and Continue</h3>

<p>Edit and Continue allows developers to change source code during a debugging session and then continue-without forcing an end to the session, a subsequent rebuild, and reinstantiation of the debugger. For example, while debugging, a user could change the number of times a particular loop is executed, and the Edit and Continue feature will allow the debugging session to continue uninterrupted. Changes are simply recompiled and applied to the executable's image in memory. Edit and Continue dramatically improves turnaround, especially on large projects, by saving developers the time it takes to stop and restart the debugger, rebuild, and return the application to the state it was in when the bug occurred.</p>

<p>Making Edit and Continue possible required significant changes to many parts of Visual C++. In addition to the creation of an Edit and Continue engine, changes were made in the Debugger, Compiler Driver, Compiler Front End, Compiler Back End, Project/build system, Linker, Object file interfaces, Editor, and PDB files. Thorough testing was completed to ensure:</p>
<ul>
<li><b>Stability and robustness.</b> If a failure occurs, users must be able to continue the original debugging session.</li>
<li><b>Accuracy.</b> Changes in the memory image should only occur where necessary.</li>
<li><b>Dependability.</b> Edit and Continue should occur only when a rebuild is not required and should be optional.</li> 
<li><b>Efficiency.</b> Edit and Continue must be faster than a minimal rebuild to be advantageous to users.</li>
<li><b>Usability.</b> The functionality must be understandable and support developer preferences and input settings.</li>
</ul>

<h3>Module List</h3>

<p>Windows-based programs often rely on dynamic link libraries (DLLs) to provide resources or other functionality. Developers writing these programs need to know which DLLs are being loaded, especially when a bug or crash occurs in unfamiliar code. The Modules selection in the Debug menu shows developers which DLLs are loaded. Developers can then identify which DLL was being executed when the crash occurred. When the origin of a DLL is unknown or it is only in a release (non-debug) form, this feature can save developers from needless debugging.</p>

<h3>Additional Debugger Productivity Improvements</h3>

<p>A host of other additions to the debugger make it more approachable and productive for developers. Variants are now automatically displayed in their correct form (for example, integers are displayed numerically and a BSTR appears as a string). Pointers to functions and v-table entries are displayed symbolically (including parameters) rather than as hex addresses. In addition, formatting is now supported in the Watch window so variable names can be displayed in their true form-a character can be seen as a character rather than a memory address, for example.</p>

<h2>New and Enhanced Wizards</h2>

<p>Visual C++ includes many wizards to make developers more productive in completing the widest range of tasks. For Visual C++ 6.0, several new wizards have been added to address new technologies supported in Visual C++, and many existing wizards have been enhanced.</p>

<h3>New Wizards</h3>

<p>Visual C++ 6.0 adds several new project wizards to assist developers in creating new projects:</p>
<ul>
<li><b>Cluster Resource Type Wizard</b> simplifies the development of applications that can be managed and monitored by Microsoft Cluster Server on Windows NT clusters. This wizard generates two projects: one for the resource being created and another for adding graphical management capabilities for the new resource type to the Cluster Administrator.</li>
<li><b>Extended Stored Procedure Wizard</b> creates an Extended Stored Procedure for use with Microsoft SQL Server databases.</li> 
<li><b>Utility Project Wizard</b> creates Utility projects, which by default neither contain files nor generate output files. They can be used as master projects for subprojects or as containers for files that can be built without a link step.</li>
</ul>

<h3>Enhanced Wizards</h3>

<p>Enhancements to other project wizards facilitate developer access to the latest technologies and more efficient use of the development environment:</p>
<ul>
<li><b>ATL COM AppWizard</b> now includes a checkbox for Microsoft Transaction Server support. Checking this box makes Visual C++ generate project build settings that support Microsoft Transaction Server (see the Enterprise Application Development section for more information on Microsoft Transaction Server).</li>
<li><b>MFC AppWizard,</b> one of the most commonly used wizards in Visual C++, has been enhanced to provide more support for new technologies:</li>
<ul> 
<li><i>Disabling of the default Document/View Architecture.</i> Sometimes applications do not use the traditional document paradigm. With this option, developers can take advantage of MFC while avoiding the inclusion of unnecessary functionality that would increase an application's size.</li>
<li><i>Windows Explorer-style applications.</i> Developers can use the wizard to create a two-pane application-like the Windows (File) Explorer-for them. This type of application is particularly well suited to situations in which hierarchical relationships and specific entity information must be related.</li>
<li><i>Support for Internet Explorer 4.0 (or higher)-style &quot;Rebar&quot; controls.</i> The Rebar is the type of toolbar users are familiar with from Internet Explorer 4.0 (or higher). Rebars contain &quot;bands&quot; that can be repositioned to show or hide controls or information.</li>

This class wraps the Web-browsing functionality of Microsoft Internet Explorer 4.0 (or higher) and allows users to easily create MFC applications with HTML or Dynamic HTML front ends.</li>
<li><i>Additional database connectivity through OLE DB.</i> If database support is added to an MFC application in Step 2 of the MFC AppWizard, OLE DB may be chosen as the Data Source type. For example, a developer could choose a Microsoft Exchange mail server data store as the data source if an OLE DB provider for Exchange was installed.</li>
<li><i>Check box for Active Document Container creation.</i> MFC applications can be made to act as Active Document Containers (see <i>Active Document Containment</i> described below) by selecting this option.</li>

<h2>Active Document Containment</h2>

<p>The Active Document Containment support in Visual C++ 6.0 allows developers to seamlessly provide standard functionality from applications like Microsoft Excel and Microsoft Word in their own MFC applications. Specifically, Active Document Containment is a technology that provides a single frame in which to work with documents, instead of forcing the developer to create and use multiple application frames for each document type. With Active Document Containment, an entire document is activated (that is, an entire application, including associated menus, toolbars, and so on) within the context of a single frame. An Active Document (provided by an Active Document Server such as Word or Excel) is essentially a full-scale, conventional document that is embedded as an object within an Active Document Container. Unlike embedded objects, Active Documents have complete control over their pages, and the full interface of the application (with all its underlying commands and tools) is available to the user for editing. Active Document Containment is a benefit for end users because they can manage disparate information as a single entity, which can be uniquely named, saved, printed, and so on. Moreover, developers can provide end users with the functionality and richness they are accustomed to from other applications while minimizing the amount of code they need to write on their own.</p>

<font size="1">Figure 5: Developers can include rich functionality in their MFC applications by making them Active Document containers. Active Documents from Word, Excel, and other applications can be hosted.</font></p>

<h2>Enhanced Monitor and Keyboard Support</h2>

<h3>Multiple Monitors</h3>

<p>Visual C++ now supports the display of its windows on multiple monitors.  By placing the debugger and the application being debugged on separate screens, developers can avoid obscuring their applications. Thus, a developer stepping through code can see the entire user interface of both the debugger and their application. Previously, developers had to ALT-Tab to switch between the debugger and their application or resize both windows to fit on a single screen at the same time.</p> 

<h3>Quick Macro Recorder</h3>

<p>In response to numerous requests from customers, simplified access to the Visual Basic Script macro recording functionality of the Visual C++ development environment is available in Visual C++ 6.0 through new Quick Macro Recording. To record a macro, &quot;Record Quick Macro&quot; is selected from the Tools menu. A floating toolbar with stop-and-pause functionality then appears in the IDE. Replaying the recorded operations is performed by choosing &quot;Play Quick Macro&quot; from the Tools menu or invoking a redefinable accelerator key. By bypassing the regular macro recording interfaces, Quick Macro Recording allows developers to record simple macros efficiently, just as a simple keystroke recorder would.</p>
