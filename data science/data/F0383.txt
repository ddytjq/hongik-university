<FONT FACE="Palatino" SIZE=2><P>"In Hungarian Notation," I said, "the name of the object begins with a prefix (usually 3 characters) that describes the object (in our case </FONT><FONT FACE="Helvetica" SIZE=2>frm</FONT><FONT FACE="Palatino" SIZE=2>) and the rest of the name should meaningful describe it. For that reason, all of my form names begin with </FONT><FONT FACE="Helvetica" SIZE=2>frm</FONT><FONT FACE="Palatino" SIZE=2>. Typically, I give my first form the name </FONT><FONT FACE="Helvetica" SIZE=2>Main</FONT><FONT FACE="Palatino" SIZE=2>. Therefore </FONT><FONT FACE="Helvetica" SIZE=2>frmMain</FONT><FONT FACE="Palatino" SIZE=2> is the name of the one and only form of the China Shop Project."</P>
<P>"Can a form's </FONT><FONT FACE="Helvetica" SIZE=2>Name</FONT><FONT FACE="Palatino" SIZE=2> contain spaces?" Mary asked.</P>
<P>"That's a good question and the answer is no," I replied. "In Visual Basic, the </FONT><FONT FACE="Helvetica" SIZE=2>Name</FONT><FONT FACE="Palatino" SIZE=2> property of any object cannot contain spaces. However, remember that the </FONT><FONT FACE="Helvetica" SIZE=2>Name</FONT><FONT FACE="Palatino" SIZE=2> property and the external form and project file names are different. In Windows 95 and Windows NT, it is permissible to create a file name with spaces in it. However, the </FONT><FONT FACE="Helvetica" SIZE=2>Name</FONT><FONT FACE="Palatino" SIZE=2> property may not have spaces."</P>
</FONT><B><I><FONT FACE="FranklinGothic" SIZE=4><P><A NAME="_Toc412024750"><A NAME="_Toc413993247">The Height and Width Properties</A></A></P>
</B></I></FONT><FONT FACE="Palatino" SIZE=2><P>We had previously discussed both the Height and Width properties. They affect the dimensions of the form.</P>
<P>"The values for these properties are displayed in the table as 5700 (about 4") and 8205 (about 5½") respectively," I said. "I want you to feel free to deviate from these exact dimensions if you feel the need. I'm not out to get carbon-copy projects from you all at the end of the course. Those values are just guidelines until you feel comfortable. The important thing is to design your form so that the controls we are going to place on it next week all fit."</P>
</FONT><B><I><FONT FACE="FranklinGothic" SIZE=4><P><A NAME="_Toc412024751"><A NAME="_Toc413993248">The BorderStyle, ControlBox, Moveable Properties</A></A></P>
</B></I></FONT><FONT FACE="Palatino" SIZE=2><P>"We're going to change some properties of the form," I said, "that will prevent the user (or the customer) from doing three things:"</P></DIR>


<UL>

<UL>
<LI>changing the size of the form</LI>
<LI>maximizing the form or moving the location of the form </LI>
<LI>minimizing or closing the form </LI></UL>
</UL>
<DIR>

<P>Everyone agreed that preventing these actions was important, as we wanted a stable, uniform look to our project. "Fortunately," I told the class, "Visual Basic makes building these safeguards into the program easy. To prevent the user from re-sizing the form, we will set the </FONT><FONT FACE="Helvetica" SIZE=2>BorderStyle</FONT><FONT FACE="Palatino" SIZE=2> property of the form to </FONT><FONT FACE="Helvetica" SIZE=2>1 – Fixed Single</FONT><FONT FACE="Palatino" SIZE=2>. To prevent the user from moving the form, we will set the </FONT><FONT FACE="Helvetica" SIZE=2>Moveable</FONT><FONT FACE="Palatino" SIZE=2> property to </FONT><FONT FACE="Helvetica" SIZE=2>False</FONT><FONT FACE="Palatino" SIZE=2>. However, those Windows Close, Maximize and Minimize buttons are a little more interesting."</P>
<P>I continued by explaining that there are two form properties, </FONT><FONT FACE="Helvetica" SIZE=2>MaxButton</FONT><FONT FACE="Palatino" SIZE=2> and </FONT><FONT FACE="Helvetica" SIZE=2>MinButton</FONT><FONT FACE="Palatino" SIZE=2>, that can be set to </FONT><FONT FACE="Helvetica" SIZE=2>False</FONT><FONT FACE="Palatino" SIZE=2> to prevent the user from maximizing or minimizing the form. And if you try these properties </P>

<P>out, you will see that these options are also removed from the Control Box menu. However, we are still left with the Close button.</P>
<P>Another property, the </FONT><FONT FACE="Helvetica" SIZE=2>ControlBox</FONT><FONT FACE="Palatino" SIZE=2> property, can also be set to </FONT><FONT FACE="Helvetica" SIZE=2>False</FONT><FONT FACE="Palatino" SIZE=2>. This removes the Control Box icon from the form and causes the Maximize, Minimize, and Close buttons to disappear from the form as well. Therefore, if we change the </FONT><FONT FACE="Helvetica" SIZE=2>ControlBox</FONT><FONT FACE="Palatino" SIZE=2> property to </FONT><FONT FACE="Helvetica" SIZE=2>False</FONT><FONT FACE="Palatino" SIZE=2>, we can take care of all the requirements at the same time.</P>
</FONT><B><I><FONT FACE="FranklinGothic" SIZE=4><P><A NAME="_Toc412024752"><A NAME="_Toc413993249">The StartupPosition Property</A></A></P>
</B></I></FONT><FONT FACE="Palatino" SIZE=2><P>"Why didn't you include values for the Left and Top properties?" Blaine asked. "They weren't in the table."</P>
<P>"That's a good point," I said, "but we don't really need to worry about those."</P>
<P>I explained that there's no need to concern ourselves with the location of the form within the screen, because there is a Visual Basic property called </FONT><FONT FACE="Helvetica" SIZE=2>StartupPosition</FONT><B><FONT FACE="Palatino" SIZE=2>, </B>which will center the form right in the middle of the screen. To do this we simply select option </FONT><FONT FACE="Helvetica" SIZE=2>2 - CenterScreen</FONT><FONT FACE="Palatino" SIZE=2> from the list box.</P>
</FONT><B><I><FONT FACE="FranklinGothic" SIZE=4><P><A NAME="_Toc412024753"><A NAME="_Toc413993250">The Caption</A></A> Property</P>
</B></I></FONT><FONT FACE="Palatino" SIZE=2><P>"Finally," I said, "we'll change the form's caption to display the name of the China Shop by setting the </FONT><FONT FACE="Helvetica" SIZE=2>Caption</FONT><FONT FACE="Palatino" SIZE=2> property. This will be displayed in the form's title bar."</P>
<P>I asked if there were any questions and there were none, so<A NAME="stopheree"> I distributed this exercise for the class to complete to make the property changes to the form.</P>
