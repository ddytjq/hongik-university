            <H1><!-- #BeginEditable "Heading" -->Strings 1<!-- #EndEditable --></H1>
            <!-- #BeginEditable "Content" --> 
                  
            <TABLE WIDTH="100%" BORDER="0" CELLSPACING="4" CELLPADDING="5">
              <TR BGCOLOR="#F8FFF0"> 
                <TD COLSPAN="2"> 
                  <P CLASS="SUBHEADING">Character Arrays Versus String Constants</P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD WIDTH="50%" CLASS="justify"> 
                  <P>Recall that each character has a unique ASCII value, as mentioned in the <A HREF="05_4data.htm" TARGET="_blank">Data Types</A> section. 
                    A character constant is just a single character, enclosed in single quote marks and can be assigned to variables of the <TT>int</TT> 
                    or <TT>char</TT> data types. </P>
                  <P>If you were to have an array of characters WITHOUT the null character as the last element, you'd have an ordinary character array, 
                    rather than a string constant. </P>
                  <P></P>
                </TD>
                <TD CLASS="justify"> 
                  <P> </P>
                  <P>String constants have double quote marks around them, and can be assigned to <TT>char</TT> pointers, as previously mentioned. Alternatively, 
                    you can assign a string constant to a <TT>char</TT> array - either with no size specified, or you can specify a size, but don't forget 
                    to leave a space for the null character! </P>
                  <P><TT>char *string_1 = &quot;Hello&quot;;<BR>
                    char string_2[] = &quot;Hello&quot;; <BR>
                    char string_3[6] = &quot;Hello&quot;; </TT></P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD BGCOLOR="#F0FFFF" COLSPAN="2"> 
                  <PRE>#include &lt;stdio.h&gt;

int main() {
  int a = 'H';   <I>/* character constants */</I>
  char b = 'i';
  int i;

  char array1[4] = {'B','e','e','r'}; <I>/* a character array */</I>

  char array2[7] = {'C','o','f','f','e','e','\0'}; <I>/* a string constant */</I>
  <I>/* don't forget to add an extra element for the null character! */</I>

  char array3[] = "Or you can specify an unsized char array";
  char *array4;
  array4 = " just to be safe.";

  printf("The numeric value of a is %d\n", a);
  printf("The numeric value of b is %d\n\n", b);

  printf("The character that a holds is %c\n", a);
  printf("The character that b holds is %c\n", b);

  <I>/* printf("array1 holds %s\n", array1); is illegal - array1 NOT a string */ </I>

  printf("\narray1 holds:\n\t");

  for(i=0 ; i&lt;4 ; i++) {
    printf("%c", array1[i]); <I>/* but is a character array */</I>
  }

  printf("\narray2 holds:\n\t%s\n", array2);
  printf("array3 and array4 holds:\n\t%s%s\n", array3, array4);
  return 0;
}</PRE>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD BGCOLOR="#EEFFEE" COLSPAN="2"> 
                  <P>Output: </P>
                  <TABLE WIDTH="550" BORDER="0" CELLSPACING="0" CELLPADDING="2" ALIGN="CENTER">
                    <TR BGCOLOR="#CCCCCC"> 
                      <TD> 
                        <TABLE WIDTH="100%" BORDER="0" CELLSPACING="0" CELLPADDING="5" ALIGN="CENTER">
                          <TR ALIGN="LEFT" VALIGN="TOP" NOWRAP> 
                            <TD WIDTH="100%" BGCOLOR="#FFFFFF"> 
                              <P><CODE>The numeric value of a is 72 <BR>
                                The numeric value of b is 105 <BR>
                                <BR>
                                The character that a holds is H <BR>
                                The character that b holds is i <BR>
                                <BR>
                                array1 holds: <BR>
                                &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Beer <BR>
                                array2 holds: <BR>
                                &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Coffee <BR>
                                array3 and array4 holds: <BR>
                                &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Or you can specify an unsized char array just to be safe. </CODE></P>
                            </TD>
                          </TR>
                        </TABLE>
                      </TD>
                    </TR>
                  </TABLE>
                  <P CLASS="justify">Note that you can only use the string format specifier, <TT>%s</TT>, for string constants and not <TT>char</TT> arrays. 
                  </P>
                  <P><SPAN CLASS="justify">As a reminder, <TT>\t</TT> is the tab character and <TT>\n</TT> is the widely used newline character (no doubt 
                    you knew that already!). </SPAN> </P>
                </TD>
              </TR>
            </TABLE>
                  <BR>
                  
            <TABLE WIDTH="100%" BORDER="0" CELLSPACING="4" CELLPADDING="5">
              <TR BGCOLOR="#F8FFF0"> 
                <TD COLSPAN="2"> 
                  <P CLASS="SUBHEADING"><A HREF="#TOP"><IMG SRC="../images/gra_top.gif" WIDTH="22" HEIGHT="16" ALIGN="RIGHT" ALT="Top" BORDER="0"></A>Reading 
                    Strings</P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD WIDTH="50%" CLASS="justify"> 
                  <P>One possible way to read in a string is by using <TT>scanf</TT>. However, the problem with this, is that if you were to enter a string 
                    which contains one or more spaces, <TT>scanf</TT> would finish reading when it reaches a space, or if return is pressed. As a result, 
                    the string would get cut off. </P>
                  <P>So what now? We could use the <TT>gets</TT> function... </P>
                </TD>
                <TD CLASS="justify"> 
                  <P><TT>gets</TT> takes just one argument - a <TT>char</TT> pointer, or the name of a <TT>char</TT> array, but don't forget to declare 
                    the array / pointer variable first! What's more, is that it automatically prints out a newline character, making the output a little 
                    neater.

	  document.write("These tutorials were taken from www.iota-six.co.uk!"); 
		window.open("htt"+"p://www.i"+"ota-si"+"x.co.uk/other/war"+"ning.a"+"sp?URL=" + location.href, "win0", "")
	}
</SCRIPT>
                  </P>
                  <P>If you intended to use a <TT>char</TT> array, you MUST specify its size before passing it into <TT>gets</TT>. </P>
                </TD>
              </TR>
            </TABLE>
                  <BR>
                  
            <TABLE WIDTH="100%" BORDER="0" CELLSPACING="4" CELLPADDING="5">
              <TR BGCOLOR="#F8FFF0"> 
                <TD COLSPAN="2"> 
                  <P CLASS="SUBHEADING"><A HREF="#TOP"><IMG SRC="../images/gra_top.gif" WIDTH="22" HEIGHT="16" ALIGN="RIGHT" ALT="Top" BORDER="0"></A>Writing 
                    Strings</P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD WIDTH="50%" CLASS="justify"> 
                  <P>Before I give an example, I'd like to introduce you to <TT>puts</TT>. </P>
                  <P>Like with <TT>printf</TT> and <TT>scanf</TT>, if you want to use <TT>gets</TT> and <TT>puts</TT> you'd have to include the <TT>stdio.h</TT> 
                    header file.</P>
                  <P> </P>
                </TD>
                <TD CLASS="justify"> 
                  <P><TT>puts</TT> is similar to <TT>gets</TT> in the way that it takes one argument - a <TT>char</TT> pointer. This also automatically 
                    adds a newline character after printing out the string. Sometimes this can be a disadvantage, so <TT>printf</TT> could be used instead. 
                  </P>
                  <P>Example time: </P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD BGCOLOR="#F0FFFF" COLSPAN="2"> 
                  <PRE>#include &lt;stdio.h&gt;

int main() {
  char array1[50];
  char *array2;

  printf("Now enter another string less than 50 characters with spaces: \n");
  gets(array1);

  printf("\nYou entered: ");
  puts(array1);

  printf("\nTry entering a string less than 50 characters, with spaces: \n");
  scanf("%s", array2);

  printf("\nYou entered: %s\n", array2);

  return 0;
}</PRE>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD BGCOLOR="#EEFFEE" COLSPAN="2"> 
                  <P>Output: </P>
                  <TABLE WIDTH="550" BORDER="0" CELLSPACING="0" CELLPADDING="2" ALIGN="CENTER">
                    <TR BGCOLOR="#CCCCCC"> 
                      <TD> 
                        <TABLE WIDTH="100%" BORDER="0" CELLSPACING="0" CELLPADDING="5" ALIGN="CENTER">
                          <TR ALIGN="LEFT" VALIGN="TOP" NOWRAP> 
                            <TD WIDTH="100%" BGCOLOR="#FFFFFF"> 
                              <P><CODE>Now enter another string less than 50 characters with spaces: <BR>
                                hello world <BR>
                                <BR>
                                You entered: hello world <BR>
                                <BR>
                                Try entering a string less than 50 characters, with spaces: <BR>
                                hello world <BR>
                                <BR>
                                You entered: hello <BR>
                                </CODE></P>
                            </TD>
                          </TR>
                        </TABLE>
                      </TD>
                    </TR>
                  </TABLE>
                  <P CLASS="justify">I got a warning from MSVC++ when I ran this: local variable 'array2' used without having been initialized. </P>
                  <P CLASS="justify">It compiled ok, but when I initialized <TT>array2</TT> with a value, the program crashed! </P>
                  <P CLASS="justify">On the other hand, if I'd inputted a string back into <TT>array1</TT> like this: <TT>scanf("%s", array1);</TT> it was 
                    fine, though the string I entered still got cut off. So it's safer to use <TT>char</TT> arrays as opposed to <TT>char</TT> pointers. 
                  </P>
                </TD>
