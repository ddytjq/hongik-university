            <H1><!-- #BeginEditable "Heading" -->Looping<!-- #EndEditable --></H1>
            <!-- #BeginEditable "Content" --> 
                  
            <TABLE WIDTH="100%" BORDER="0" CELLSPACING="4" CELLPADDING="5">
              <TR BGCOLOR="#F8FFF0"> 
                <TD COLSPAN="2"> 
                  <P CLASS="SUBHEADING">The Concept of Looping</P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD WIDTH="50%" CLASS="justify"> 
                  <P>The term &quot;looping&quot; describes the way in which the program executes statements over and over again, before exiting the loop 
                    and continuing with program flow. </P>
                  <P>In C, there are three types of loops: <TT>for</TT> loops, <TT>while</TT> loops and <TT>do while</TT> loops. </P>
                </TD>
                <TD WIDTH="50%" BGCOLOR="#EEFFEE" CLASS="justify"> 
                  <P>It is possible to simulate each type by writing code using other loops. </P>
                  <P>Tell a lie - there is a fourth kind of loop but has no specific keyword. <SPAN CLASS="RED">INFINITE LOOPS</SPAN> can cause your computer 
                    to hang! They sometimes end after a while, but in theory they should go on forever. </P>
                </TD>
              </TR>
            </TABLE>
                  <BR>
                  
            <TABLE WIDTH="100%" BORDER="0" CELLSPACING="4" CELLPADDING="5">
              <TR BGCOLOR="#F8FFF0"> 
                <TD COLSPAN="2"> 
                  <P CLASS="SUBHEADING"><SPAN CLASS="SUBHEADINGCODE">for</SPAN> Loops</P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD WIDTH="50%" CLASS="justify"> 
                  <P>I use <TT>for</TT> loops a lot. Here's the basic form: </P>
                  <PRE>for (a ; b ; c) {
<I> /* statements */</I>
}
</PRE>
                  <P>Brief example:</P>
                  <PRE>for(i=10, j=0 ; i!=j ; i--, j++) {
  printf(&quot;%d - %d = %d\n&quot;, i, j, i-j);
}</PRE>
                  <P><TT>a</TT>, <TT>b</TT> and <TT>c</TT> are expressions that are evaluated at different times. </P>
                  <P><TT>a</TT> is evaluated once only - before entering the loop. It is a very good place to assign values to variables. If you wanted 
                    to assign values to multiple variables, you can separate each assignment with a comma like this: <TT>i=10, j=0</TT> </P>
                </TD>
                <TD WIDTH="50%" BGCOLOR="#EEFFEE" CLASS="justify"> 
                  <P><TT>b</TT> determines whether or not the program loops. If <TT>b</TT> returns a non zero value, an iteration of the loop is performed. 
                    In other words, the program enters the loop and executes the statements inside. <TT>b</TT> should therefore be some sort of condition. 
                    You can use logical operators in your condition as usual. </P>
                  <P><TT>c</TT> is evaluated after an iteration of the loop, i.e. after all the statements inside the loop body have been executed. If you 
                    have multiple expressions you want evaluating, you can separate each one with a comma like this: <TT>i--, j++ </TT></P>
                  <P>The round brackets and semi colons are required at all times. The three expressions aren't compulsory, but omitting <TT>b</TT> and 
                    failing to place a <TT>break</TT> statement somewhere in the loop can result in an infinite loop! </P>
                  <P>Let's look at a proper example:

	}
</SCRIPT>
                  </P>
                </TD>
              </TR>
            </TABLE>
                  <BR>

int main() {
 int i,j;

 }

</PRE>

                  <P CLASS="justify">The sixth evaluation of <TT>i!=j</TT> returns <TT>0</TT> for false, because <TT>i</TT> and <TT>j</TT> are equal at 
                    that time, so the loop body is not executed. </P>
                </TD>
              </TR>
              <TR> 
                <TD BGCOLOR="#F0FFF0" WIDTH="50%" VALIGN="TOP">

                  <P CLASS="justify">You should be familiar with the logic behind the <TT>printf</TT> statement, so I'll focus on the loop itself. </P>
                </TD>

            <TABLE WIDTH="100%" BORDER="0" CELLSPACING="4" CELLPADDING="5">

                  <P CLASS="SUBHEADING"><SPAN CLASS="SUBHEADINGCODE"><A HREF="#TOP"><IMG SRC="../images/gra_top.gif" WIDTH="22" HEIGHT="16" ALIGN="RIGHT" ALT="Top" BORDER="0"></A>while</SPAN> 
                    Loops</P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD WIDTH="50%" CLASS="justify"> 
                  <P>Some might say that <TT>while </TT>loops are simpler to use, since they only require one expression in the brackets: </P>
                  <PRE>while (expression) {
<I>  /* statements */
</I>}</PRE>
                  <P>The loop body is executed if <TT>expression</TT> returns a non zero value. Most of the time, you'd want to put some sort of condition 
                    to be evaluated. </P>
                  <P>I mentioned earlier on that you can simulate one type of loop with another type - there is no unique way to loop. </P>
                  <P>The previous example can be rewritten using a <TT>while </TT>loop. </P>
                </TD>
                <TD WIDTH="50%" BGCOLOR="#F0FFFF"> 
                  <PRE>#include &lt;stdio.h&gt;

int main() {
 int i=10; <I>/* initialize variables */</I>
 int j=0;  <I>/* part a of a for loop */</I>

 while (i!=j) { <I>/* test for condition 
                   part b of for loop */</I>
   printf(&quot;%d - %d = %d\n&quot;, i, j, i-j);
   i--;  <I>/* do something to variables */</I>
   j++;  <I>/* part c of for loop        */</I>
 }
 return 0;
}</PRE>
                </TD>
              </TR>
            </TABLE>
                  <BR>
                  
            <TABLE WIDTH="100%" BORDER="0" CELLSPACING="4" CELLPADDING="5">
              <TR BGCOLOR="#F8FFF0"> 
                <TD COLSPAN="2"> 
                  <P CLASS="SUBHEADING"><SPAN CLASS="SUBHEADINGCODE"><A HREF="#TOP"><IMG SRC="../images/gra_top.gif" WIDTH="22" HEIGHT="16" ALIGN="RIGHT" ALT="Top" BORDER="0"></A></SPAN>Infinite 
                    Loops</P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD WIDTH="50%" CLASS="justify"> 
                  <P>Having an expression that always returns a non zero value in the brackets of a <TT>while</TT> loop (or the second expression of a <TT>for</TT> 
                    loop) can result in an infinite loop. Missing the second expression in a <TT>for</TT> loop will also result in an infinite loop, but 
                    missing the expression in the brackets of a <TT>while</TT> loop causes a compilation error. This program should loop forever: </P>
                </TD>
                <TD WIDTH="50%" BGCOLOR="#F0FFF0" CLASS="justify"> 
                  <P>I say &quot;should&quot; loop forever, but sometimes the computer crashes and the program is shut down. </P>
                  <P><TT>while (1) {} </TT>also causes an infinite loop. </P>
                  <P>To combat infinite loops, you can use the <TT>break</TT> statement (or in the last resort, Ctrl-Alt-Delete!). </P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD WIDTH="50%" BGCOLOR="#F0FFFF"> 
                  <PRE>#include &lt;stdio.h&gt;

int main() {
 int i;
 for(i=0 ; <I>/* no expression */</I> ; i++) {
  printf(&quot;%d\n&quot;, i);
 }
 return 0;
}</PRE>
                </TD>
                <TD WIDTH="50%" BGCOLOR="#F0FFFF"> 
                  <PRE>#include &lt;stdio.h&gt;

int main() {
 int i;
 for(i=0 ; <I>/* no expression */</I> ; i++) {
  printf(&quot;%d\n&quot;, i);
  if(i==1000) {
   break; <I>/* exits the loop when */
</I>  }       <I>/* i equals 1000       */</I>
 }
 return 0;
}</PRE>
                </TD>

                  <P CLASS="SUBHEADING"><SPAN CLASS="SUBHEADINGCODE"><A HREF="#TOP"><IMG SRC="../images/gra_top.gif" WIDTH="22" HEIGHT="16" ALIGN="RIGHT" ALT="Top" BORDER="0"></A>do 
                    while</SPAN> Loops</P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD WIDTH="50%" CLASS="justify"> 
                  <P>The last type of loop I rarely use. </P>
                  <P>It's almost identical to a while loop, except the loop body is executed at least once - the <TT>while (expression)</TT> part is after 
                    the loop body, like this: </P>
                  <PRE>do {
  <I>/* loop body */</I>
} while (expression);</PRE>
                  <P>This example prints out the numbers 0-9: </P>
                </TD>
                <TD WIDTH="50%" BGCOLOR="#F0FFF0" CLASS="justify"> 
                  <P>Don't forget the semi colon after the<BR>
                    <TT>while(expression)</TT> part! </P>
                  <P>If you run the example again, but initialize <TT>i</TT> with <TT>11</TT> you'll notice that the loop body is still executed. </P>
                  <P>Simulating a do while loop with either a <TT>for</TT> or <TT>while</TT> loop is fiddley - most of the time I'd write the statements 
                    in the loop body once, before the loop: </P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD WIDTH="50%" BGCOLOR="#F0FFFF"> 
                  <PRE>#include &lt;stdio.h&gt;

int main() {
  int i = 0;
  do {
    printf(&quot;%d&quot;, i);
    i++;
  } while (i&lt;10);
  return 0;
}</PRE>
                </TD>
                <TD WIDTH="50%" BGCOLOR="#F0FFFF"> 
                  <PRE>#include &lt;stdio.h&gt;

int main() {
  int i = 0;

  printf(&quot;%d&quot;, i); <I>/* &quot;loop body&quot; */</I>
  i++;

  while (i&lt;10) {
    printf(&quot;%d&quot;, i);
    i++;
  }
  return 0;
}</PRE>
