THIS WEEKS FEATURE ARTICLE: The Frustrations of a Newbie
------------------------------------------------------------

	Don't you hate it how a lot of the time when you 
compile, you get tons of errors? Don't you hate it when you
take time and energy to program something then it doesn't
work? I personally think it is  really annoying. In this 
article I will let you know some of the biggest problems I 
have encountered in the course of learning Java.

	One of the most annoying things I can think of is 
not having event handlers work. I admit that this has 
happened to me quite a few times, but all because of the same
problem. This problem was that I didn't add the event listener.

	For example, I might have an applet that implements
MouseListener. Unless you add the following line, your objects
won't be able to tell when a MouseEvent occurs:

objectname.addMouseListener(this);

	Almost all GUI Components have the addMouseListener()
method because it is part of the Component class. As easy as
it seems, I have seen many people who are new to Java forget
this. (As I have said, I have done it myself!) Any time your
events aren't being handled when you think they should be, 
make sure to check that you added a Listener to your object!

	Another big thing that myself and others have 
encountered while learning Java is getting so many errors at
one time that you go insane. The main reason that this 
happens is because beginners try to do too much at one. I
remember trying to program entire programs at once, and it
just doesn't work unless you are extremely good. 

	What can help is splitting up the program into 
sections, and compiling after you finish each section. This
is what I did when I wrote the game Space Shooters. Getting
the variables working, getting the thread working, getting
the double buffering working, changing the variables, and 
all of the other parts were each done seperately. I didn't 
even begin to try to write it all at once. Don't try to do
everything at once........

	For people completely new to programming, writing
semicolons at the right time is also a huge problem. Semicolons
end Java statements, which usually edit a variable or call
a method. For example, the following are all statements:

objectname.addMouseListener(this);
repaint();
x=1;
x+=5;
	
	Hopefully you can see the pattern among these lines of
code and realize that they are statements. If you don't right 
now, it shouldn't be too long before you understand them, so
don't get discouraged!

	Thats it for this issue, its getting to be almost
midnight, so I better stop writing! Thanks for listening, and
I hope this helped!
