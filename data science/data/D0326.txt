Our own experiences developing J2EE applications have indicated that the use of session beans instead of entity beans can overcome such performance difficulties. While we've found that other experienced J2EE developers share this view, the practice of using session beans for database access remains undocumented and has not been advanced as a best practice. Hoping that some quantitative figures will help convince developers of the performance improvement that session beans offer, we developed this EJB Performance Benchmark. <BR><BR>The benchmark data demonstrates that there is between a 3 and 90 fold performance improvement available just by switching from entity beans to stateless session beans. <BR><BR>The benchmark is an interactive web based application that you can use to gather your own data.<BR><BR>Recent Updates
