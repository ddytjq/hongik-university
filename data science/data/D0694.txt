                            <h1 align="CENTER"> C#: A language alternative or just J--?, Part 2</h1>
                            <h3 align="CENTER"> The semantic differences and design choices between C# and Java</h3>
                        </p>
                        <!--<BLOCKQUOTE><STRONG>Summary</STRONG><BR>-->
                        <BLOCKQUOTE>
<STRONG>Summary</STRONG><BR>


Despite their enormous similarities, Java and C# differ greatly in many language details and also in their basic technical intent. This second article of a two-part series covers C# language constructs and concludes with some speculation on the idea of standardizing C#. (<em>4,200 words</em>)

<img align="left" width="12" height="29" src="/javaworld/abcs/I.gif" alt="I">n <a href="http://www.javaworld.com/javaworld/jw-11-2000/jw-1122-csharp1.html">Part 1</a> of this series, I covered the similarities between C# and Java, explained C#'s role in .Net, speculated on the possible outcomes of using multiple languages in application development, and looked at some high-level differences between Java and C#. This article covers C# language constructs in depth, points out various language features, and provides short pieces of sample code that illustrate C# programming principles. It concludes with a rant on C# and the tension between proprietary interests on one hand and open standardization on the other. <P><FONT SIZE="+1"><STRONG> C# features </STRONG></FONT><BR>
Some of the differences between C# and Java are simply cosmetic. In particular, there's the question of capitalization. I don't know for sure, but I'm guessing that the capitalization conventions in C# come from Delphi, as I suspect the <code>WriteLine</code> method also does. The capitalization in C#, with its <code>Main()</code> method and <code>string</code> built-in type, feel odd to the Java programmer. Are these conventions simply lifted from Delphi, or are they used rather to make C# look, on the surface, less like Java? If the reason was the latter, I'm afraid it wasn't very successful. The effect is rather like putting lipstick on your dog. He's still your dog, only with lipstick, and he doesn't necessarily look any better or worse for it. <P>
Fortunately, C# has several features that go beyond the cosmetic. Some of these, such as enumerated values ("enums"), are simple "syntactic sugar," providing self-documentation and arguably clearer source code. Other features, such as delegates and events, are quite useful (though implemented in a confusing way) and provide functionality built into the language that, in Java, requires coding and a nontrivial understanding of Java design patterns. <P>
This section will cover several interesting C# features. <P><STRONG> Type system </STRONG><BR>
Several features of the C# type system are interesting. Primitive values can always be treated as objects. There are a few class member types (properties, structs, enumerations, and delegates) that do not exist in Java. C# arrays work differently from Java arrays. Operators are overloadable in a fashion similar, but not identical, to that of C++. You can access indexed collections through a mechanism called an <em>indexer,</em> which works similarly to an operator. <P><STRONG> Primitive/object unification </STRONG><BR>
One feature of the C# type system that is bound to be popular is the use of all primitive types as objects. In fact, you can use primitive literal values, such as strings and integers, as if they were objects, without first constructing a <code>String</code> or <code>Integer</code> object in code, as is necessary in Java. So, for example, the Java code: <P>
<CODE><P>
<BR>

<BR>

Integer iobj = new Integer(12);<BR>

<BR>

System.out.println(iobj.toString());<BR>

<BR>
</CODE><P>
could be expressed in C# more clearly as: <P>
<CODE><P>
<BR>

<BR>

Console.WriteLine(12.ToString());<BR>

<BR>
</CODE><P>
You'll notice in this example that the literal <code>12</code> can be used as an object. Since every object in C# is a subclass of <code>class object</code> and every literal value is an object in its own right, it is always possible to call any method of <code>class object</code> against a primitive variable or primitive literal value. Many Java programmers feel that the distinction between primitive value objects and primitive values is an unfortunate one in Java, and the designers of C# apparently agree. The processes of automatically converting a primitive value into an object and vice versa in C# are called <em>boxing</em> and <em>unboxing,</em> respectively. <P><STRONG> Array types </STRONG><BR>

multiArray2[1] =&nbsp;&nbsp;{ 1, 2, 1 };<BR>

multiArray2[2] = { 1, 3, 3, 1 };<P>
</CODE><P>
</CODE></TD></TR> <caption align="bottom"><b><font size="-1">Table 1. Java and C# array definition and initialization</font></b></caption></TABLE> <P>
Table 1 shows the differences in initialization syntax for Java and C# arrays. Oddly, Java's initialization syntax is more compact than C#'s, but C# has rectangular arrays that Java lacks. The following example shows the definition of two rectangular arrays in C#: <P>
<CODE><P>
<BR>

<BR>

int[,] r2dArray&nbsp;&nbsp;= new int[,] { {1, 2, 3}, <BR>

<BR>

{2, 3, 4} };<BR>

<BR>

int[,,] r3dArray = { { { 1, 2 },<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; { 3, 4 } },<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; { { 5, 6 },<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; { 7, 8 } } };<BR>

<BR>
</CODE><P>
You'll notice that the second preceding example, <code>r3dArray</code>, uses an acceptable shorthand (omitting <code>new int [,,]</code>) to initialize the array. The C# specification states that the element type in the array and the number of dimensions define the array type, but the size of each dimension does not. For example, <code>int[4][3][5]</code>and <code>int [3][2][1]</code> are the same type, because they have the same base type and dimensions. <P>
Arrays are reference types, meaning they do not copy on assignment. All arrays inherit from the <code>System.Array</code> base class, and therefore inherit all of that class's properties, including the <code>Length</code> property. Since C# arrays are collection types, you can use them with the C# <code>foreach</code> keyword. <P><STRONG> Indexers </STRONG><BR>
A class's indexer lets you access any instance of that class as if it were an array. A class may define multiple indexers, each of which differs by the number and type of its arguments. Indexers are very similar to properties, especially in the syntax for defining them, as shown in the following code example: <P>
<CODE><P>
<BR>

<BR>

public class Sparse2DArray {<BR>

<BR>

&nbsp;&nbsp; private double[][] _values;<BR>

<BR>

&nbsp;&nbsp; private int _i, _j;<BR>

<BR>

&nbsp;&nbsp; public Sparse2DArray(int i, int j) {<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;_i = i;<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;_j = j;<BR>

<BR>

&nbsp;&nbsp; }<BR>

<BR>

&nbsp;&nbsp; public double this[int i, int j] {<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;set {<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <FONT COLOR="RED">// Check bounds</FONT><BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; if (i &lt; 0 || i &gt;= _i || j &lt; 0 || j &gt;= _j) {<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;throw new IndexOutOfRangeException();<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; }<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <FONT COLOR="RED">// Create values matrix if it doesn't<BR>

<BR>

exist</FONT><BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; if (_values == null) {<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;_values = new double[_i][];<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; }<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <FONT COLOR="RED">// Create j'th array if it doesn't<BR>

<BR>

exist</FONT><BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; if (_values[i] == null) {<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;_values[i] = new double[_j];<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; }<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; _values[i][j] = value;<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;}<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;get {<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;<FONT COLOR="RED">// Check bounds</FONT><BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; if (i &lt; 0 || i &gt;= _i || j &lt; 0 || j &gt;= _j) {<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;throw new IndexOutOfRangeException();<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; }<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <FONT COLOR="RED">// Sparse matrix is zero where no<BR>

<BR>

values exist</FONT><BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; if (_values[i] == null || values[i][j] == <BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; null) {<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;return 0.0;<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; } else {<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;return _values[i][j];<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; }<BR>

<BR>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;}<BR>

<BR>

&nbsp;&nbsp; }<BR>

<BR>

};<BR>

<BR>
</CODE><P>
The indexer is defined with a syntax similar to the definition of a method, with the text: <code>public double this[int i, int j]</code>. The <code>set</code> block for the indexer (everything inside the block within the <code>set { ... }</code>) creates the private <code>_values</code> array, or any subarray of that array, as needed to store the value passed to the indexer. The <code>get</code> block (inside <code>get { ... }</code> is the value present at the given <code>[i, j]</code> position, or zero if the value does not exist. You can index instances of <code>Sparse2DArray</code> with the square bracket notation used for arrays, like so: <P>
<CODE><P>
<BR>

<BR>

Sparse2DArray s2d = new Sparse2DArray(1000, 10);<BR>

<BR>

s2d[512, 6] = 3.14159265;<BR>

<BR>

Console.WriteLine(s2d[12,5]);&nbsp;&nbsp;<FONT COLOR="RED">// will print<BR>

<BR>

zero</FONT><BR>

<BR>
</CODE><P>
