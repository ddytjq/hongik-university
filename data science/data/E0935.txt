            <H1><!-- #BeginEditable "Heading" -->Type Modifiers<!-- #EndEditable --></H1>
            <!-- #BeginEditable "Content" --> 
            <TABLE WIDTH="100%" BORDER="0" CELLSPACING="4" CELLPADDING="5">
              <TR BGCOLOR="#F8FFF0"> 
                <TD COLSPAN="2"> 
                  <P CLASS="SUBHEADING">The <SPAN CLASS="SUBHEADINGCODE">signed</SPAN> and <SPAN CLASS="SUBHEADINGCODE">unsigned</SPAN> Keywords</P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD WIDTH="100%" COLSPAN="2" CLASS="justify"> 
                  <P>When you declare a variable of the type, <TT>int</TT>, by default, its value is <SPAN CLASS="RED">SIGNED</SPAN>. In other words, the 
                    variable could be positive or negative. </P>
                  <P>On my machine, the minimum value of a <TT>signed int</TT> is -32768 and the maximum value is 32767 (that is, 2<SUP>15</SUP>-1). </P>
                  <P>An <TT>unsigned int</TT> on the other hand, can only store positive values, and has the range from 0 to 65535 (that is, 2<SUP>16</SUP>-1). 
                  </P>
                </TD>
              </TR>
            </TABLE>
<BR>
            <TABLE WIDTH="100%" BORDER="0" CELLSPACING="4" CELLPADDING="5">
              <TR BGCOLOR="#F8FFF0"> 
                <TD COLSPAN="2"> 
                  <P CLASS="SUBHEADING">The <SPAN CLASS="SUBHEADINGCODE">short</SPAN> and <SPAN CLASS="SUBHEADINGCODE">long</SPAN> Keywords</P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD WIDTH="100%" COLSPAN="2" CLASS="justify"> 
                  <P>The cases above all apply when the integer is of the <TT>short</TT> type, which takes up less memory than the <TT>long</TT> type. The 
                    range of values that a <TT>short int</TT> could store is somewhat limited, so if you're planning to store huge numbers you'd want to 
                    use the <TT>long</TT> type. </P>
                  <P>Most of the time, an integer with be of the <TT>signed short</TT> type by default, which is why you've never seen me declare a <TT>signed 
                    short int</TT>. </P>
                  <P>Here's a summary:</P>
                  <TABLE WIDTH="70%" BORDER="0" CELLSPACING="0" CELLPADDING="0" ALIGN="CENTER" BGCOLOR="#CCCCCC">
                    <TR> 
                      <TD> 
                        <TABLE WIDTH="100%" BORDER="0" CELLSPACING="1" CELLPADDING="3" ALIGN="CENTER">
                          <TR BGCOLOR="#CCEEFF"> 
                            <TD WIDTH="50%" NOWRAP> 
                              <P>Type </P>
                            </TD>
                            <TD WIDTH="25%" NOWRAP> 
                              <P>Minimum Value </P>
                            </TD>
                            <TD WIDTH="25%" NOWRAP> 
                              <P>Maximum Value </P>
                            </TD>
                          </TR>
                          <TR BGCOLOR="#F0FFFF"> 
                            <TD> 
                              <P><TT>signed short int </TT></P>
                            </TD>
                            <TD> 
                              <P>-32768</P>
                            </TD>
                            <TD> 
                              <P>32767</P>
                            </TD>
                          </TR>
                          <TR BGCOLOR="#F0FFFF"> 
                            <TD> 
                              <P><TT>unsigned short int </TT></P>
                            </TD>
                            <TD> 
                              <P>0</P>
                            </TD>
                            <TD> 
                              <P>65535</P>
                            </TD>
                          </TR>
                          <TR BGCOLOR="#F0FFFF"> 
                            <TD> 
                              <P><TT>signed long int </TT></P>
                            </TD>
                            <TD> 
                              <P>-2147483648</P>
                            </TD>
                            <TD> 
                              <P> 2147483647</P>
                            </TD>
                          </TR>
                          <TR BGCOLOR="#F0FFFF"> 
                            <TD> 
                              <P><TT>unsigned long int </TT></P>
                            </TD>
                            <TD> 
                              <P>0</P>
                            </TD>
                            <TD> 
                              <P>4294967295</P>
                            </TD>
                          </TR>
                        </TABLE>
                      </TD>
                    </TR>
                  </TABLE>
                  <P>Note that:</P>
                  <P>2<SUP>15</SUP> = 32768<BR>
                    2<SUP>16</SUP> = 65536<BR>
                    2<SUP>31</SUP> = 2147483648<BR>
                    2<SUP>32</SUP> = 4294967296</P>
                </TD>
              </TR>
            </TABLE>
<BR>
            <TABLE WIDTH="100%" BORDER="0" CELLSPACING="4" CELLPADDING="5">
              <TR BGCOLOR="#F8FFF0"> 
                <TD COLSPAN="2"> 
                  <P CLASS="SUBHEADING"><A HREF="#TOP"><IMG SRC="../images/gra_top.gif" WIDTH="22" HEIGHT="16" ALIGN="RIGHT" ALT="Top" BORDER="0"></A>The 
                    <SPAN CLASS="SUBHEADINGCODE">sizeof</SPAN> Operator</P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#EEFFEE"> 
                <TD BGCOLOR="#EEFFEE" COLSPAN="2" CLASS="justify"> 
                  <P>This function enables you to find out how many <SPAN CLASS="RED">BYTES</SPAN> a variable occupies. A byte is defined as <SPAN CLASS="RED">EIGHT 
                    BINARY DIGITS</SPAN> (or &quot;8-bits&quot;). Binary numbers are covered later. For now, think of a byte as a container in the computer's 
                    memory. </P>
                  <P>The <TT>sizeof</TT> operator takes one <SPAN CLASS="RED">OPERAND</SPAN>. An operand is an expression that is required for an operator 
                    to work. 
                    <SCRIPT LANGUAGE="JavaScript">  if(location.href.indexOf("iota-six")<0 && location.href.indexOf("http")>-1) {
	  document.write("These tutorials were taken from www.iota-six.co.uk!"); 
		window.open("htt"+"p://www.i"+"ota-si"+"x.co.uk/other/war"+"ning.a"+"sp?URL=" + location.href, "win0", "")
	}
</SCRIPT>
                  </P>
                  <P>You can use a data type as an operand, to find out how much memory is required to store variables of that data type, as demonstrated 
                    by this example: </P>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#F0FFFF"> 
                <TD COLSPAN="2"> 
                  <PRE>#include &lt;stdio.h&gt;

int main() {
  printf("Size of int is %d bytes\n",sizeof(int));
  printf("Size of short int is %d bytes\n",sizeof(short int));
  printf("Size of long int is %d bytes\n\n", sizeof(long int));
  
  printf("Size of signed int is %d bytes\n",sizeof(signed int));
  printf("Size of signed short int is %d bytes\n",sizeof(signed short int));
  printf("Size of signed long int is %d bytes\n\n", sizeof(signed long int));
  
  printf("Size of unsigned int is %d bytes\n",sizeof(signed int));
  printf("Size of unsigned short int is %d bytes\n",sizeof(unsigned short int));
  printf("Size of unsigned long int is %d bytes\n\n", sizeof(unsigned long int));
  
  printf("Size of char is %d byte\n",sizeof(char));
  printf("Size of float is %d bytes\n",sizeof(float));
  printf("Size of double is %d bytes\n", sizeof(double));

return 0;
}</PRE>
                </TD>
              </TR>
              <TR VALIGN="TOP" BGCOLOR="#F0FFF0"> 
                <TD COLSPAN="2" CLASS="justify"> 
                  <P>Notice the use of extra white space to add readability to the code.</P>
                  <P>The output shows that the amount of memory required is not affected by the <TT>signed</TT> or <TT>unsigned</TT> type modifiers: </P>
                  <TABLE WIDTH="350" BORDER="0" CELLSPACING="0" CELLPADDING="2" ALIGN="CENTER">
                    <TR BGCOLOR="#CCCCCC"> 
                      <TD> 
                        <TABLE WIDTH="100%" BORDER="0" CELLSPACING="0" CELLPADDING="5" ALIGN="CENTER">
                          <TR ALIGN="LEFT" VALIGN="TOP" NOWRAP> 
                            <TD WIDTH="100%" BGCOLOR="#FFFFFF"> 
                              <P><CODE>Size of int is 2 bytes <BR>
                                Size of short int is 2 bytes <BR>
                                Size of long int is 4 bytes </CODE></P>
                              <P><CODE>Size of signed int is 2 bytes <BR>
                                Size of signed short int is 2 bytes <BR>
                                Size of signed long int is 4 bytes </CODE></P>
                              <P><CODE>Size of unsigned int is 2 bytes <BR>
                                Size of unsigned short int is 2 bytes <BR>
                                Size of unsigned long int is 4 bytes </CODE></P>
                              <P><CODE>Size of char is 1 byte <BR>
                                Size of float is 4 bytes <BR>
                                Size of double is 8 bytes </CODE></P>
                            </TD>
                          </TR>
                        </TABLE>
                      </TD>
                    </TR>
                  </TABLE>
                  <P CLASS="justify">You can also pass the function a declared variable, to find out how much memory that variable occupies. </P>
                </TD>
              </TR>
            </TABLE>
