<p ALIGN="CENTER">Introduction to BASIC</b></u></font><font COLOR="#0000ff" SIZE="2"></p>

<p>Before one can start programming in <b>Visual Basic</b> it is important to become
familiar with basics of BASIC</p>

<p>&quot;<b>B</b>eginners <b>A</b>ccess <b>S</b>ymbolic <b>I</b>nstruction <b>C</b>ode&quot;
known as BASIC was developed at Dartmouth College by John Kemmey and Thomas Kurtz in the
mid 1960&#146;s</p>

<p>Basic is a computer language whose instructions resemble algebraic formulas, augmented
by English words such as GO TO , INPUT , PRINT , IF , THEN ELSE</p>

<p>Because of its similarity to elementary algebra Basic is particularly suited to solving
problems in Science , Mathematics ,Engineering and Finance.</p>

<p>Basic programming can be a lot of fun </p>

<p>Structure of a BASIC Program 

<ul>
  <li>Every statement appears on a single line</li>
  <li>For longer statements (multi-line) an underscore _ is placed at the end of every line
    except the last line</li>
  <li>Blank spaces , tabs can be inserted to improve the readability of the statement</li>
</ul>

<p>An example of a simple Basic program to get the radius and calculate the diameter of
the circle</p>

<p>&nbsp;</p>

<p>R = InputBox</p>

<p>A = 3.14159 * R</p>

<p>MsgBox A</p>

<p>End<u><b></p>
</font><font COLOR="#ff0000">

(Prompt) and has a text field for the user to type some input . This will also be waiting

<p ALIGN="CENTER">3.1Variable Declaration</u></font><font COLOR="#0000ff" SIZE="2"></p>

<p>Implicit Declaration</b></p>

<p>By default you don&#146;t need to declare variables before you use them</p>

<p>However by default they become of the Variant Type</p>

<p>If you don&#146;t want that then you need to declare them using the Dim statement</p>

<p>Dim MyVar as Integer</p>

<p>Dim MyVar as string</p>

<p>Dim Name as string</p>

<p><b>&nbsp;</p>

<p>Explicit Declaration</b></p>

<p>If you use the following statement you are forced to declare all variables before using
them</p>

<p>Option Explicit</font><font SIZE="2" COLOR="#ff0000"><u><b></p>

<p ALIGN="CENTER">3.2 Naming Variables</b></u></font><font COLOR="#0000ff" SIZE="2"></p>

<p>When declaring variables observe the following rules 

<ol>
  <li>You must Start with a letter</li>
  <li>You can mix letters and numbers</li>
  <li>Use variable names that are meaningful (Someone else will one day be debugging your
    code)</li>
</ol>

<p>Different programmers have different styles in declaring Variable names</p>

<p>Dim Y2K as integer</p>

<p>Dim My_Var as integer</p>

<p>Dim MyVar as integer</p>

<p>Dim nMyVar as integer &#145;Hungarian Naming Convention</p>

<p>Today most of the Guru&#146;s are switching to using Hungarian Variable names</p>

<p>Refer to the Hungarian Naming Convention Chart</font><font COLOR="#ff0000"><u><b></p>

<p ALIGN="CENTER">4.0 Array Variables</b></u></font><font COLOR="#0000ff" SIZE="2"></p>

<p>Arrays are variables that can simultaneously hold more that one value to text string.</p>
<u><b>

<p>Declaring Arrays</b></u></p>

<p>Dim MemberNames(24) as string &#145; This will alloy you to save 25 names or strings in
this array</p>

<p>Dim Results(33) as integer &#145; This will alloy you to store 34 numbers in this array</p>

<p><u><b>&nbsp;</p>

<p>Array Index&#146;s</b></u></p>

<p>If you wish to store the 15<sup>th</sup> name in the above array you need to specify
the specific array element to save the data this is how its done</p>

<p>MembersNames(14)=&quot;Nick Michael&quot;</p>

<p>And if you wish to recall the 19<sup>th</sup> name this is how you access the specific
array element</p>

<p>MsgBox MembersNames(18)</p>

<p>Array Indexes always start with 0 unless 

<ul>
  <li>Option Base=1</li>
  <li>The Array is declared like this Dim Myarray (1 to 20) as integer</li>
</ul>

<p><b><u>&nbsp;</p>

<p>The Upper And Lower Bounds of the Array</u></p>

<p>Ubound</b> returns the highest valid index value</p>
<b>

<p>Lbound</b> Returns the lowest valid index value (Option Base can change this)</font><u><b><font COLOR="#ff0000"></p>

<p ALIGN="CENTER"></font><font SIZE="2" COLOR="#ff0000">&nbsp;</p>

<p ALIGN="CENTER">4.1 Dynamic Arrays</font></b></u><font COLOR="#0000ff" SIZE="2"></p>

<p>Sometimes you don&#146;t know how large to make the array </p>

<p>Or you want to free the memory when you don&#146;t need it</p>

<p>Then you would declare your arrays as Dynamic Arrays</p>

<p><u><b>&nbsp;</p>

<p>Declaring Dynamic Arrays</b></u></p>

<p>You declare the array with no value between the brackets </p>

<p>Dim MyArry( ) as Integer</p>

<p><b><u>&nbsp;</p>

<p>Setting the Size or Resizing of the Dynamic Array in real time</u></p>

<p>ReDim</b> MyArry(20) </p>
<b>

<p>ReDim</b> MyArray(30) </p>
<b>

<p>ReDim</b> MyArray(0)</p>
<b>

<p>ReDim</b> MyArray(100 + <b>Ubound</b>( MyArry( ) ) add another 100 elements to the
array</p>

<p>The problem with doing that is you loose all the data in the array if you need that use
the <b>Preserve</b> option</p>
<b>

<p>ReDim</b> <b>Preserve</b> MyArray(45) this will keel all elements whose index is
smaller than 45</p>

<p><b><u>&nbsp;</p>

<p>Freeing up the Memory used by your array</u></p>

<p>Erase</b> MyArry</p>

<p ALIGN="CENTER"></b></u>&nbsp;</p>

<p>The problem with Basic and even VB is that&nbsp; Angles are passed to Trig functions in
Radians</p>
<u><b>

<p>&quot;All trigonometric functions are calculated from angles expressed in Radians&quot;</b></u></p>

<p>In order to work with angles in degrees you need to define Pi and or Rad</p>

<p>Const Pi = 3.14159265359</p>

<p>or</p>

<p>Pi = 4 * Atn(1) &#145; This is an approximation for the value of Pi (not so accurate)</p>

<p>180 degrees=Pi Radians</p>

<p>1 Degree=Pi/180 Radians</p>

<p>1 Degree=3.14/180=0.017Radians </p>

<p>1 Degree=1.7E2 Scientific format (Single)&nbsp;</p>

<p>Const Rad = 1.74532925199433E-02 (double) This is the most accurate representation to
use for VB</p>

<p>If you multiply the angle in degrees by Rad you will convert it to radians and can use
the trigonometric</p>

<p>Functions e.g. Sin of x(degrees)=Sin(x * Rad) <b>but don&#146;t forget to define Rad</b></font></p>

<p ALIGN="CENTER">7.1 Subs (Subroutines)</b></u></font><font SIZE="2" COLOR="#0000ff"></p>

<p>Subs are called and no values are returned</p>

<p>However they can take values passed to them in the form of Parameters</p>
<u><b>

<p>Declaring a Sub</b></u></p>

<p>Sub MySub</p>

<p>End Sub</p>
<u><b>

<p>Declaring a Sub with Parameters</b></u></p>

<p>Sub MySub(Param1 as integer, Param2 as string)</p>

<p>End Sub</p>
<u><b>

<p>Calling a Sub</b></u></p>

<p>MySub or</p>

<p>Call MySub( ) or</p>

<p>MySub 123,&quot;John Doe&quot; or</p>

<p>Call MySub(123,&quot;John Doe&quot;) or</p>

<p>If you use the Call command you must add the Parenthesis ( ) to your subroutine</p>
<u><b>

<p>Optional Parameters</b></u></p>

<p>Sometimes you may want to use default parameter values but allow for overriding these
values</p>

<p>Sub MyFunct(optional Param1 as integer, optional Param2 as string)</p>

<p>End Sub</p>

<p ALIGN="CENTER">7.2 Functions</b></u></font><font SIZE="2" COLOR="#0000ff"></p>

<p>Functions are procedures that return a value or a string </p>

<p>They can also take values passed to them in the form of Parameters</p>
<u><b>

<p>Declaring a Function</b></u></p>

<p>Function MyFunct as integer</p>

<p>End Function</p>
<u><b>

<p>Declaring a Function with Parameters</b></u></p>

<p>Function MyFunct(Param1 as integer, Param2 as string) as integer</p>

<p>End Function</p>
<u><b>

<p>Calling a Function</b></u></p>

<p>MyVar=MyFunct or</p>

<p>MyVar=MyFunct 123,&quot;John Doe&quot; or</p>

<p>MyVar=MyFunct(123,&quot;John Doe&quot;)</p>
<u><b>

<p>Declaring Variables within a Function or Sub Local Variables</b></u></p>

<p>This is done the same way as above </p>

<p>However: 

<ol>
  <li>This variable is only accessible within the Procedure (<b>Procedural Scope</b>)</li>
  <li>This variable is discarded/destroyed as soon as the Procedure ends (<b>Variable Life</b>)</li>
  <li>If the variable is declared as a <b>static variable </b>it will not be destroyed after
    the Procedure ends</font></li>
</ol>
