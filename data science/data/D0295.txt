<SPAN Class="smbl">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;In some recent applet development
that I have been doing, I found how useful methods can be. I always knew about them, but never
used them much. A method is basically a routine. To better explain what a method is, I will give you an example.
Suppose I was creating a applet in which you can play billiards. Each time you need to begin a new game, there
would be numerous lines of code you would have to use in order to start a new game. Instead of retyping this code
over and over again, you can assign a name to these lines of code, which would allow you to just type the name instead
of all the lines of code. This "name" is basically what a method is.<br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;A method often "returns" a certain type
of value. For example, you may create a method that can multiply two numbers
together then add 2 to their product. This method would "return" the number
that is 2 plus the product of the two numbers put in. Here is the basic format
for a method:<br><br>
<i><pre>
public type name(arguments){
	//commands here
}
</pre>
</i><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;The first word, public, means that this
method can be accessed from other classes. If you want the method to be able to
be accessed only by the class it is in, change "public" to "private". The word
"type" in the basic format above simply means the datatype that the method is
going to return. For example, you could replace "type" with "String", "int",
"double", "long", or others.
Whatever you do, <b>do not</b> leave "type" in there. If you do not want to
return any value, replace "type" with "void". Replace "name" with whatever you
want to name the method.<br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Arguments were discussed in the tutorial I
wrote on JavaScript functions. (By the way, a function and a method are the
same thing.) An argument is a parameter that is sent to a method. In the method
which I described above that would multiply two numbers and add 2 to the
product, I would use arguments to get the original numbers. The method that I just
described could be written as the following:<br><br>
<i><pre>
public int mult2(int p, int q){
	int n;
	n=(p*q)+2;
	return n;
}
</pre></i><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;There it is. Now I will explain it. The fact that the word "public" is present
means that this method can be accessed by other classes. Since "int" is there, this method will return an integer.
Then, in between the parentheses, there is "int p, int q". These variables will be assigned a value when the method
is executed. For example, if I wanted to find out what the product of 5 and 4 with a 2 added after is, I could put the following
line in my code:<br><br>
<i>int h=mult2(5,4);</i><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;The line above creates an integer named h, and then sets its value equal to mult2(5,4). The method
named mult2 is executed with 5 replacing p, and 4 replacing q. So (5*4)+2 is equal to 22. The method returns this value because of the line
"return n;" . The integer n is simply a temporary variable that holds the value of (5*4)+2 . In between the braces is where the code which
actually makes the method do something goes. The first line in the braces just makes an integer named n. The second line changes the value of
n so that it is equal to the product of the two inputted numbers plus 2. Then there is the return statement........<br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;You probably have executed methods before. A lot of the time they return a value to you. To return a value,
you have to use the return statement in your method. You just put the word return, then a space, and then whatever you want to return. A lot of the time,
like in the example above, a variable is returned. The variable or data the you return has to be the same type as the word after public that begins the method.
For example, if my method started out with "public int mult()", I wouldn't be able to return a String value. I would only be able to return an int. If I used
"public String" as the first two words of my method, I <i>would</i> be able to return a String, but <b>not</b> an int. I think you get the idea! :)<br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;I can really think of only one more thing I need to talk about. Sometimes you may not want a value to be returned by a method.
You may just want a bunch of statements to be executed. To make a function so it doesn't return anything, put the word "void". For example, the first line might be
"public void methodname()" . The word "void" replaces the datatype that might be there if you wanted the method to return a value. Also, you have to make
sure you don't have any return statements in your method if you don't want it to return anything. To call a method that you created, just put "methodname();"
, and if you have any arguments you need to pass to the method, just put the values in between the parentheses. If there is more than 1 argument, seperate them
using commas.
