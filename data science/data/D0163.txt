    <i>Microsoft makes a point that developping for PCs and for the XBox is mostly the same thing and hence, you can release a game both for Windows and for the XBox using mostly the same codebase. I suspect this is partly why Sony is releasing a Linux distro for the PlayStation2</i><br/><br/>Isn't XBox based on a Windows PC? Since MS own both Windows and XBox, this is not surprising. I would be more surprised if XBox could run PlayStation games and vice versa! The OS is the application interface of the platform for non-Java apps, so if XBox runs a version of Windows and PlayStation runs a Linux, I would not consider this cross-platform. In this case it would be a game for Linux platform (using any hardware that runs Linux) or a game for Windows platform(running any hardware that runs Windows e.g. PC or XBox)<br/><br/><i>They would still get the same revenues because you pay to &quot;release&quot; the game. If the game is written in Perl, they don't care...</i><br/><br/>But the game developer would be releasing a game for the Java platform, so why should they pay the console vendor to release it. Unless there was some technical way to prevent users from running Java games that were not 'officially' released for a particular console, I am sure many game players would do it anyway. Considering the mass, global market, I don't think it is realistic to enforce any legal barrier. Any technical barriers are likely to be hacked soon after release.<br/><br/>A fundamental difference between the Java platform and others such as Windows, Linux is that it is easy and typical to run Java applications alongside native applications on the same hardware at the same time.<br/><br/><BR>
