<h1><a href="homepage.html">Bjarne Stroustrup</a>'s C++ Glossary</h1>
</center>
This is a glossary of C++ terms, organized alphabetically by concept.
<p>
The definitions/explanations of individual terms are necessarily very brief.
To compensate, each entry includes one or more references to
<a href="3rd.html">The C++ Programming language (Special Edition)</a>
(<a href="#GTC++PL#">TC++PL</a>)
where you can find more comprehensive explanations and code examples.
I use section references, rather than page numbers, so that this glossary can
be used together with translations of my books.
It is always wise to read a whole section rather than trying to glean
information from a few semi-random sentences.
<p>
For people interested in the reasons behind the design of C++,
most entries also refer to
<a href="dne.html">The Design and Evolution of C++</a>
(<a href="#GD&E#">D&E</a>).
Some entries contain links other parts of my home pages, especially to
my
<a href="bs_faq.html">FAQ</a> and
<a href="bs_faq2.html">C++ Style and Technique FAQ</a>.
If I find the time, I'll add references to the ISO standard.
<p>
This glossary is specifically "C++ oriented". That is, it defines terms in
the context of C++. For example, it defines generic programming in terms of
templates and object-oriented programming in terms of virtual functions,
rather than trying to be sufficiently abstract and general to cover all
languages and all usages.
<p>
The entries are meant to be brief explanations, rather than precise
definitions.
<p>
Suggestions for improved explanations, terms to add, or anything else that
would make the glossary more useful, are most wellcome: bs@research.att.com.
<p>
<hr>
<h2>Glossary</h2>
<p>
<a name="G!="><b>!=</b></a> - 
	the <a href="#Ginequality-operator">inequality operator</a>;
	compares values for inequality returning a <a href="#Gbool">bool</a>.
	TC++PL 2.3.1, 6.2, 16.3.10, 20.3.8, 22.4.3, 22.5.

<p>
<a name="G#define"><b>#define</b></a> - 
	a directive that defines a <a href="#Gmacro">macro</a>.

<p>
<a name="G#include"><b>#include</b></a> - 
	a mechanism for textual inclusion of one <a href="#Gsource-file">source file</a> into another.
	Typically used to compose a <a href="#Gtranslation-unit">translation unit</a> out of a <a href="#G.c-file">.c file</a>
	and the <a href="#Gheader-file">header file</a>s it needs to define its view if the rest
	of the <a href="#Gprogram">program</a>. 
	TC++PL 2.7, 13, D&E 18.1.

<p>
<a name="G+="><b>+=</b></a> - 
	add-and-assign <a href="#Goperator">operator</a>; a+=b is roughly equivalent to a=a+b.
	Often a useful operation for <a href="#Guser-defined-type">user-defined type</a>s.
	TC++PL 6.1.1, 6.2, 11,3,2, 20.3.9, 22.5.

<p>
<a name="G.c-file"><b>.c file</b></a> - 
	<a href="#Gfile">file</a> containing <a href="#Gdefinition#">definition</a>s.

<p>
<a name="G.cpp-file"><b>.cpp file</b></a> - 
	see <a href="#G.c-file">.c file</a>

<p>
<a name="G.cxx-file"><b>.cxx file</b></a> - 
	see <a href="#G.c-file">.c file</a>

<p>
<a name="G.h-file"><b>.h file</b></a> - 
	see <a href="#Gheader-file">header file</a>.

<p>
<a name="G14882"><b>14882</b></a> - 
	<a href="#GISO">ISO</a>/IEC 14882 - Standard for the <a href="#GC++">C++</a> <a href="#Gprogramming-language">Programming Language</a>.

<p>
<a name="G<<"><b><<</b></a> - 
	(1) <a href="#Giostream">iostream</a> output <a href="#Goperator">operator</a>.
	TC++PL 3.4, 21.2.1, D&E 8.3.1.
	(2) integer left-shift operator.
	TC++PL 6.2.

<p>
<a name="G="><b>=</b></a> - 
	the <a href="#Gassignment-operator">assignment operator</a>; <i>not</i> an <a href="#Gequality-operator">equality operator</a>.
	= can be used for non-<a href="#Gconst">const</a> <a href="#Gbuilt-in-type">built-in type</a>s (except <a href="#Garray">array</a>s),
	<a href="#Genumeration">enumeration</a>s, <a href="#Gstring">string</a>s, <a href="#Gcontainer">container</a>s, <a href="#Giterator">iterator</a>s, <a href="#Gcomplex">complex</a>, and <a href="#Gvalarray">valarray</a>.
	= is not by default defined for a <a href="#Gclass">class</a>, but the author of a
	class can define it.
	for a <a href="#Guser-defined-type">user-defined type</a>.
	TC++PL 2.3.1, 6.2, 11.2, 16.3.4, 20.3.6, 22.4.3, 22.5,
	D&E 2.12.1,11.4.4.

<p>
<a name="G=0"><b>=0</b></a> - 
	curious notation indicating that a <a href="#Gvirtual">virtual</a> <a href="#Gfunction">function</a> is a
	<a href="#Gpure-virtual-function">pure virtual function</a>.
	TC++PL 12.3. D&E 13.2.1.

<p>
<a name="G=="><b>==</b></a> - 
	the <a href="#Gequality-operator">equality operator</a>; compares values for equality returning a <a href="#Gbool">bool</a>.
	<a href="#G=">=</a>= can be used for <a href="#Gbuilt-in-type">built-in type</a>s, <a href="#Genumeration">enumeration</a>s, <a href="#Gstring">string</a>s, <a href="#Giterator">iterator</a>s,
	<a href="#Gcomplex">complex</a>, and <a href="#Gvalarray">valarray</a>.
	== is not by default defined for a <a href="#Gclass">class</a>, but a user can define it
	for a <a href="#Guser-defined-type">user-defined type</a>.
	Note that == doesn't have the naively expected meaning for
	<a href="#GC-style-string">C-style string</a>s or arrays.
	TC++PL 2.3.1, 6.2, 16.3.10, 20.3.8, 22.4.3, 22.5.

<p>
<a name="G>>"><b>>></b></a> - 
	(1) <a href="#Giostream">iostream</a> input <a href="#Goperator">operator</a>.
	TC++PL 3.6, 21.3.2, D&E 8.3.1.
	(2) integer right-shift operator.
	TC++PL 6.2.

<p>
<a name="Gabstract-class"><b>abstract class</b></a> - 
	a <a href="#Gclass">class</a> defining an <a href="#Ginterface">interface</a> only; used as a <a href="#Gbase-class">base class</a>.
	Declaring a <a href="#Gmember-function">member function</a> pure <a href="#Gvirtual">virtual</a> makes its class abstract
	and prevents creation of <a href="#Gobject">object</a>s of the abstract class.
	Use of abstract classes is one of the most effective ways of
	minimizing the impact of changes in a <a href="#GC++">C++</a> <a href="#Gprogram">program</a> and for minimizing
	compilation time.
	<a href="bs_faq2.html#abstract-class">Example</a>.
	TC++PL 2.5.4, 12.4.2, D&E 13.2.

<p>
<a name="Gabstract-type"><b>abstract type</b></a> - 
	see <a href="#Gabstract-class">abstract class</a>.

<p>
<a name="Gabstraction"><b>abstraction</b></a> - 
	the act of specifying a general <a href="#Ginterface">interface</a> <a href="#Ghiding">hiding</a> implementation
	details. <a href="#Gclass">Class</a>es, <a href="#Gabstract-class">abstract class</a>es, and <a href="#Gtemplate">template</a>s are the primary
	abstraction mechanisms in <a href="#GC++">C++</a>.
	See also: <a href="#Gencapsulation">encapsulation</a>.

<p>
<a name="Gaccess-control"><b>access control</b></a> - 
	access to bases and <a href="#Gmember">member</a>s of a <a href="#Gclass">class</a> can be controlled by declaring
	them <a href="#Gpublic">public</a>, <a href="#Gprotected">protected</a>, or <a href="#Gprivate">private</a>.
	TC++PL 15.3, D&E 2.3, 13.9.

<p>
<a name="GACCU"><b>ACCU</b></a> - 
	<a href="http://www.accu.org/">Association of C and C++ Users</a>.
	A users group that among
	other things maintains a collection of professional book reviews.

<p>
<a name="Gadapter"><b>adapter</b></a> - 
	a <a href="#Gclass">class</a> that takes arguments producing a <a href="#Gfunction-object">function object</a> that
	performs an operation based on those arguments. A simple form of
	a <a href="#Ghigher-order-function">higher-order function</a>.
	For example, <a href="#Gmem_fun()">mem_fun()</a> adapts a <a href="#Gmember-function">member function</a> for use by the
	standard <a href="#Galgorithm">algorithm</a>s.
	See also: <a href="#Gsequence-adapter">sequence adapter</a>.
	TC++PL 18.4.4.

<p>
<a name="Gaddress"><b>address</b></a> - 
	a <a href="#Gmemory">memory</a> location.
	TC++PL 5.1.

<p>
<a name="Galgorithm"><b>algorithm</b></a> - 
	a precise definition of a computation.
	The <a href="#Gstandard-library">standard library</a> provides about 60 standard algorithms, such as
	<a href="#Gsort()">sort()</a>, search(), and copy_unique().
	TC++PL 3.8, 18.

<p>
<a name="Galignment"><b>alignment</b></a> - 
	placing <a href="#Gobject">object</a>s in <a href="#Gmemory">memory</a> to suit hardware requirements. On many
	machines, an object must be aligned on a <a href="#Gword#">word</a> boundary for acceptable
	performance.

<p>
<a name="Gallocator"><b>allocator</b></a> - 
	<a href="#Gobject">object</a> used by <a href="#Gstandard-library">standard library</a> <a href="#Gcontainer">container</a>s to allocate and deallocate
	<a href="#Gmemory">memory</a>.
	TC++PL 19.4.

<p>
<a name="Gand#"><b>and</b></a> - 
	synonym for &, the logical and <a href="#Goperator">operator</a>.
	TC++PL C.3.1.

<p>
<a name="GANSI"><b>ANSI</b></a> - 
	The American national standards organization. Cooperates closely
	with <a href="#GISO">ISO</a> over the <a href="#GC++-standard">C++ standard</a>.

<p>
<a name="GANSI-C"><b>ANSI C</b></a> - 
	see <a href="#GC">C</a>.

<p>
<a name="GANSI-C++"><b>ANSI C++</b></a> - 
	<a href="#GC++">C++</a>

<p>
<a name="Gargument-passing"><b>argument passing</b></a> - 
	The semantics of <a href="#Gfunction">function</a> call is to pass a copy of an argument.
	The copy operation is defined by the argument <a href="#Gtype">type</a>'s
	<a href="#Gcopy-constructor">copy constructor</a> or by binding to a <a href="#Greference">reference</a>.
	In either case the semantics is those of <a href="#Ginitialization">initialization</a>.
	TC++PL 7.2.

<p>
<a name="Gargument-based-lookup"><b>argument-based lookup</b></a> - 
	lookup of a <a href="#Gfunction">function</a> <a href="#Gname">name</a> or <a href="#Goperator">operator</a> based on the <a href="#Gnamespace">namespace</a> of the
	arguments or operands.
	Often called <a href="#GKoenig-lookup">Koenig lookup</a> after Andrew Koenig who proposed the
	scheme to the <a href="#Gstandards-committee">standards committee</a>.
	TC++PL 8.2.6, 11.2.4, C.13.8.4.

<p>
<a name="GARM"><b>ARM</b></a> - 
	<a href="arm.html">The Annotated C++ Reference Manual</a> by
	Margaret Ellis and <a href="#GBjarne-Stroustrup">Bjarne Stroustrup</a>. The 1990 C++ reference
	manual with detailed <a href="#Gcomment">comment</a>s about <a href="#Gdesign">design</a> details and implementation
	techniques.
	Now outdated.
	See also: <a href="#GC++-standard">C++ standard</a>.

<p>
<a name="Garray"><b>array</b></a> - 
	contiguous sequence of elements. An array doesn't know its own size;
	the programmer must take care to avoid range errors.
	Where possible use the <a href="#Gstandard-library">standard library</a> <a href="#Gvector">vector</a>.
	TC++PL 5.2-3, C.7.

<p>
<a name="Gassignment-operator"><b>assignment operator</b></a> - 
	see <a href="#G=">=</a>.

<p>
<a name="GAT&T-Bell-Labs."><b>AT&T Bell Labs.</b></a> - 
	the industrial research and development labs where <a href="#GC">C</a> and <a href="#GC++">C++</a> were
	invented, initially developed, and initially used.
	D&E 2.14.

<p>
<a name="Gautomatic-garbage-collection"><b>automatic garbage collection</b></a> - 
	see <a href="#Ggarbage-collection">garbage collection</a>.

<p>
<a name="Gauto_ptr"><b>auto_ptr</b></a> - 
	<a href="#Gstandard-library">standard library</a> <a href="#Gclass-template">class template</a> for representing ownership of an
	<a href="#Gobject">object</a> in a way that
	guarantees proper release (<a href="#Gdelete">delete</a>) even when an <a href="#Gexception">exception</a> is thrown.
	See also: <a href="#Gresource-management">resource management</a>, <a href="#Gresource-acquisition-is-initialization">resource acquisition is initialization</a>.
	TC++PL 14.4.2.

<p>
<a name="Gback-end"><b>back-end</b></a> - 
	the parts of a <a href="#Gcompiler">compiler</a> that generates code given an internal
	representation of a correct <a href="#Gprogram">program</a>.
	This representation is produced by a compiler <a href="#Gfront-end">front-end</a>.
	See also: front-end.

<p>
<a name="Gbackslash"><b>backslash</b></a> - 
	see <a href="#Gescape-character">escape character</a>

<p>
<a name="Gback_inserter()"><b>back_inserter()</b></a> - 
	returns an <a href="#Giterator">iterator</a> that can be used to add elements at the back
	of a <a href="#Gcontainer">container</a>.
	TC++PL 19.2.4.

<p>
<a name="Gbad_alloc"><b>bad_alloc</b></a> - 
	standard <a href="#Gexception">exception</a> thrown by <a href="#Gnew">new</a> in case of failure to allocate
	<a href="#Gfree-store">free store</a>.
	TC++PL 6.2.6.2, 19.4.5.

<p>
<a name="Gbad_cast"><b>bad_cast</b></a> - 
	standard <a href="#Gexception">exception</a> thrown if a <a href="#Gdynamic_cast">dynamic_cast</a> to a <a href="#Greference">reference</a> fails.
	TC++PL 15.4.1.1, D&E 14.2.2.
	.
out_of_range
	standard exception thrown by <a href="#Gvector">vector</a> if an argument to at() is out
	of range.
	TC++PL 16.3.3.

<p>
<a name="Gbase-class"><b>base class</b></a> - 
	a <a href="#Gclass">class</a> from which another is derived. 
	TC++PL 2.6.2, 12, 15, D&E 2.9.

<p>
<a name="Gbase-initializer"><b>base initializer</b></a> - 
	initializer for a <a href="#Gbase-class">base class</a> specified in the <a href="#Gconstructor">constructor</a> for a
	<a href="#Gderived-class">derived class</a>.
	TC++PL 12.2.2, 15.2.4.1, D&E 12.9.

<p>
<a name="Gbasic-guarantee"><b>basic guarantee</b></a> - 
	the guarantee that basic <a href="#Ginvariant">invariant</a>s are maintained if an <a href="#Gexception">exception</a> is
	thrown and that no <a href="#Gresource">resource</a>s are leaked/lost.
	Provided by all <a href="#Gstandard-library">standard library</a> operations.
	See also exception safety.
	TC++PL E.2.

<p>
<a name="Gbasic_string"><b>basic_string</b></a> - 
	general standard-library <a href="#Gstring">string</a> <a href="#Gtemplate">template</a>  parameterized by
	<a href="#Gcharacter-type">character type</a>.
	See also: string, <a href="#GC-style-string">C-style string</a>.
	TC++PL 20.3.

<p>
<a name="GBCPL"><b>BCPL</b></a> - 
	ancestor to <a href="#GC">C</a> and <a href="#GC++">C++</a> designed and implemented by Martin Richards.
	TC++PL 1.4, D&E 1.1, 3.1.

<p>
<a name="GBell-labs"><b>Bell labs</b></a> - 
	see <a href="#GAT&T-Bell-Labs.">AT&T Bell Labs.</a>

<p>
<a name="Gbinder"><b>binder</b></a> - 
	a <a href="#Gfunction">function</a> taking a function and a value, returning a <a href="#Gfunction-object">function object</a>;
	when called, that function object will invoke the function with the
	value as an argument in addition to other arguments supplied in the
	call.
	The <a href="#Gstandard-library">standard library</a> provides bind1st() and bind2nd() for binding
	the first and second argument of a binary function, respectively.
	TC++PL 18.4.4.

<p>
<a name="Gbit"><b>bit</b></a> - 
	a unit of <a href="#Gmemory">memory</a> that can hold 0 or 1. An individual bit cannot be
	directly accessed in <a href="#GC++">C++</a> (the unit of addressing is a <a href="#Gbyte">byte</a>), but a
	bit can be accessed through a <a href="#Gbitfield">bitfield</a> or by using the bitwise
	logical <a href="#Goperator">operator</a>s & and |.
	TC++PL 6.2.4.

<p>
<a name="Gbitand#"><b>bitand</b></a> - 
	synonym for &, the bitwise and <a href="#Goperator">operator</a>.
	TC++PL C.3.1.

<p>
<a name="Gbitfield"><b>bitfield</b></a> - 
	a number of <a href="#Gbit">bit</a>s in a <a href="#Gword#">word</a> made accessible as a <a href="#Gstruct">struct</a> <a href="#Gmember">member</a>.
	TC++PL C.8.1	

<p>
<a name="Gbitor#"><b>bitor</b></a> - 
	synonym for |, the bitwise or <a href="#Goperator">operator</a>
	TC++PL C.3.1.

<p>
<a name="Gbitset"><b>bitset</b></a> - 
	a <a href="#Gstandard-library">standard library</a> "almost <a href="#Gcontainer">container</a>" holding N <a href="#Gbit">bit</a>s and providing
	logical operations on those.
	TC++PL 17.5.3.

<p>
<a name="GBjarne-Stroustrup"><b>Bjarne Stroustrup</b></a> - 
	the designer and original implementor of <a href="#GC++">C++</a>.
	The author of this <a href="#Gglossary">glossary</a>.
	See also: <a href="homepage.html">my home page</a>.

<p>
<a name="Gblock"><b>block</b></a> - 
	see <a href="#Gcompound-statement">compound statement</a>.
	See also: <a href="#Gtry-block">try-block</a>.

<p>
<a name="Gblock-comment"><b>block comment</b></a> - 
	<a href="#Gcomment">comment</a> started by /* and terminated by */.
	TC++PL 6.4, D&E 3.11.1.

<p>
<a name="Gbool"><b>bool</b></a> - 
	the built-in Boolean <a href="#Gtype">type</a>.
	A bool can have the values <a href="#Gtrue">true</a> and <a href="#Gfalse">false</a>.
	TC++PL 4.2, D&E 11.7.2.

<p>
<a name="Gboost.org"><b>boost.org</b></a> - 
	a collection people - many with ties to the <a href="#GC++-standard">C++ standard</a>s committee -
	devoted to creating a body of quality - peer reviewed - open source
	libraries designed to interoperate with the <a href="#Gstandard-library">standard library</a>.
	Their central "home" is
	<a href="http://www.boost.org">their website</a>.

<p>
<a name="GBorland-C++-Builder"><b>Borland C++ Builder</b></a> - 
	Borland's implementation of <a href="#GC++">C++</a> together with proprietary libraries
	for Windows programming in an IDE.

<p>
<a name="Gbuilt-in-type"><b>built-in type</b></a> - 
	A <a href="#Gtype">type</a> provided directly by <a href="#GC++">C++</a>, such as <a href="#Gint">int</a>, <a href="#Gdouble">double</a>, and <a href="#Gchar*">char*</a>.
	See also: <a href="#Gintegral-type">integral type</a>s, <a href="#Gfloating-point-type">floating-point type</a>, <a href="#Gpointer">pointer</a>, <a href="#Greference">reference</a>.
	TC++PL 4.1.1, 5.1, 5.2, 5.5, D&E 4.4, 15.11.3.

<p>
<a name="Gbyte"><b>byte</b></a> - 
	a unit of <a href="#Gmemory">memory</a> that can hold a character of the <a href="#GC++">C++</a> representation
	<a href="#Gcharacter-set">character set</a>.
	The smallest unit of memory that can be directly addressed in C++.
	Usually, a byte is 8 <a href="#Gbit">bit</a>s.
	TC++PL 4.6.

<p>
<a name="GC"><b>C</b></a> - 
	<a href="#Gprogramming-language">programming language</a> designed and originally implemented by
	Dennis <a href="#GRitchie">Ritchie</a>. <a href="#GC++">C++</a> is based on C and maintains a high degree of
	<a href="bs_faq.html#C-is-subset">compatibility with C</a>.
	See also: <a href="#GK&R-C">K&R C</a>, <a href="#GC89">C89</a>, <a href="#GC99">C99</a>, <a href="#GANSI-C">ANSI C</a>.
	TC++PL B, D&E 3.12.

<p>
<a name="GC-standard-library"><b>C standard library</b></a> - 
	the library defined for <a href="#GC">C</a> in the C standard.
	Inherited by <a href="#GC++">C++</a>.
	Most C <a href="#Gstandard-library">standard library</a> <a href="#Gfunction">function</a>s have safer and more convenient
	alternatived in the <a href="#GC++-standard">C++ standard</a> library.
	See also: <a href="#Galgorithm">algorithm</a>, <a href="#Gcontainer">container</a>, <a href="#Gstream-I/O">stream I/O</a>, <a href="#Gstring">string</a>, <a href="#Glocale">locale</a>.

<p>
<a name="GC++"><b>C++</b></a> - 
	a <a href="#Ggeneral-purpose-programming-language">general-purpose programming language</a> with a bias towards
	systems programming that supports <a href="#Gprocedural-programming">procedural programming</a>,
	<a href="#Gdata-abstraction">data abstraction</a>,
	<a href="#Gobject-oriented-programming">object-oriented programming</a>, and <a href="#Ggeneric-programming">generic programming</a>.
	<a href="#GC">C</a>++ was designed and originally implemented by <a href="#GBjarne-Stroustrup">Bjarne Stroustrup</a>.
	C++ is defined by
	<a href="#GISO">ISO</a>/IEC <a href="#G14882">14882</a> - Standard for the C++ Programming Language.
	<a href="#GTC++PL#">TC++PL</a> describes C++ and the fundamental techniques for its use.
	A description of the design considerations for C++ can be found
	in <a href="#GD&E#">D&E</a>.
	Many commercial and
	<a href="bs_faq.html#free">free implementations</a> exist.
	TC++PL 1.3,-5, 2.1, D&E 0.

<p>
<a name="GC++-standard"><b>C++ standard</b></a> - 
	the definition of <a href="#GC++">C++</a> provided by <a href="#GISO">ISO</a>.
	Available from <a href="#GANSI">ANSI</a>; see <a href="C++.html">my C++ page</a>.
	TC++PL 1.4, B.1. D&E 6.1.

<p>
<a name="GC++-standards-committees"><b>C++ standards committees</b></a> - 
	the <a href="#GISO">ISO</a> committee for <a href="#GC++">C++</a>
	(<a href="http://anubis.dkuug.dk/jtc1/sc22/wg21/">WG21</a>)
	and the various national standards
	committees that closely cooperate with it (BIS, AFNOR, DIN, etc.).
	<a href="http://www.research.att.com/~bs/bs_faq.html#spoil-C++">Did
	the ANSI/ISO standards committee spoil C++?</a>.
	See also: <a href="#GC++-standard">C++ Standard</a>.
	D&E 6.2.

<p>
<a name="GC++-standards-process"><b>C++ standards process</b></a> - 
	see <a href="#GC++-standards-committees">C++ standards committees</a>

<p>
<a name="GC-style-cast"><b>C-style cast</b></a> - 
	dangerous form of <a href="#Gexplicit-type-conversion">explicit type conversion</a>; prefer <a href="#Gnew-style-cast">new-style cast</a>
	if you must use explicit type conversion.
	TC++PL 6.2.7, D&E 14.3.5.1.

<p>
<a name="GC-style-string"><b>C-style string</b></a> - 
	<a href="#Gzero">zero</a>-terminated <a href="#Garray">array</a> of characters, supported by <a href="#GC-standard-library">C standard library</a>
	<a href="#Gfunction">function</a>s.
	A low-level and error-prone mechanism; where possible prefer <a href="#Gstring">string</a>s.
	TC++PL 3.5.1, 20.3.7, 20.4.

<p>
<a name="GC/C++"><b>C/C++</b></a> - 
	(1) an abbreviation used when discussing similarities, differences,
	and <a href="#Gcompatibility">compatibility</a> issues of <a href="#GC">C</a> and <a href="#GC++">C++</a>.
	(2) a mythical language referred to by people who cannot or do not
	want to recognize the magnitude of differences between the facilities
	offered by C and C++ or the significant differences in the
	programming styles supported by the two language.
	See also: <a href="#Gmulti-paradigm-programming">multi-paradigm programming</a>,
	<a href="#Gobject-oriented-programming">object-oriented programming</a>, <a href="#Ggeneric-programming">generic programming</a>, <a href="#Gexception">exception</a>, <a href="#Gtemplate">template</a>,
	<a href="#Guser-defined-type">user-defined type</a>, <a href="#GC++-standard">C++ standard</a> library.

<p>
<a name="GC/C++-compatibility"><b>C/C++ compatibility</b></a> - 
	<a href="#GC++">C++</a> was designed to be as compatible as possible to C, but no more.
	This basically means as compatible as can be without compromising
	C++'s level of <a href="#Gtype-safety">type safety</a>.
	You can download Appendix B of <a href="#GTC++PL#">TC++PL</a>,.
	<a href="3rd_compat.pdf">Compatibility</a>, which describes
	incompatibilities and differences in facilities offered by C and C++.
	TC++PL B. D&E 2.7, 3.12, 4.5.

<p>
<a name="GC89"><b>C89</b></a> - 
	The 1989 <a href="#GANSI">ANSI</a> standard for <a href="#GC">C</a> based on <a href="#GK&R-C">K&R C</a> with a few additions
	borrowed from <a href="#GC++">C++</a>, such as <a href="#Gfunction-prototype">function prototype</a>s and <a href="#Gconst">const</a>.
	See also: K&R C, <a href="#GC99">C99</a>.

<p>
<a name="GC99"><b>C99</b></a> - 
	The 1999 <a href="#GISO">ISO</a> standard for <a href="#GC">C</a> based on <a href="#GC89">C89</a> with additions to support
	Fortran-style numeric computation. It also borrows a few more features,
	such as <a href="#Gline-comment">line comment</a>s (// comments) and <a href="#Gdeclaration">declaration</a>s as <a href="#Gstatement">statement</a>s,
	from <a href="#GC++">C++</a>.

<p>
<a name="Gcall-by-reference"><b>call-by-reference</b></a> - 
	declaring a <a href="#Gfunction">function</a> argument <a href="#Gtype">type</a> to be a <a href="#Greference">reference</a>, thus passing
	a reference rather than a value to the called function.
	See Also: <a href="#Gcall-by-value">call-by-value</a>.
	TC++PL 5.5, D&E 3.7.

<p>
<a name="Gcall-by-value"><b>call-by-value</b></a> - 
	passing a copy of an argument to the called <a href="#Gfunction">function</a>.
	The semantics of function call is to pass a copy of an argument.
	The copy operation is defined by the argument <a href="#Gtype">type</a>'s
	<a href="#Gcopy-constructor">copy constructor</a>.
	See Also: <a href="#Gcall-by-reference">call-by-reference</a>.
	TC++PL 7.2.

<p>
<a name="Gcast"><b>cast</b></a> - 
	<a href="#Goperator">operator</a> for <a href="#Gexplicit-type-conversion">explicit type conversion</a>; most often best avoided.
	See also <a href="#Gdynamic_cast">dynamic_cast</a>, <a href="#GC-style-cast">C-style cast</a>, <a href="#Gnew-style-cast">new-style cast</a>.
	TC++PL 6.2.7, D&E 7.2, 14.2.2.1.

<p>
<a name="Gcatch"><b>catch</b></a> - 
	keyword used to introduce a <a href="#Gcatch-clause">catch-clause</a>.

<p>
<a name="Gcatch(...)"><b>catch(...)</b></a> - 
	<a href="#Gcatch">catch</a> every <a href="#Gexception">exception</a>.
	TC++PL 14.3.2, D&E 16.5.

<p>
<a name="Gcatch-clause"><b>catch-clause</b></a> - 
	a part of a <a href="#Gtry-block">try-block</a> that <a href="#Ghandle">handle</a>s <a href="#Gexception">exception</a>s of a specified <a href="#Gtype">type</a>.
	Also called a <a href="#Ghandler">handler</a> or an <a href="#Gexception-handler">exception handler</a>.
	TC++PL 8.3.1, 14.3, D&E 16.3-4.

<p>
<a name="Gcerr"><b>cerr</b></a> - 
	standard unbuffered <a href="#Gostream">ostream</a> for error or diagnostic output.
	TC++PL 21.2.1.

<p>
<a name="GCfront"><b>Cfront</b></a> - 
	the <a href="#Gfront-end">front-end</a> of <a href="#GBjarne-Stroustrup">Bjarne Stroustrup</a>'s original <a href="#GC++">C++</a> <a href="#Gcompiler">compiler</a>.
	D&E 3.3.

<p>
<a name="Gchar"><b>char</b></a> - 
	<a href="#Gcharacter-type">character type</a>; typically an 8-<a href="#Gbit">bit</a> <a href="#Gbyte">byte</a>.
	See also: <a href="#Gwchar_t">wchar_t</a>.
	TC++PL 4.3, C.3.4.

<p>
<a name="Gchar*"><b>char*</b></a> - 
	<a href="#Gpointer">pointer</a> to a <a href="#Gchar">char</a> or an <a href="#Garray">array</a> of char. Typically assumed to point
	to a <a href="#GC-style-string">C-style string</a>.
	Prefer a <a href="#Gstandard-library">standard library</a> string over a C-style string when you can.
	TC++PL 2.3.3, 13.5.2.

<p>
<a name="Gcharacter-set"><b>character set</b></a> - 
	a set of integer values with a mapping to character representations;
	for example, ASCII (<a href="#GANSI">ANSI</a>13.4-1968) gives meaning to the values 0-127.
	ASCII is <a href="#GC++">C++</a>'s representation character set, the character set used
	to represent <a href="#Gprogram">program</a> source text.
	TC++PL C.3. D&E 6.5.3.

<p>
<a name="Gcharacter-type"><b>character type</b></a> - 
	<a href="#Gchar">char</a>, unsigned char, and signed char. These are three distinct <a href="#Gtype">type</a>s.
	See also: <a href="#Gwchar_t">wchar_t</a>.
	TC++PL 2.3.1, 4.3, C.3.4.

<p>
<a name="Gcin"><b>cin</b></a> - 
	standard <a href="#Gistream">istream</a>.
	TC++PL 3.6, 21.3.1 D&E 8.3.1.

<p>
<a name="Gclass"><b>class</b></a> - 
	a <a href="#Guser-defined-type">user-defined type</a>. A class can have <a href="#Gmember-function">member function</a>s, <a href="#Gmember-data">member data</a>,
	<a href="#Gmember-constant">member constant</a>s, and <a href="#Gmember-type">member type</a>s.
	A class is thee primary mechanism for representing concepts in <a href="#GC++">C++</a>.
	See also: <a href="#Gtemplate-class">template class</a>.
	TC++PL 2.5.2, 10, D&E 2.3.

<p>
<a name="Gclass-hierarchy"><b>class hierarchy</b></a> - 
	a collection of <a href="#Gclass">class</a>es organized into a directed acyclic graph (DAG)
	by derived/base relationships.
	TC++PL 2.6.2, 12, 15, D&E 1.1, 7.2, 8.2.3.

<p>
<a name="Gclass-template"><b>class template</b></a> - 
	see <a href="#Gtemplate-class">template class</a>.

<p>
<a name="GClassic-C"><b>Classic C</b></a> - 
	see <a href="#GK&R-C">K&R C</a>.

<p>
<a name="Gclosure"><b>closure</b></a> - 
	<a href="#Gobject">object</a> representing a context. <a href="#GC++">C++</a> does not have general closures,
	but <a href="#Gfunction-object">function object</a>s can be efficiently used to hold specific parts
	of a context relevant to a computation.
	TC++PL 22.4.7, 18.4.

<p>
<a name="Gco-variant-return-type"><b>co-variant return type</b></a> - 
	see <a href="#Greturn-type-relaxation">return type relaxation</a>.

<p>
<a name="Gcode-generator"><b>code generator</b></a> - 
	the part of a <a href="#Gcompiler">compiler</a> that takes the output from the <a href="#Gfront-end">front-end</a> and
	generates code from it.
	See also: <a href="#Gback-end">back-end</a>, <a href="#Goptimizer">optimizer</a>.

<p>
<a name="GComeau-C++"><b>Comeau C++</b></a> - 
	a family of ports of the <a href="#GEDG-C++-front-end">EDG C++ front-end</a>.

<p>
<a name="Gcomment"><b>comment</b></a> - 
	<a href="#Gblock-comment">block comment</a> /* ... */ or <a href="#Gline-comment">line comment</a> // ...

<p>
<a name="Gcompatibility"><b>compatibility</b></a> - 
	see <a href="#GC/C++-compatibility">C/C++ compatibility</a>.

<p>
<a name="Gcompiler"><b>compiler</b></a> - 
	the part of a <a href="#GC++">C++</a> implementation that produces <a href="#Gobject-code">object code</a> from a
	<a href="#Gtranslation-unit">translation unit</a>.
	See also: <a href="#Gfront-end">front-end</a>, <a href="#Gback-end">back-end</a>.

<p>
<a name="Gcomplex"><b>complex</b></a> - 
	<a href="#Gstandard-library">standard library</a> complex number <a href="#Gtemplate">template</a> parameterized by scalar <a href="#Gtype">type</a>.
	TC++PL 11.3, 22.5, D&E 3.6.1, 8.5, 15.10.2.1.

<p>
<a name="Gcompound-statement"><b>compound statement</b></a> - 
	sequence of <a href="#Gstatement">statement</a>s enclosed in curly braces: { ... }
	See also: <a href="#Gtry-block">try-block</a>.
	TC++PL 2.3, 6.3.

<p>
<a name="Gconcept-checking"><b>concept checking</b></a> - 
	see <a href="#Gconstraint">constraint</a>.

<p>
<a name="Gconcrete-type"><b>concrete type</b></a> - 
	a <a href="#Gtype">type</a> without <a href="#Gvirtual">virtual</a> <a href="#Gfunction">function</a>s, so that <a href="#Gobject">object</a>s of the type can
	be allocated on the <a href="#Gstack">stack</a> and manipulated directly (without a need to
	use <a href="#Gpointer">pointer</a>s or <a href="#Greference">reference</a>s to allow the possibility for derived
	<a href="#Gclass">class</a>es). Often, small <a href="#Gself">self</a>-contained classes.
	See also <a href="#Gabstract-class">abstract class</a>, <a href="#Gvector">vector</a>, <a href="#Glist#">list</a>, <a href="#Gstring">string</a>, <a href="#Gcomplex">complex</a>.
	TC++PL 25.2.

<p>
<a name="Gconst"><b>const</b></a> - 
	attribute of a <a href="#Gdeclaration">declaration</a> that makes the entity to which it refers
	readonly.
	See also: <a href="#Gconst-member-function">const member function</a>.
	TC++PL 5.4, D&E 3.8.

<p>
<a name="Gconst-definition"><b>const definition</b></a> - 
	<a href="#Gdeclaration">declaration</a> of a <a href="#Gconst">const</a> including an initializer.

<p>
<a name="Gconst-member-function"><b>const member function</b></a> - 
	<a href="#Gmember-function">member function</a> declared not to modify the state of the <a href="#Gobject">object</a> for
	which it is called. Can be called for <a href="#Gconst">const</a> objects only.
	TC++PL 10.2.6, D&E 13.3.

<p>
<a name="Gconstant"><b>constant</b></a> - 
	<a href="#Gliteral">literal</a>, <a href="#Gobject">object</a> or value declared <a href="#Gconst">const</a>, or <a href="#Genumerator">enumerator</a>.

<p>
<a name="Gconstant-expression"><b>constant expression</b></a> - 
	<a href="#Gexpression">expression</a> of <a href="#Gintegral-type">integral type</a> that is evaluated at compile time.
	TC++PL C.5.

<p>
<a name="Gconstraint"><b>constraint</b></a> - 
	rule that restricts the set of acceptable arguments for a <a href="#Gtemplate">template</a>
	parameter. For example "the argument must have + and - <a href="#Goperator">operator</a>s".
	<a href="bs_faq2.html#constraints">Examples</a>.
	D&E 15.4.

<p>
<a name="Gconstructor"><b>constructor</b></a> - 
	<a href="#Gmember-function">member function</a> with the same <a href="#Gname">name</a> as its <a href="#Gclass">class</a>, used to initialize
	<a href="#Gobject">object</a>s of its class.
	Often used to establish an <a href="#Ginvariant">invariant</a> for the class.
	Often used to acquire <a href="#Gresource">resource</a>s.
	A constructor establishes a local environment in which
	member functions execute.
	See also: <a href="#Gorder-of-construction">order of construction</a>, <a href="#Gdestructor">destructor</a>.
	TC++PL 10.2.3, D&E 2.11.1.

<p>
<a name="Gconst_cast"><b>const_cast</b></a> - 
	a <a href="#Gtype-conversion">type conversion</a> operation that conversion between types that
	differ in <a href="#Gconst">const</a> and <a href="#Gvolatile">volatile</a> type modifiers only.
	See also: <a href="#Gcast">cast</a>.
	TC++PL 15.4.2.1, D&E 14.3.4.

<p>
<a name="Gcontainer"><b>container</b></a> - 
	(1) <a href="#Gobject">object</a> that holds other objects.
	(2) <a href="#Gtype">type</a> of object that holds other objects.
	(3) <a href="#Gtemplate">template</a> that generates types of objects that hold other objects.
	(4) <a href="#Gstandard-library">standard library</a> template such as <a href="#Gvector">vector</a>, <a href="#Glist#">list</a>, and <a href="#Gmap">map</a>.
	TC++PL 16.2, 16.2.3, 17, D&E 15.3.

<p>
<a name="Gconversion"><b>conversion</b></a> - 
	<a href="#Gexplicit-type-conversion">explicit type conversion</a> or <a href="#Gimplicit-type-conversion">implicit type conversion</a>.
	See also: <a href="#Guser-defined-type-conversion">user-defined type conversion</a>.

<p>
<a name="Gconversion-operator"><b>conversion operator</b></a> - 
	<a href="#Goperator-function">operator function</a> specifying a <a href="#Gconversion">conversion</a> from a <a href="#Guser-defined-type">user-defined type</a>
	to either another user-defined type or a <a href="#Gbuilt-in-type">built-in type</a>.
	Note that <a href="#Gconstructor">constructor</a>s cannot define conversions to built-in types.
	TC++PL 11.4, D&E 3.6.3.

<p>
<a name="Gcopy-assignment"><b>copy assignment</b></a> - 
	an assignment accepting an <a href="#Gobject">object</a> of the <a href="#Gclass">class</a> itself as its argument,
	typically Z::<a href="#Goperator">operator</a>=(<a href="#Gconst">const</a> Z&).
	A copy assignment is used for assignment
	of an object of <a href="#Gtype">type</a> T with an object of type T.
	If a copy assignment is not declared for a class,
	<a href="#Gmemberwise-copy">memberwise copy</a> is used.
	See also: <a href="#Gcopy-constructor">copy constructor</a>.
	TC++PL 10.4.4.1, 10.4.6.3 D&E 11.4.

<p>
<a name="Gcopy-constructor"><b>copy constructor</b></a> - 
	a <a href="#Gconstructor">constructor</a> accepting an <a href="#Gobject">object</a> of the <a href="#Gclass">class</a> itself as its argument,
	typically Z::Z(const Z&). A copy constructor is used for <a href="#Ginitialization">initialization</a>
	of objects of <a href="#Gtype">type</a> T with objects of type T.
	If a copy constructor is not declared for a class,
	memberwise initialization is used.
	See also: <a href="#Gcall-by-value">call-by-value</a>, <a href="#Gargument-passing">argument passing</a>, <a href="#Gvalue-return">value return</a>,
	<a href="#Gcopy-assignment">copy assignment</a>.
	TC++PL 10.4.4.1, 10.4.6.3, D&E 11.4.

<p>
<a name="Gcopy()"><b>copy()</b></a> - 
	standard <a href="#Galgorithm">algorithm</a> for copying one sequence into another.
	The two sequences need not be of the same <a href="#Gtype">type</a>.
	TC++PL 18.6.1.

<p>
<a name="Gcopying-class-object"><b>copying class object</b></a> - 
	an <a href="#Gobject">object</a> of a <a href="#Gclass">class</a> is copied by the class' <a href="#Gcopy-assignment">copy assignment</a> and
	<a href="#Gcopy-constructor">copy constructor</a>s.
	The default meaning of these operations is <a href="#Gmemberwise-copy">memberwise copy</a>.
	TC++PL 10.4.4.1, 10.4.6.3 D&E 11.4.

<p>
<a name="Gcout"><b>cout</b></a> - 
	standard <a href="#Gostream">ostream</a>.
	TC++PL 3.4, 21.2.1, D&E 8.3.1.

<p>
<a name="Gcpp"><b>cpp</b></a> - 
	see <a href="#Gpreprocessor">preprocessor</a>.

<p>
<a name="Gcrosscast"><b>crosscast</b></a> - 
	a <a href="#Gcast">cast</a> from a <a href="#Gclass">class</a> to a <a href="#Gsibling-class">sibling class</a>.
	See also: <a href="#Gdynamic_cast">dynamic_cast</a>, <a href="#Gupcast">upcast</a>, <a href="#Gdowncast">downcast</a>.
	TC++PL 15.4.

<p>
<a name="GCurrying"><b>Currying</b></a> - 
	producing a <a href="#Gfunction">function</a> of N-M arguments by specifying M arguments for
	a function of N arguments.
	See also: <a href="#Gbinder">binder</a>, <a href="#Gdefault-argument">default argument</a>.
	TC++PL 18.4.4.1.

<p>
<a name="GD&E#"><b>D&E</b></a> - 
	<a href="#GBjarne-Stroustrup">Bjarne Stroustrup</a>:
	<a href="dne.html">The Design and Evolution of C++</a>.
	Addison Wesley. 1994.
	A book describing why C++ looks the way it does - the closest to a
	design rationale that we have for C++.

<p>
<a name="Gdata-abstraction"><b>data abstraction</b></a> - 
	programming relying on <a href="#Guser-defined-type">user-defined type</a>s with well-defined <a href="#Ginterface">interface</a>s.
	See also: <a href="#Ggeneric-programming">generic programming</a> and <a href="#Gobject-oriented-programming">object-oriented programming</a>.
	TC++PL 2.5, 24.2.2, D&E 9.2.1.

<p>
<a name="Gdata-hiding"><b>data hiding</b></a> - 
	see <a href="#Ginformation-hiding">information hiding</a>

<p>
<a name="Gdata-member"><b>data member</b></a> - 
	<a href="#Gmember">member</a> of a <a href="#Gclass">class</a> that can hold a value.
	A memer can be a <a href="#Gstatic-member">static member</a> or a <a href="#Gnon-static-member">non-static member</a>.
	TC++PL 2.5.2-3, 10.2, D&E 2.3, 2.5.2.

<p>
<a name="Gdeclaration"><b>declaration</b></a> - 
	an introduction of a <a href="#Gname">name</a> into a <a href="#Gscope">scope</a>. The <a href="#Gtype">type</a> of the name must be
	specified. If the declaration also specifies the entity to which the
	name refers, the declaration is also a <a href="#Gdefinition#">definition</a>.
	TC++PL 4.9, D&E 3.11.5.

<p>
<a name="Gdefault-argument"><b>default argument</b></a> - 
	a value specified for an argument in a <a href="#Gfunction-declaration">function declaration</a>, to be
	used if a call of the function doesn't specify a value for that
	argument.
	This is commonly used to allow a simple <a href="#Ginterface">interface</a> for common uses
	while making it easy to use less common facilities by specifying
	more arguments.
	See also: <a href="#Gdefault-template-argument">default template argument</a>, <a href="#Gbinder">binder</a>.
	TC++PL 7.5, 10.2.3, D&E 2.12.2.

<p>
<a name="Gdefault-constructor"><b>default constructor</b></a> - 
	<a href="#Gconstructor">constructor</a> requiring no arguments. Used for default <a href="#Ginitialization">initialization</a>.
	TC++PL 10.4.2, 10.4.6, D&E 2.12.2, 15.11.3.

<p>
<a name="Gdefault-template-argument"><b>default template argument</b></a> - 
	a <a href="#Gtype">type</a> or value specified for an argument in a <a href="#Gtemplate">template</a> <a href="#Gdeclaration">declaration</a>,
	to be used if a use of the template doesn't provide a type or value
	for that argument.
	This is commonly used to allow a simple <a href="#Ginterface">interface</a> for common uses
	while making it easy to use less common facilities by specifying
	more arguments.
	See also: <a href="#Gdefault-argument">default argument</a>.
	TC++PL 13.4.1, B.3.5.

<p>
<a name="Gdefault-value"><b>default value</b></a> - 
	value defined by a <a href="#Gdefault-constructor">default constructor</a>. For <a href="#Gbuilt-in-type">built-in type</a>s, the
	default value is defined to be 0.
	TC++PL 4.9.5, 10.3.1, 10.4.2 D&E 15.11.3.

<p>
<a name="Gdefinition#"><b>definition</b></a> - 
	a <a href="#Gdeclaration">declaration</a> that specifies the entity to which the declared <a href="#Gname">name</a>
	refers.
	See also: <a href="#Gone-definition-rule">one definition rule</a>, <a href="#Gvariable-definition">variable definition</a>, <a href="#Gconst-definition">const definition</a>,
	<a href="#Gtemplate-definition">template definition</a>, <a href="#Gfunction-definition">function definition</a>.
	TC++PL 4.9, D&E 15.11.3.

<p>
<a name="Gdelayed-evaluation"><b>delayed evaluation</b></a> - 
	technique for eliminating temporary values, and in general to delay
	a computation until sufficient information is available to do it well.
	TC++PL 21.4.6.3, 22.4.7.

<p>
<a name="Gdelete"><b>delete</b></a> - 
	<a href="#Gobject">object</a> destruction <a href="#Goperator">operator</a>. Invokes <a href="#Gdestructor">destructor</a>, if any.
	See also: <a href="#Gresource-management">resource management</a>, <a href="#Gmemory-management">memory management</a>, <a href="#Ggarbage-collection">garbage collection</a>.
	TC++PL 6.2.6, D&E 2.3, 10.2.

<p>
<a name="Gdeprecated-feature"><b>deprecated feature</b></a> - 
	feature left in a <a href="#Gprogramming-language">programming language</a> for historical reasons only.
	The standard s committee recommends against its use and warns that
	it may be removed in future revisions of the standard.

<p>
<a name="Gdeque"><b>deque</b></a> - 
	<a href="#Gdouble">double</a>-ended <a href="#Gqueue">queue</a> (pronounced "deck"). A <a href="#Gstandard-library">standard library</a> <a href="#Gtemplate">template</a>
	alowing insertions and deletions at both ends. Use a <a href="#Gvector">vector</a> if you
	need insertions and deletions only at one end (as is typical). Use a

	<a href="#Gfile">file</a> holding <a href="#Gdeclaration">declaration</a>s used in more than one <a href="#Gtranslation-unit">translation unit</a>.
	Thus, a <a href="#Gheader">header</a> file acts as an <a href="#Ginterface">interface</a> between <a href="#Gseparately-compiled">separately compiled</a>
	parts of a <a href="#Gprogram">program</a>.
	A header file often contains <a href="#Ginline-function">inline function</a> definitions,
	<a href="#Gconst-definition">const definition</a>s, <a href="#Genumeration">enumeration</a>s, and <a href="#Gtemplate-definition">template definition</a>s,
	but it cannot
	be #included from for than one <a href="#Gsource-file">source file</a> if it contain non-inline
	function definitions or <a href="#Gvariable-definition">variable definition</a>s.
	TC++PL 2.4.1, 9.2.1. D&E 2.5, 11.3.3.

<p>
<a name="Ghiding"><b>hiding</b></a> - 
	see <a href="#Ginformation-hiding">information hiding</a>

<p>
<a name="Ghierarchy"><b>hierarchy</b></a> - 
	see <a href="#Gclass-hierarchy">class hierarchy</a>.

<p>
<a name="Ghigher-order-function"><b>higher-order function</b></a> - 
	<a href="#Gfunction">function</a>s that produce other functions. <a href="#GC++">C++</a> does not have general
	higher-order functions, but by returning <a href="#Gfunction-object">function object</a>s a function
	can efficiently emulate some techniques traditionally relying of
	higher-order functions.
	See also: <a href="#Gbinder">binder</a>. 
	TC++PL 18.4.4.

<p>
<a name="Ghistory-of-C++"><b>history of C++</b></a> - 
	The work on what became <a href="#GC++">C++</a> started by <a href="#GBjarne-Stroustrup">Bjarne Stroustrup</a> in
	AT&T <a href="#GBell-labs">Bell Labs</a> in 1979. The first commercial release was in 1985.
	Standards work stared in 1990 leading to ratification of the <a href="#GISO">ISO</a>
	standard in 1998.
	TC++PL 1.4. D&E Part 1.

<p>
<a name="GHungarian-notation"><b>Hungarian notation</b></a> - 
	a coding convention that encodes <a href="#Gtype">type</a> information in <a href="#Gvariable">variable</a> <a href="#Gname">name</a>s.
	Its main use is to compensate for lack of <a href="#Gtype-checking">type checking</a> in
	weakly-typed or untyped languages. It is totally unsutable for <a href="#GC++">C++</a>
	where it complicates <a href="#Gmaintenance">maintenance</a> and gets in the way of <a href="#Gabstraction">abstraction</a>.

<p>
<a name="Ghybrid-language"><b>hybrid language</b></a> - 
	derogative term for a <a href="#Gprogramming-language">programming language</a> that supports more
	programming styles (<a href="#Gparadigm">paradigm</a>s) rather than just
	<a href="#Gobject-oriented-programming">object-oriented programming</a>.

<p>
<a name="GI/O"><b>I/O</b></a> - 
	see <a href="#Giostream">iostream</a>

<p>
<a name="Gidentifier"><b>identifier</b></a> - 
	see <a href="#Gname">name</a>.

<p>
<a name="Gif-statement"><b>if-statement</b></a> - 
	<a href="#Gstatement">statement</a> selecting between two alternatives based on a condition.
	TC++PL 6.3.2.

<p>
<a name="Gifstream"><b>ifstream</b></a> - 
	an <a href="#Gfile-stream">file stream</a> for input.

<p>
<a name="Gimplementation-defined"><b>implementation defined</b></a> - 
	an aspect of <a href="#GC++">C++</a>'s semantics that is defined for each implementation
	rather than specified in the standard for every implementation.
	An example is the size of an <a href="#Gint">int</a> (which must be at least 16 <a href="#Gbit">bit</a>s
	but can be longer).
	Avoid implementation defined behavior whenever possible.
	TC++PL C.2.

<p>
<a name="Gimplementation-inheritance"><b>implementation inheritance</b></a> - 
	see <a href="#Gprivate-base">private base</a>.

<p>
<a name="Gimplicit-type-conversion"><b>implicit type conversion</b></a> - 
	<a href="#Gconversion">conversion</a> applied implicitly based on an expected <a href="#Gtype">type</a> and the
	type of a value.
	See also, <a href="#Gexplicit-type-conversion">explicit type conversion</a>, <a href="#Guser-defined-type-conversion">user-defined type conversion</a>.
	TC++PL 11.3.3, 11.3.5, 11.4, C.6, D&E 2.6.2, 3.6.1, 3.6.3, 11.2.

<p>
<a name="Gin-class"><b>in-class</b></a> - 
	lexically within the <a href="#Gdeclaration">declaration</a> of a <a href="#Gclass">class</a>.
	TC++PL 10.2.9, 10.4.6.2.

<p>
<a name="Ginclude"><b>include</b></a> - 
	see <a href="#G#include">#include</a>.

<p>
<a name="Gincomplete-type"><b>incomplete type</b></a> - 
	<a href="#Gtype">type</a> that allows an <a href="#Gobject">object</a> to be copied, but not otherwise used.
	A <a href="#Gpointer">pointer</a> to an undeclared type is the typical example of an
	incomplete type.

<p>
<a name="Ginequality-operator"><b>inequality operator</b></a> - 
	see <a href="#G!=">!=</a>.

<p>
<a name="Ginformation-hiding"><b>information hiding</b></a> - 
	placing information where it can be accessed only through
	a well-defined <a href="#Ginterface">interface</a>.
	See also: <a href="#Gaccess-control">access control</a>, <a href="#Gabstract-class">abstract class</a>, <a href="#Gseparate-compilation">separate compilation</a>.
	TC++PL 2.4.

<p>
<a name="Ginheritance"><b>inheritance</b></a> - 
	a <a href="#Gderived-class">derived class</a> is said to inherit the <a href="#Gmember">member</a>s of its <a href="#Gbase-class">base class</a>es.
	TC++PL 2.6.2, 12.2, 23.4.3.1, D&E 3.5, 7.2, 12.

<p>
<a name="Ginitialization"><b>initialization</b></a> - 
	giving an <a href="#Gobject">object</a> an initial value. Initialization differs from
	assignment in that there is no previous value involved.
	Initialization is done by <a href="#Gconstructor">constructor</a>s.

<p>
<a name="Ginitializer-list"><b>initializer list</b></a> - 
	comma-separated list of <a href="#Gexpression">expression</a>s enclosed in curly braces, e.g.
	{ 1, 2, 3 } used to initialize a <a href="#Gstruct">struct</a> or an <a href="#Garray">array</a>.
	TC++PL 5.2.1, 5.7, 11.3.3.

<p>
<a name="Ginline-function"><b>inline function</b></a> - 
	<a href="#Gfunction">function</a> declared inline using the inline keyword or by being a
	<a href="#Gmember-function">member function</a> defined <a href="#Gin-class">in-class</a>.
	<a href="#Gcompiler">Compiler</a>s are encouraged to generate inline code rather than
	function calls for inline functions.
	Most benefits from <a href="#Ginlining">inlining</a> comes with very <a href="#Gshort">short</a> functions.
	TC++PL 7.1.1, 9.2, 10.2.9, D&E 2.4.1 .

<p>
<a name="Ginlining"><b>inlining</b></a> - 
	see <a href="#Ginline-function">inline function</a>.

<p>
<a name="Ginserter"><b>inserter</b></a> - 
	(1) an <a href="#Giostream">iostream</a> <a href="#G<<"><<</a> (put to) function.
	(2) an STL operation yielding an iterator to be used for adding
	elements to a containter.
	TC++PL 19.2.4, 21.2, D&E 8.3.1.
	See also: extracter, back_inserter, front_inserter.

<p>
<a name="Ginstantiation"><b>instantiation</b></a> - 
	see <a href="#Gtemplate-instantiation">template instantiation</a>.

<p>
<a name="Gint"><b>int</b></a> - 
	basic signed <a href="#Ginteger-type">integer type</a>;
	its precision is implementation-defined,
	but an int has at least 32 <a href="#Gbit">bit</a>s.
	TC++PL 2.3.1, 4.4.

<p>
<a name="Ginteger-type"><b>integer type</b></a> - 
	a <a href="#Gshort">short</a>, <a href="#Gint">int</a>, or long. <a href="#GStandard-C++">Standard C++</a> doesn't support long long.
	TC++PL 4.4.

<p>
<a name="Gintegral-type"><b>integral type</b></a> - 
	a <a href="#Gbool">bool</a>, <a href="#Gcharacter-type">character type</a>, or <a href="#Ginteger-type">integer type</a>.
	Supports arithmetic and logical operations.
	TC++PL 4.1.1.

<p>
<a name="Ginterface"><b>interface</b></a> - 
	a set of <a href="#Gdeclaration">declaration</a>s that defines how a part of a <a href="#Gprogram">program</a> can be
	accessed.
	The <a href="#Gpublic-member">public member</a>s and the <a href="#Gfriend">friend</a>s of a <a href="#Gclass">class</a> defines that class'
	interface for other code to use. A class without <a href="#Gdata-member">data member</a>s defines
	a pure interface.
	The <a href="#Gprotected-member">protected member</a>s provide an additional interface for use by
	members of <a href="#Gderived-class">derived class</a>es.
	See also: <a href="#Gabstract-class">abstract class</a>.

<p>
<a name="Ginterface-function"><b>interface function</b></a> - 
	A <a href="#Gfunction">function</a> that can access the representation of a <a href="#Gclass">class</a>.
	See also: <a href="#Gfriend">friend</a>, <a href="#Gmember-function">member function</a>, <a href="#Gderived-class">derived class</a>, <a href="#Gprotected">protected</a>.

<p>
<a name="Ginterface-inheritance"><b>interface inheritance</b></a> - 
	see <a href="#Gabstract-class">abstract class</a>, <a href="#Gpublic-base">public base</a>.

<p>
<a name="Ginvariant"><b>invariant</b></a> - 
	a condition of the representation of an <a href="#Gobject">object</a> (the object's state)
	that should hold each time an <a href="#Ginterface-function">interface function</a> is called;
	usually established by a <a href="#Gconstructor">constructor</a>
	TC++PL 24.3.7, E.3.5.

<p>
<a name="Giostream"><b>iostream</b></a> - 
	(1) <a href="#Gstandard-library">standard library</a> flexible, extensible, <a href="#Gtype">type</a>-safe input and output
	framework.
	(1) <a href="#Gstream">stream</a> that can be used for both input and output.
	See also: <a href="#Gfile-stream">file stream</a>, <a href="#Gstring-stream">string stream</a>.
	TC++PL 3.4, 3.6, 21, D&E 3.11.4.1, 8.3.1.

<p>
<a name="GISO"><b>ISO</b></a> - 
	the international standards organization. It defines and maintains
	the standards of the major non-proprietary <a href="#Gprogramming-language">programming language</a>s,
	notably <a href="#GC++">C++</a>.

<p>
<a name="GISO-C"><b>ISO C</b></a> - 
	see <a href="#GC">C</a>.

<p>
<a name="GISO-C++"><b>ISO C++</b></a> - 
	<a href="#GC++">C++</a>.

<p>
<a name="Gistream"><b>istream</b></a> - 
	input <a href="#Gstream">stream</a> <a href="#Gtype">type</a>.
	TC++PL 3.6, 21.3.

<p>
<a name="Gistringstream"><b>istringstream</b></a> - 
	a <a href="#Gstring-stream">string stream</a> for input.

<p>
<a name="Giteration"><b>iteration</b></a> - 
	traversal of data structure, directly or indirectly using
	an <a href="#Giteration-statement">iteration-statement</a>.
	See also: <a href="#Grecursion">recursion</a>.
	The <a href="#Gstandard-library">standard library</a> offer <a href="#Galgorithm">algorithm</a>s, such as <a href="#Gcopy()">copy()</a> and <a href="#Gfind()">find()</a>,
	that can be effective alternatives to <a href="#Gexplicit">explicit</a> iteration.
	TC++PL 6.3.3. 18.

<p>
<a name="Giteration-statement"><b>iteration-statement</b></a> - 
	for-<a href="#Gstatement">statement</a>, while-statement, or do-statement.

<p>
<a name="Giterator"><b>iterator</b></a> - 
	a <a href="#Gstandard-library">standard library</a> <a href="#Gabstraction">abstraction</a> for <a href="#Gobject">object</a>s referring to
	elements of a sequence.
	TC++PL 3.8.1, 19.2-3.

<p>
<a name="GK&R-C"><b>K&R C</b></a> - 
	<a href="#GC">C</a> as defined by <a href="#GKernighan">Kernighan</a> and <a href="#GRitchie">Ritchie</a>.

<p>
<a name="GKernighan"><b>Kernighan</b></a> - 
	Brian Kernighan is a co-author of Kernighan & <a href="#GRitchie">Ritchie</a>:
	"The <a href="#GC">C</a> <a href="#Gprogramming-language">programming Language</a>".

<p>
<a name="GKoenig-lookup"><b>Koenig lookup</b></a> - 
	see <a href="#Gargument-based-lookup">argument-based lookup</a>.

<p>
<a name="Glanguage-extension"><b>language extension</b></a> - 
	(1) relatively <a href="#Gnew">new</a> feature that people haven't yet gotten used to.
	(2) proposed new feature.
	(3) feature provided by one or more implementations, but not adopted
	by the standard; the use of some such features implies lock-in to a
	particular <a href="#Gcompiler">compiler</a> supplier.

<p>
<a name="Glearning-C++"><b>learning C++</b></a> - 
	focus on concepts and techniques.
	<a href="bs_faq.html#prerequisite">You don't need to learn C first</a>.
	See also "Learning <a href="#GStandard-C++">Standard C++</a> as a <a href="#Gnew">New</a> Language", available from
	<a href="papers.html">my papers page</a>.
	<a href="bs_faq.html#how-to-start">How do I start?</a>.
	TC++PL 1.2, 1.7, D&E 7.2.

<p>
<a name="Gline-comment"><b>line comment</b></a> - 
	<a href="#Gcomment">comment</a> started by // and terminated by end-of-line.
	TC++PL 6.4, D&E 3.11.1.

<p>
<a name="Glinkage"><b>linkage</b></a> - 
	the process of merging code from <a href="#Gseparately-compiled">separately compiled</a> <a href="#Gtranslation-unit">translation unit</a>s
	into a <a href="#Gprogram">program</a> or part of a program.
	TC++PL 9.

<p>
<a name="Glinker"><b>linker</b></a> - 
	the part of a <a href="#GC++">C++</a> implementation that merge the code generated from
	<a href="#Gseparately-compiled">separately compiled</a> <a href="#Gtranslation-unit">translation unit</a>s into a <a href="#Gprogram">program</a> or part of a
	program.
	TC++PL 9.1, D&E 4.5, 11.3.

<p>
<a name="GLiskov-Substitution-Principle"><b>Liskov Substitution Principle</b></a> - 
	<a href="#Gdesign">design</a> <a href="#Gclass">class</a>es so that any <a href="#Gderived-class">derived class</a> will be acceptable where
	its <a href="#Gbase-class">base class</a> is.
	<a href="#GC++">C++</a> <a href="#Gpublic-base">public base</a>s enforce that as far as the <a href="#Ginterface">interface</a> provided
	by the base class.
	TC++PL 24.3.4, D&E 2.10.

<p>
<a name="Glist#"><b>list</b></a> - 
	<a href="#Gstandard-library">standard library</a> linked <a href="#Gcontainer">container</a>.
	See also: <a href="#Gvector">vector</a>, <a href="#Gmap">map</a>. 
	TC++PL 3.7.3, 17.2.2.

<p>
<a name="Gliteral"><b>literal</b></a> - 
	notation for <a href="#Gbool">bool</a>, <a href="#Gcharacter-type">character type</a>s, <a href="#Ginteger-type">integer type</a>s, or
	<a href="#Gfloating-point-type">floating-point type</a>s.
	See also: <a href="#Genumerator">enumerator</a>s.
	TC++PL 4.2, 4.3.1, 4.4.1, 4.5.1, 5.2.2, D&E 11.2.1.

<p>
<a name="Glocal-class"><b>local class</b></a> - 
	<a href="#Gclass">class</a> defined within a <a href="#Gfunction">function</a>. Most often, the use of a local class
	is a sign that a function is too large. Beware that a local class
	cannot be a valid <a href="#Gtemplate">template</a> argument.

<p>
<a name="Glocal-function"><b>local function</b></a> - 
	<a href="#Gfunction">function</a> defined within a function. Not supported by <a href="#GC++">C++</a>. Most often,
	the use of a local function is a sign that a function is too large.

<p>
<a name="Glocale"><b>locale</b></a> - 
	<a href="#Gstandard-library">standard library</a> <a href="#Gclass">class</a> for representing culture dependencies relating
	to input and output, such as floating-point output formats,
	<a href="#Gcharacter-set">character set</a>s, and collating rules.
	A locale is a <a href="#Gcontainer">container</a> of <a href="#Gfacet">facet</a>s.
	TC++PL 21.1, D.

<p>
<a name="Glong-double"><b>long double</b></a> - 
	extended-precision floating-point number.
	TC++PL 4.5.

<p>
<a name="Glong-int"><b>long int</b></a> - 
	integer of a size greater than or equal to the size of an int.
	TC++PL 4.4.

<p>
<a name="GLSP"><b>LSP</b></a> - 
	see <a href="#GLiskov-Substitution-Principle">Liskov Substitution Principle</a>.

<p>
<a name="Glvalue"><b>lvalue</b></a> - 
	an <a href="#Gexpression">expression</a> that may appear on the left-hand side of an assignment;
	for example, v[7] if v is an <a href="#Garray">array</a> or a <a href="#Gvector">vector</a>.
	An lvalue is modifiable unless it is <a href="#Gconst">const</a>.
	TC++PL 4.9.6, D&E 3.7.1.

<p>
<a name="Gmacro"><b>macro</b></a> - 
	facility for character substitution; doesn't obey <a href="#GC++">C++</a> <a href="#Gscope">scope</a> or
	<a href="#Gtype">type</a> rules.
	C++ provides alternatives to most uses of macros;
	see <a href="#Gtemplate">template</a>, inline, <a href="#Gconst">const</a>, and <a href="#Gnamespace">namespace</a>.
	Don't use macros unless you absolutely have to.
	TC++PL 7.8, D&E 2.9.2, 4.4, 18.

<p>
<a name="Gmain()"><b>main()</b></a> - 
	the <a href="#Gfunction">function</a> called by the system to start a <a href="#GC++">C++</a> <a href="#Gprogram">program</a>.
	TC++PL 3.2, 6.1.7, 9.4 .

<p>
<a name="Gmaintenance"><b>maintenance</b></a> - 
	work on a <a href="#Gprogram">program</a> after its initial release. Typical maintenance
	activities <a href="#Ginclude">include</a> bug fixing, minor feature enhancements, porting to
	<a href="#Gnew">new</a> systems, improvements of <a href="#Gerror-handling">error handling</a>, modification to
	use different natural languages, improvements to documentation,
	and performance tuning.
	Maintenance typically consumes more than 80% of the total
	effort and cost expended on a program.

<p>
<a name="Gmalloc()"><b>malloc()</b></a> - 
	<a href="#GC">C</a> standard allocation <a href="#Gfunction">function</a>. Use <a href="#Gnew">new</a> or <a href="#Gvector">vector</a> instead.

<p>
<a name="Gmap"><b>map</b></a> - 
	<a href="#Gstandard-library">standard library</a> associative <a href="#Gcontainer">container</a>, based on "less than" ordering.
	See also: <a href="#Ghash_map">hash_map</a>, <a href="#Gvector">vector</a>, list.
	TC++PL 3.7.4, 17.4.1.

<p>
<a name="GMax-Munch"><b>Max Munch</b></a> - 
	(1) mythical participant in the <a href="#GC++-standards-process">C++ standards process</a>. (2) the
	rule that says that while parsing C++ always chooses the lexically
	or syntactically longest alternative. Thus ++ is the
	increment operation, not two additions, and <a href="#Glong-int">long int</a> is a single
	<a href="#Ginteger-type">integer type</a> rather than the long integer followed by an int.
	Cross <a href="#Greference">reference</a>s in this <a href="#Gglossary">glossary</a> follow this rule.

<p>
<a name="Gmember"><b>member</b></a> - 
	<a href="#Gtype">type</a>, <a href="#Gvariable">variable</a>, <a href="#Gconstant">constant</a>, or <a href="#Gfunction">function</a> declared in the <a href="#Gscope">scope</a> of a <a href="#Gclass">class</a>.
	TC++PL 5.7, 10.2, D&E 2.3, 2.5.2, 2.11.

<p>
<a name="Gmember-class"><b>member class</b></a> - 
	a <a href="#Gclass">class</a> that is a <a href="#Gmember">member</a> of another; also called a <a href="#Gnested-class">nested class</a>.
	TC++PL 11.12, D&E 3.12, 13.5.

<p>
<a name="Gmember-constant"><b>member constant</b></a> - 
	<a href="#Gconst">const</a> or <a href="#Genumeration">enumeration</a> declared as a <a href="#Gmember">member</a>. If initialized <a href="#Gin-class">in-class</a>,
	such a <a href="#Gconstant">constant</a> can be used in <a href="#Gconstant-expression">constant expression</a>s within the class.
	TC++PL 10.4.6.2.

<p>
<a name="Gmember-data"><b>member data</b></a> - 
	see <a href="#Gdata-member">data member</a>.

<p>
<a name="Gmember-function"><b>member function</b></a> - 
	a <a href="#Gfunction">function</a> declared in the <a href="#Gscope">scope</a> of a <a href="#Gclass">class</a>. A <a href="#Gmember">member</a> function that
	is not a <a href="#Gstatic-member-function">static member function</a> must be called for an <a href="#Gobject">object</a>
	of its class.
	TC++PL 10.2.1, D&E 2.3, 3.5.

<p>
<a name="Gmember-initializer"><b>member initializer</b></a> - 
	initializer for a <a href="#Gmember">member</a> specified in the <a href="#Gconstructor">constructor</a> for its <a href="#Gclass">class</a>.
	TC++PL 10.4.6, 12.2.2, D&E 12.9.

<p>
<a name="Gmember-type"><b>member type</b></a> - 
	<a href="#Gmember-class">member class</a>, member <a href="#Genumeration">enumeration</a>, or member <a href="#Gtypedef">typedef</a>.

<p>
<a name="Gmemberwise-copy"><b>memberwise copy</b></a> - 
	copying a <a href="#Gclass">class</a> <a href="#Gobject">object</a> by copying each of its <a href="#Gmember">member</a>s in turn, using
	proper <a href="#Gcopy-constructor">copy constructor</a>s or <a href="#Gcopy-assignment">copy assignment</a>s. That's the default
	meaning of copy.
	TC++PL 10.4.4.1, 10.4.6.3, D&E 11.4.4.

<p>
<a name="Gmemory"><b>memory</b></a> - 
	<a href="#Gstatic-memory">static memory</a>, <a href="#Gstack">stack</a>, or <a href="#Gfree-store">free store</a>.

<p>
<a name="Gmemory-management"><b>memory management</b></a> - 
	a way of allocating and freeing <a href="#Gmemory">memory</a>. In <a href="#GC++">C++</a> memory is either
	<a href="#Gstatic">static</a>, allocated on the <a href="#Gstack">stack</a>, or allocated on the <a href="#Gfree-store">free store</a>.
	When people talk about memory management, they usually think of
	free store or even specifically about <a href="#Ggarbage-collection">garbage collection</a>.
	Memory can often be effectively managed through
	<a href="#Gstandard-library">standard library</a> <a href="#Gcontainer">container</a>s, such as <a href="#Gvector">vector</a> or <a href="#Gstring">string</a>,
	or through general <a href="#Gresource-management">resource management</a> techniques.
	See also: <a href="#Gauto_ptr">auto_ptr</a>, <a href="#Gconstructor">constructor</a>, <a href="#Gdestructor">destructor</a>,
	<a href="#Gresource-acquisition-is-initialization">resource acquisition is initialization</a>.
	TC++PL C.9, D&E 3.9, 10.

<p>
<a name="Gmem_fun()"><b>mem_fun()</b></a> - 
	an <a href="#Gadapter">adapter</a> that allows a <a href="#Gmember-function">member function</a> to be used as an argument to
	a standard <a href="#Galgorithm">algorithm</a> requiring a <a href="#Gfree-standing-function">free-standing function</a>. 
	TC++PL 18.4.4.2.

<p>

<p>

	programming using <a href="#Gclass">class</a> hierarchies and <a href="#Gvirtual">virtual</a> <a href="#Gfunction">function</a>s to allow
	manipulation of <a href="#Gobject">object</a>s of a variety of <a href="#Gtype">type</a>s through well-defined
	<a href="#Ginterface">interface</a>s and allow a program to be extended incrementally through
	derivation.
	See also: <a href="#Gpolymorphism">polymorphism</a>, <a href="#Gdata-abstraction">data abstraction</a>.
	TC++PL 2.6, 12, D&E 3.5, 7.2.

<p>
<a name="Gobject-oriented-programming-language"><b>object-oriented programming language</b></a> - 
	a <a href="#Gprogramming-language">programming language</a> designed to support or enforce some notion
	of <a href="#Gobject-oriented-programming">object-oriented programming</a>.
	<a href="#GC++">C++</a> supports <a href="#GOOP">OOP</a> and other effective forms of programming, but does
	not try to enforce a single style of programming.
	See also: <a href="#Ggeneric-programming">generic programming</a>, <a href="#Gmulti-paradigm-programming">multi-paradigm programming</a>,
	<a href="#Ghybrid-language">hybrid language</a>.

	conventional notation for built-in operation, such as +, *, and &.

	See also: <a href="#Goperator-overloading">operator overloading</a>.

	<a href="#Gfunction">function</a> defining one of the standard <a href="#Goperator">operator</a>s; e.g. operator+().
	See also: operator, <a href="#Goperator-overloading">operator overloading</a>, <a href="#Gconversion-operator">conversion operator</a>.

<p>
<a name="Goperator-overloading"><b>operator overloading</b></a> - 
	having more than one <a href="#Goperator">operator</a> with the same <a href="#Gname">name</a> in the same <a href="#Gscope">scope</a>.
	Built-in operators, such as + and *, are overloaded for <a href="#Gtype">type</a>s such as
	<a href="#Gint">int</a> and <a href="#Gfloat">float</a>. Users can define their own additional meanings for
	<a href="#Guser-defined-type">user-defined type</a>s. It is not possible to define <a href="#Gnew">new</a> operators or
	to give new meanings to operators for <a href="#Gbuilt-in-type">built-in type</a>s.
	The <a href="#Gcompiler">compiler</a> picks the operator to be used based on argument types
	based <a href="#Goverload-resolution">overload resolution</a> rules.
	See also: overload resolution.
	TC++PL 6.2, D&E 3.6, 11.7.1.

<p>
<a name="Goptimizer"><b>optimizer</b></a> - 
	a part of a <a href="#Gcompiler">compiler</a> that eliminates redundant operations from
	code and adjusts code to perform better on a given computer.
	See also, <a href="#Gfront-end">front-end</a>, <a href="#Gback-end">back-end</a>, <a href="#Gcode-generator">code generator</a>.
	D&E 3.3.3.

<p>
<a name="Gor#"><b>or</b></a> -

<p>
<a name="Gorder-of-construction"><b>order of construction</b></a> - 
	a <a href="#Gclass">class</a> <a href="#Gobject">object</a> is constructed from the bottom up: first bases
	in <a href="#Gdeclaration">declaration</a> order, then <a href="#Gmember">member</a>s in declaration order, and
	<a href="#Gfinally">finally</a> the body of the <a href="#Gconstructor">constructor</a> itself.
	TC++PL 10.4.6, 12.2.2, 15.2.4.1, 15.4.3. D&E 2.11.1, 13.2.4.2.

<p>
<a name="Gorder-of-destruction"><b>order of destruction</b></a> -

	See also: <a href="#Gdestructor">destructor</a>.

<p>
<a name="Goverload-resolution"><b>overload resolution</b></a> - 
	a set of rules for selecting the best version of an <a href="#Goperator">operator</a> based
	on the <a href="#Gtype">type</a>s of its operands.
	A set of rules for selecting the best version of an overloaded
	<a href="#Gfunction">function</a> based on the types of its arguments.
	The intent of the overload resolution rules is to reject ambiguous
	uses and to select the simplest function or operator for each use.
	TC++PL 6.2, D&E 11.2.

<p>
<a name="Goverloaded-function"><b>overloaded function</b></a> - 
	see <a href="#Goverloading">overloading</a>.

<p>
<a name="Goverloaded-operator"><b>overloaded operator</b></a> - 
	see <a href="#Goperator-overloading">operator overloading</a>

<p>
<a name="Goverloading"><b>overloading</b></a> - 
	having more than one <a href="#Gfunction">function</a> with the same <a href="#Gname">name</a> in the same <a href="#Gscope">scope</a> or
	having more than one <a href="#Goperator">operator</a> with the same name in the same scope.
	It is not possible to <a href="#Goverload">overload</a> across different scopes.
	See also: <a href="#Gusing-declaration">using-declaration</a>.
	TC++PL 6.2, D&E 3.6, 11.2.

<p>
<a name="Goverride"><b>override</b></a> - 
	see <a href="#Goverriding">overriding</a>.

<p>
<a name="Goverriding"><b>overriding</b></a> - 
	declaring a <a href="#Gfunction">function</a> in a <a href="#Gderived-class">derived class</a> with the same <a href="#Gname">name</a> and a
	matching <a href="#Gtype">type</a> as a <a href="#Gvirtual">virtual</a> function in a <a href="#Gbase-class">base class</a>.
	The argument types must match exactly.
	The return types must match exactly or be co-variant.
	The overriding function will be invoked when the virtual function
	is called.
	TC++PL 15.6.2, 6.2, D&E 3.5.2-3, 13.7.

<p>
<a name="Gparadigm"><b>paradigm</b></a> - 
	pretentious and overused term for a way of thinking.
	Often used with the erroneous assumption that all "paradigms" are
	mutually exclusive, and often assuming that one paradigm is inherently
	superior to all others. Derived from Kuhn's theory of science.
	TC++PL 2.2.

<p>
<a name="Gpartial-specialization"><b>partial specialization</b></a> - 
	a <a href="#Gtemplate">template</a> used (only) for the subset of its template parameters
	that matches a <a href="#Gspecialization">specialization</a> pattern.
	TC++PL 13.5.

<p>
<a name="Gplacement-delete"><b>placement delete</b></a> -

<p>
<a name="Gplacement-new"><b>placement new</b></a> - 
	a version of the <a href="#Gnew">new</a> <a href="#Goperator">operator</a> where the user can add arguments to
	guide allocation. The simplest form, where the <a href="#Gobject">object</a> is placed
	in a specific location, is supported by the <a href="#Gstandard-library">standard library</a>.
	<a href="bs_faq2.html#placement-delete">Example</a>.
	For example, placement new is used in the implementation
	of standard library <a href="#Gcontainer">container</a>s.
	See also: <a href="#Gexplicit-call-of-destructor">explicit call of destructor</a>.
	TC++PL 10.4.11, E.3.1, D&E 10.4.

<p>
<a name="GPOD"><b>POD</b></a> - 
	"Plain Old Data" - (roughly) a <a href="#Gclass">class</a> that doesn't contain <a href="#Gdata-member">data member</a>s
	that would be illegal in <a href="#GC">C</a>. A POD can therefore be used for data that
	needs to be share with C <a href="#Gfunction">function</a>s.
	A POD can have non-<a href="#Gvirtual-member-function">virtual member function</a>s.

<p>
<a name="Gpointer"><b>pointer</b></a> - 
	an <a href="#Gobject">object</a> holding an <a href="#Gaddress">address</a> or 0.
	TC++PL 2.3.3, 5.1, D&E 9.2.2.1, 11.4.4.

<p>
<a name="Gpolicy-object"><b>policy object</b></a> -

	See also <a href="#Gtrait">trait</a>, <a href="#Gfacet">facet</a>.
	TC++PL 13.4, 24.4.1.

<p>
<a name="Gpolymorphism"><b>polymorphism</b></a> - 
	providing a single <a href="#Ginterface">interface</a> to entities of different <a href="#Gtype">type</a>s.
	<a href="#Gvirtual">virtual</a> <a href="#Gfunction">function</a>s provide dynamic (run-time) polymorphism through
	an interface provided by a <a href="#Gbase-class">base class</a>. <a href="#Goverloaded-function">Overloaded function</a>s and
	<a href="#Gtemplate">template</a>s provide <a href="#Gstatic">static</a> (compile-time) polymorphism.

<p>
<a name="Gpreprocessor"><b>preprocessor</b></a> - 
	the part of a <a href="#GC++">C++</a> implementation that removes <a href="#Gcomment">comment</a>s, performs
	<a href="#Gmacro">macro</a> substitution and <a href="#G#include">#include</a>s. Avoid using the preprocessor

	See also: macro, #include, inline, <a href="#Gconst">const</a>, <a href="#Gtemplate">template</a>, <a href="#Gnamespace">namespace</a>.

<p>

<p>
<a name="Gprivate"><b>private</b></a> -

<p>
<a name="Gprocedure"><b>procedure</b></a> -

<p>
<a name="Gprogram"><b>program</b></a> -

<p>
<a name="Gprogramming-language"><b>programming language</b></a> -

	in a way that lends itself to solving problems using computers.
	There do not appear to be a general consensus on what a programming

	TC++PL 1.3.2, 2.1-2, D&E page 7.

<p>
<a name="Gprohibiting-operations"><b>prohibiting operations</b></a> -

<p>

	base's <a href="#Gpublic-member">public member</a>s are accessible to the users of that derived
	class.
	TC++PL 15.3.2, D&E 2.3.

<p>
<a name="Gpublic-member"><b>public member</b></a> - 
	a <a href="#Gmember">member</a> accessible to all users of a <a href="#Gclass">class</a>.
	TC++PL 2.5.2, 10.2.2, 15.3, D&E 2.10.

<p>
<a name="Gpure-object-oriented-language"><b>pure object-oriented language</b></a> - 
	<a href="#Gprogramming-language">programming language</a> claiming to support only
	<a href="#Gobject-oriented-programming">object-oriented programming</a>.
	<a href="#GC++">C++</a> is designed to support several programming <a href="#Gparadigm">paradigm</a>s, including
	traditional C-style programming, <a href="#Gdata-abstraction">data abstraction</a>,
	object-oriented programming, and <a href="#Ggeneric-programming">generic programming</a>.
	For a longer explanation, read
	<a href="oopsla.pdf">Why C++ isn't just an object-oriented programming language</a>.
	See also: <a href="#Ghybrid-language">hybrid language</a>.

<p>
<a name="Gpure-virtual-function"><b>pure virtual function</b></a> - 
	<a href="#Gvirtual">virtual</a> <a href="#Gfunction">function</a> that must be overridden in a <a href="#Gderived-class">derived class</a>.
	Indicated by the curious <a href="#G=0">=0</a> syntax. A pure virtual function
	can be defined in the class where it is declared pure, but
	needn't be and usually isn't. A class with at least one
	pure virtual function is an <a href="#Gabstract-class">abstract class</a>.
	TC++PL 12.3. D&E 13.2.1.

<p>
<a name="Gpush_back()"><b>push_back()</b></a> - 
	<a href="#Gmember-function">member function</a> that adds an element at the end of a standard
	<a href="#Gcontainer">container</a>, such as <a href="#Gvector">vector</a>, thereby increasing the container's size

	If program termination is unacceptable, a user can provide an
	alternative terminate() <a href="#Gfunction">function</a>.
	If you are worried about <a href="#Guncaught-exception">uncaught exception</a>s, make the body of <a href="#Gmain()">main()</a>
	a <a href="#Gtry-block">try-block</a>.

<p>
<a name="Gtermination-semantics"><b>termination semantics</b></a> - 
	a somewhat ominous terminology for the idea that throwing an <a href="#Gexception">exception</a>
	"terminates" an operation and returns through the <a href="#Gfunction">function</a> call
	chain to a <a href="#Ghandler">handler</a>.
	The handler can initiate any <a href="#Gerror-handling">error handling</a> it likes, including calling
	the function that caused the exception again (presumably after fixing
	the problem that caused the problem).
	What a handler can't do is simply tell the thrower to just carry on;
	by the time the handler is invoked we have returned from the
	<a href="#Gblock">block</a>/function
	that threw and all blocks/functions that led to it from the handler's
	<a href="#Gtry-block">try-block</a>.
	See also: <a href="#Gresumption-semantics">resumption semantics</a>.
	TC++PL 14.4.5, D&E 16.6.

	to an appropriate <a href="#Gexception-handler">exception handler</a> identifyed by the <a href="#Gtype">type</a> of the
	exception throw.
	See also: <a href="#Gcatch">catch</a>, <a href="#Gexception-handling">exception handling</a>.
	TC++PL 8.3.1, 14.3, D&E 16.3.

<p>
<a name="Gtrait"><b>trait</b></a> - 
	a small <a href="#Gpolicy-object">policy object</a>, typically used to describe aspects of a <a href="#Gtype">type</a>.
	For example, <a href="#Giterator">iterator</a>_trait<T> specifies the types resulting from
	operations on an iterator T.
	TC++PL 19.2.2.

<p>
<a name="Gtranslation-unit"><b>translation unit</b></a> - 
	a part of a <a href="#Gprogram">program</a> that can be <a href="#Gseparately-compiled">separately compiled</a>.
	TC++PL 9.1.

<p>
<a name="Gtrigraph"><b>trigraph</b></a> - 
	alternative representation for <a href="#GC++">C++</a> representation characters that
	doesn't exist in every national <a href="#Gcharacter-set">character set</a>, such as {, }, [, ],
	and #: ??<, ??>, ??(, ??), and ??=.
	TC++PL C.3.1.

<p>
<a name="Gtrue"><b>true</b></a> - 
	<a href="#Gbool">bool</a> value; converts to 1.
	TC++PL 4.2, D&E 11.7.2.

<p>
<a name="Gtry#"><b>try</b></a> - 
	keyword used to start a <a href="#Gtry-block">try-block</a>.

<p>
<a name="Gtry-block"><b>try-block</b></a> - 
	a <a href="#Gblock">block</a>, prefixed by the keyword <a href="#Gtry#">try</a>, specifying <a href="#Ghandler">handler</a>s for
	<a href="#Gexception">exception</a>s.
	See also: <a href="#Gcatch">catch</a>, <a href="#Gexception-handling">exception handling</a>.
	TC++PL 8.3.1,14.3, D&E 16.3.

<p>
<a name="Gtype"><b>type</b></a> - 
	a <a href="#Gbuilt-in-type">built-in type</a> or a <a href="#Guser-defined-type">user-defined type</a>.
	A type defines the proper use of a <a href="#Gname">name</a> or an <a href="#Gexpression">expression</a>.
	TC++PL 2.3.1, 4.1.

<p>
<a name="Gtype-checking"><b>type checking</b></a> - 
	the process of checking that every <a href="#Gexpression">expression</a> is used according
	to its <a href="#Gtype">type</a>.
	the <a href="#Gcompiler">compiler</a> checks every expression based on the declared types of
	the <a href="#Gname">name</a>s involved.
	TC++PL 7.2-3, 24.2.3, D&E 2.3, 2.6, 3.10, 3.15, 9.2.2.1.

<p>
<a name="Gtype-conversion"><b>type conversion</b></a> - 
	producing a value of one <a href="#Gtype">type</a> from a value of another type.
	A type <a href="#Gconversion">conversion</a> can be an implicit conversion or
	an <a href="#Gexplicit">explicit</a> conversion.
	See also: <a href="#Guser-defined-type-conversion">user-defined type conversion</a>, <a href="#Gcast">cast</a>.
	TC++PL 6.2.7.

<p>
<a name="Gtype-safety"><b>type safety</b></a> - 
	the property that an <a href="#Gobject">object</a> can be accessed only according to its
	definition. <a href="#GC++">C++</a> approximates this ideal. A programmer can violate
	<a href="#Gtype">type</a> safety by explicitly using a <a href="#Gcast">cast</a>, by using an uninitialized
	<a href="#Gvariable">variable</a>, by
	using a <a href="#Gpointer">pointer</a> that doesn't point to an object, by accessing beyond
	the end of an <a href="#Garray">array</a>, and by misusing a <a href="#Gunion">union</a>. For low-level systems
	code, it can be necessary to violate type safety (e.g. to write out
	the <a href="#Gbyte">byte</a> representation of some objects), but generally type safety
	must be preserved for a program to be correct and maintainable.

<p>
<a name="Gtype-system"><b>type system</b></a> - 
	the set of rules for how <a href="#Gobject">object</a>s can be used according to their <a href="#Gtype">type</a>s.
	See also: <a href="#Gtype-checking">type checking</a>.

<p>
<a name="Gtypedef"><b>typedef</b></a> - 
	synonym for some <a href="#Gtype">type</a> declared using the keyword"typedef".

<p>
<a name="Gtypeid()"><b>typeid()</b></a> - 
	<a href="#Goperator">operator</a> returning basic <a href="#Gtype">type</a> information.
	TC++PL 15.4.4, D&E 14.2.5.

<p>
<a name="Gtype_info"><b>type_info</b></a> - 
	<a href="#Gclass">class</a> containing basic <a href="#Grun-time-type-information">run time type information</a>.
	TC++PL 15.4.4, D&E 14.2.5.1.

<p>
<a name="Guncaught-exception"><b>uncaught exception</b></a> - 
	<a href="#Gexception">Exception</a> for which no <a href="#Ghandler">handler</a> was found. Invokes <a href="#Gterminate()">terminate()</a>, which
	by default terminates the <a href="#Gprogram">program</a>.
	TC++PL 14.7.

<p>
<a name="Gundefined"><b>undefined</b></a> - 
	an aspect of <a href="#GC++">C++</a>'s semantics for which no reasonable behavior is
	required. An example is dereferencing a <a href="#Gpointer">pointer</a> with the value <a href="#Gzero">zero</a>.
	Avoid undefined behavior.
	TC++PL C.2.

<p>
<a name="Guninitialized-memory"><b>uninitialized memory</b></a> - 
	<a href="#Gmemory">memory</a> that hasn't been initialized to hold a specific value of a <a href="#Gtype">type</a>.
	TC++PL 19.4.4.

<p>
<a name="Gunion"><b>union</b></a> - 
	a <a href="#Gstruct">struct</a> with all <a href="#Gmember">member</a>s allocated at the same offset within an
	<a href="#Gobject">object</a>.
	The language does not guarantee <a href="#Gtype-safety">type safety</a> for all uses of unions.
	Primarily used to save space.
	TC++PL C.8.2.

<p>
<a name="Gupcast"><b>upcast</b></a> - 
	a <a href="#Gcast">cast</a> from a <a href="#Gderived-class">derived class</a> to one of its bases.
	See also: <a href="#Gdowncast">downcast</a>, <a href="#Gcrosscast">crosscast</a>.
	TC++PL 15.4.

<p>
<a name="Guser-defined-type"><b>user-defined type</b></a> - 
	<a href="#Gclass">Class</a> or <a href="#Genumeration">enumeration</a>.
	A programmer can define meanings for <a href="#Goperator">operator</a>s for user-defined <a href="#Gtype">type</a>s.
	See also: <a href="#Goperator-overloading">operator overloading</a>.
	TC++PL 6.2, 11, D&E 3.6, 11.7.1.

<p>
<a name="Guser-defined-type-conversion"><b>user-defined type conversion</b></a> - 
	a user can define <a href="#Gconversion">conversion</a>s either as <a href="#Gconstructor">constructor</a>s or 
	<a href="#Gconversion-operator">conversion operator</a>s.
	These conversions are applied explicitly or implicitly just like
	built-in conversions. 
	TC++PL 11.3.5, 11.4, D&E 3.6.1, 3.6.3.

<p>
<a name="Gusing#"><b>using</b></a> - 
	see <a href="#Gusing-directive">using-directive</a> and <a href="#Gusing-declaration">using-declaration</a>.

<p>
<a name="Gusing-declaration"><b>using-declaration</b></a> - 
	<a href="#Gdeclaration">declaration</a> of a local synonym for a <a href="#Gname">name</a> in another <a href="#Gnamespace">namespace</a>
	or <a href="#Gclass">class</a>.
	<a href="http://www.research.att.com/~bs/bs_faq2.html#overloadderived">
	Example of using-declaration used to simplify overloading</a>.
	See also: overloading, <a href="#Gargument-based-lookup">argument-based lookup</a>.
	TC++PL 8.2.2. D&E 17.4.

<p>
<a name="Gusing-directive"><b>using-directive</b></a> - 
	directive making a <a href="#Gnamespace">namespace</a> accessible.
	See also: <a href="#Gargument-based-lookup">argument-based lookup</a>.
	TC++PL 8.2.3. D&E 17.4.

<p>
<a name="Gvalarray"><b>valarray</b></a> - 
	<a href="#Gstandard-library">standard library</a> numeric <a href="#Gvector">vector</a> <a href="#Gtype">type</a> supporting vector operations.
	TC++PL 22.4.

<p>
<a name="Gvalue-return"><b>value return</b></a> - 
	The semantics of <a href="#Gfunction">function</a> return is to pass a copy of the return
	value. The copy operation is defined by the return <a href="#Gtype">type</a>'s
	<a href="#Gcopy-constructor">copy constructor</a>.
	TC++PL 7.4.

<p>
<a name="Gvariable"><b>variable</b></a> - 
	named <a href="#Gobject">object</a> in a <a href="#Gscope">scope</a>.
	TC++PL 2.3.1, 10.4.3, D&E 2.3.

<p>
<a name="Gvariable-definition"><b>variable definition</b></a> - 
	<a href="#Gdeclaration">declaration</a> of a named <a href="#Gobject">object</a> of a data <a href="#Gtype">type</a> without an
	extern specifier.

<p>
<a name="Gvector"><b>vector</b></a> - 
	<a href="#Gstandard-library">standard library</a> <a href="#Gtemplate">template</a> providing contiguous storage, re-sizing
	and the useful <a href="#Gpush_back()">push_back()</a> <a href="#Gfunction">function</a>s for adding elements at the end.
	Vector is the default <a href="#Gcontainer">container</a>.
	See also: <a href="#Gmap">map</a>, <a href="#Gmultimap">multimap</a>, list, <a href="#Gdeque">deque</a>.
	TC++PL 3.7.1, 16.3.

<p>
<a name="Gvirtual"><b>virtual</b></a> - 
	keyword used to declare a <a href="#Gmember-function">member function</a> virtual.

<p>
<a name="Gvirtual-base"><b>virtual base</b></a> - 
	a base that is shared by all <a href="#Gclass">class</a>es in a <a href="#Gclass-hierarchy">class hierarchy</a> that has
	declared it <a href="#Gvirtual">virtual</a>.
	TC++PL 15.2.4, D&E 12.3, 12.4.1.

<p>
<a name="Gvirtual-constructor"><b>virtual constructor</b></a> - 
	a <a href="#Gconstructor">constructor</a> cannot be <a href="#Gvirtual">virtual</a>, because to create an <a href="#Gobject">object</a>, we
	need complete information of its <a href="#Gtype">type</a>. "virtual constructor" is the
	<a href="#Gname">name</a> of a technique for calling a virtual <a href="#Gfunction">function</a> to create an
	object of an appropriate type.
	<a href="http://www.research.att.com/~bs/bs_faq2.html#virtual-ctor">Example</a>.
	TC++PL 12.4.4, 15.6.2.

<p>
<a name="Gvirtual-destructor"><b>virtual destructor</b></a> - 
	a <a href="#Gdestructor">destructor</a> declared <a href="#Gvirtual">virtual</a> to ensure that the proper
	<a href="#Gderived-class">derived class</a> destructor
	is called if an <a href="#Gobject">object</a> of a derived class is deleted through a
	<a href="#Gpointer">pointer</a> to a <a href="#Gbase-class">base class</a>. If a class has any virtual <a href="#Gfunction">function</a>s,
	it should have a virtual destructor.
	<a href="bs_faq.html#virtual-dtor">Example</a>.
	TC++PL 12.4.2, D&E 10.5.

<p>
<a name="Gvirtual-member-function"><b>virtual member function</b></a> - 
	a <a href="#Gmember-function">member function</a> that a <a href="#Gderived-class">derived class</a> can <a href="#Goverride">override</a>;
	the primary mechanism for run-time <a href="#Gpolymorphism">polymorphism</a> in <a href="#GC++">C++</a>.
	A <a href="#Gvirtual">virtual</a> member function is sometimes called a <a href="#Gmethod">method</a>.
	See also: <a href="#Goverriding">overriding</a>, <a href="#Gpure-virtual-function">pure virtual function</a>.
	TC++PL 2.5.4, 2.5.5, 12.2.6, D&E 3.5, 12.4.

<p>
<a name="Gvirtual-function-pointer"><b>virtual-function pointer</b></a> - 
	a <a href="#Gpointer">pointer</a> to a <a href="#Gclass">class</a>' <a href="#Gvirtual">virtual</a> <a href="#Gfunction">function</a> table.

<p>
<a name="Gvirtual-function-table"><b>virtual-function table</b></a> - 
	table of all <a href="#Gvirtual">virtual</a> <a href="#Gfunction">function</a>s for a <a href="#Gclass">class</a>. The most common way
	of implementing virtual functions is to have each <a href="#Gobject">object</a> of a
	class with virtual functions contain a virtual function <a href="#Gpointer">pointer</a>
	pointing to the class' virtual function table.

<p>
<a name="GVisual-C++"><b>Visual C++</b></a> - 
	Microsoft's implementation of <a href="#GC++">C++</a> together with proprietary libraries
	for Windows programming in an IDE.

<p>
<a name="Gvoid"><b>void</b></a> - 
	a keyword used to indicate an absence of information.
	TC++PL 4.1.1, 4.7.

<p>
<a name="Gvoid*"><b>void*</b></a> - 
	<a href="#Gpointer">pointer</a> to <a href="#Gvoid">void</a>; that is, a pointer to an <a href="#Gobject">object</a> of unknown <a href="#Gtype">type</a>;
	also called pointer to <a href="#Graw-memory">raw memory</a>.
	A void* cannot be used or assigned without a <a href="#Gcast">cast</a>.
	TC++PL 5.6, D&E 11.2.1, 11.2.3.

<p>
<a name="Gvolatile"><b>volatile</b></a> - 
	attribute of a <a href="#Gdeclaration">declaration</a> telling the <a href="#Gcompiler">compiler</a> that an entity
	can have its value changed by extralinguistic means; for example,
	a real time clock: "extern volatile <a href="#Gconst">const</a> long clock;".
	Limits optimizations.
	TC++PL A.7.1.

<p>
<a name="Gvptr"><b>vptr</b></a> - 
	see <a href="#Gvirtual-function-pointer">virtual-function pointer</a>.

<p>
<a name="Gvtbl"><b>vtbl</b></a> - 
	see <a href="#Gvirtual-function-table">virtual-function table</a>.

<p>
<a name="Gwchar_t"><b>wchar_t</b></a> - 
	wide <a href="#Gcharacter-type">character type</a>. Used to hold characters of <a href="#Gcharacter-set">character set</a>s that
	require more than a <a href="#Gbyte">byte</a> to represent, such as unicode.
	TC++PL 4.3, C.3.3.
	See also: large character sets, universal character <a href="#Gname">name</a>.

<p>
<a name="GWG21"><b>WG21</b></a> - 
	a common abbreviation of the <a href="#Gname">name</a> of the <a href="#GISO-C++">ISO C++</a> standards committee.

<p>
<a name="Gword#"><b>word</b></a> - 
	a number of <a href="#Gbyte">byte</a>s that on a given machine is particularly
	suied to holding an integers or a <a href="#Gpointer">pointer</a>. On many machines, an <a href="#Gobject">object</a>
	must be aligned on a word boundary for acceptable performance.
	An int is typically a stored in a word.
	Often, a word is 4 bytes. 
	See also: <a href="#Galignment">alignment</a>.
	TC++PL 4.6.

<p>
<a name="Gxor#"><b>xor</b></a> - 
	synonym for ^, the bitwise exclusive or <a href="#Goperator">operator</a>
	TC++PL C.3.1.

<p>
<a name="Gzero"><b>zero</b></a> - 
	see <a href="#GNULL">NULL</a>
