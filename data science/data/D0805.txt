	<font class=articletitle>Upload / Download and database
</font>
	
	<br><br>
	
	<font class="articlebyline"></font>
	
	<br><br>
	
	<i>JSPSMART.COM</i><br><br>
	
	
	
	
	<font class="content">The upload and download of files are very useful
  processes in the world of the web, and especially for those who want to update a remote
  site in a very simple way. The uploaded or downloaded files are often stored physically on
  a disk but they can also be stored in a database. This article describes the upload and
  the download mechanisms, and the additional concepts when those are carried out in a
  database. </em>
  <strong>
</blockquote><BR><BR>UPLOAD</strong><BR><BR><a name="Upload"></a>When sending a file from a browser
by using a HTML form to a Web server: <BR><BR><ul>
  <li>All browsers do not allow to include a file in a HTML form</li>
  <li>The execution of the form returns rough data that needs to
    beprocessed</li>
</ul>
<b><BR><BR>The HTML Form</b><br>
The upload of a file from the client to Web server in a Web
page requires that this one contains a special form whose characteristics will be evoked
below. To deal with this type of form, the browser which will be used to display the page
will have to be compatible with the <a href=" http://www.jspsmart.com/liblocal/docs/en/articles/RFC1867.txt">RFC1867</a> (Netscape 3+ and
Microsoft IE3.02+). The RFC1867 is a document that describes the HTML specifications of
this kind of form used to upload files.       <BR><BR>The form must have the following characteristics :<BR><BR><ul>
  <li>The method is POST</li>
  <li>The action to be carried out is the script being used to
    process the returned data</li>
  <li>The form must specify a precise MIME type : <strong>multipart/form-data
      </strong></li>
  <li>The form must contain <em>at least</em> one element <strong><INPUT
    TYPE="file"></strong>. A compatible browser RFC1867 will interpret this tag
    by displaying a text field and a "Browse" button allowing the user to select the
    file to upload. </li>
</ul><BR><BR>Here is an example of HTML code defining a form of this
type:   <font face="Verdana">    <BR><BR><table border="0" width="99%" bgcolor="#EFEFEF" height="94">
  <tr>
    <td width="100%" height="88"><font face="Courier" size="2"><FORM METHOD="<strong>POST</strong>"
    ACTION="/jspsmartupload/upload.jsp" ENCTYPE="<strong>multipart/form-data</strong>"><br>
         Your name          :
    <INPUT TYPE="TEXT" NAME="Nom" SIZE="50">
                  <br>
         1st file to upload : <<strong>INPUT TYPE="FILE"</strong>
    NAME="Fichier1" SIZE="50"><br>
         2nd file to upload : <<strong>INPUT TYPE="FILE"</strong>
    NAME="Fichier2" SIZE="50"><br>
                             
    <INPUT TYPE="SUBMIT" VALUE="Upload"><br>
    </FORM></td>
  </tr>
</table><BR><BR><b>Processing of date returned by the
HTML Form</b><br>
When the form is submitted, the client's browser will
generate the data to be sent which will look like this : <BR><BR><table border="0" width="100%" bgcolor="#EFEFEF" height="184">
  <tr>
    <td width="100%" height="178"><font face="Courier" size="2"><strong>Content-type:
    multipart/form-data, boundary=AaB03x</strong><br>
    <font color="#068E0A">--AaB03x<br>
    content-disposition: form-data; name="Nom" <br>
    John Doe<br>
    <font color="#068E0A">--AaB03x<br>
    content-disposition: form-data; name="file1"; filename="file.txt"<br>
    Content-Type: text/plain<br>
    ... file.txt file contents ...<br>
    <font color="#068E0A">--AaB03x<br>
    content-disposition: form-data; name="file2"; filename="image.gif"<br>
    Content-type: image/gif <br>
    Content-Transfer-Encoding: binary ...<br>
    ..image.gif file contents .. <br>
    <font color="#068E0A">--AaB03x--</td>
  </tr>
</table><BR><BR>Each element of the form is returned with the following
information : <BR><BR><ul>
  <li>The name of the element ("NAME" attribute). If
    it is about a <TYPE="File INPUT">, the information
    "filename=file_name" with the name of the specified file is added,</li>
  <li>Some headers (MIME type, etc.). Ex : Content-type :
    image/gif,</li>
  <li>Element contents as a continuation of bytes,</li>
  <li>A separator or "boundary" such as
    "--AaB03x--". </li>
</ul><BR><BR>Next, these data must be parsed in order to recover
desired data and in particular the contents of the file we wish to save on the server. It
is the script specified in the ACTION attribute of the form which deals with this parsing
: it recovers the data and next, after processing if necessary, rewrites them in a new
file on the server : the upload is performed.       <font
face="Verdana"> <BR><BR><b>Example with jspSmartUpload</b><br>
To perform these operations, the script can call upon a
component such as jspSmartUpload. The importation of classes in JSP is carried out with
the <em><%@ import="ma.classe " %></em> directive and the instantiation of
the JavaBean which we will use with the name <em>myBean</em> by <em><jsp:useBean
id="myBean " class="my.class " /></em>.       <BR><BR>
The following example shows the ease of use of such a
component. After instantiation, we initialize next the jspSmartUpload JavaBean with the
execution data of the JSP contained in the implicit object <em>pageContext</em>. The
upload method proceeds with a parsing of data received by the server. Then, we can save
these data in a file, for example with the <em>save</em> method. For more information on
the options of this method or the other methods of jspSmartUpload, please refer to the <a
href="http://www.jspsmart.com">jspSmart</a> site. <BR><BR><table border="0" width="100%" bgcolor="#EFEFEF" height="152">
  <tr>
    <td width="100%" height="146"><font face="Courier" size="2" color="#999999">// Importation
    of the JavaBean class<font face="Courier" size="2"><br>
        <%@ page language="java"
    import="com.jspsmart.upload.*"%><br>
    <font face="Courier" size="2" color="#999999">// Bean instantiation<font
    face="Courier" size="2"><br>
        <jsp:useBean id="mySmartUpload" scope="page"
    class="com.jspsmart.upload.SmartUpload" /><br>
    <%<br>
    <font face="Courier" size="2" color="#999999">// Initialization with the
    pageContext implicit object<font face="Courier" size="2"><br>
        mySmartUpload.initialize(pageContext);<br>
    <font color="#999999">// Upload<br>
        mySmartUpload.<strong>upload</strong>();<br>
    <font face="Courier" size="2" color="#999999">// Save the files with their original
    names in a virtual directory of the web server <font face="Courier" size="2"> <br>
       mySmartUpload.<strong>save</strong>("/jspsmartupload/upload");<br>
    %> </td>
  </tr>
</table><BR><BR><p align="center"><img src=" http://www.jspsmart.com/liblocal/docs/en/articles/upload_en.jpg " width="617" height="235"> <BR><BR><hr noshade size="1"><BR><BR><strong>DOWNLOAD<a name="Download"></a></strong><br>
How to download a file from the server to the client with
the following possibilities : <BR><BR><ul>
  <li>Personalize data you want to send to the client.</li>
  <li>Force the browser to open a "Save as" dialog box
    systematically.</li>
  <li>Propose a different file name to the downloaded file.</li>
</ul><BR><BR>The tag does not allow to process such
operations. In particular, when the browser recognizes MIME type of the downloaded file,
the call of this tag will not force the browser to open the save dialog box but only
display the file according to the associated application.<font face="Verdana"><br>
<br>
<b>Data sent to the client</b><br>
Like for the upload, you can use a script to proceed with
the download of a file. This script reads data of the file you want to download and
returns the desired data to the client.    
      <BR><BR>To do that, it must send some headers which it will have
generated and then it must send the file contents. For example the <strong>Content-Disposition</strong>
header (described in details in the <a href=" http://www.jspsmart.com/liblocal/docs/en/articles/RFC2183.txt ">RFC2183</a>) specifies to the
client browser how to process data.<BR><BR>On the contrary to the HTML tag , the use
of a JSP script, a servlet or a JavaBean allows you to customize your own headers, this
way : <BR><BR><ul>
  <li>The client's browser displays the downloaded file or open the SaveAs dialog box</li>
  <li>Change the name of the downloaded file</li>
  <li>Change the MIME type of the downloaded file.</li>
</ul><BR><BR>
Notice that the sending of these headers implies that no
other headers have been already sent, that is to say no HTML code must be sent before a
download.  <BR><BR><b>The Save as dialog box</b><br>
The opening of such a box is carried out when the client
receives a Content-Disposition header with 'attachment' value. If you want to proceed that
way for any kind of files, you need to use a script, servlet or javabean which generates
this type of header.<BR><BR>In fact, Content-Disposition could be set with two
values: <strong>'inline'</strong> and <strong>'attachment'</strong>. The 'inline' value
indicates that the file should be immediately displayed by the browser (if there is an
associated application), whereas 'attachment' value forces the browser to open the dialog
box so that the user could save it on its disk.<BR><BR>Here is an example of data sent in order to directly
display a JPEG image (<strong>inline</strong>) :  <font face="Verdana">      <BR><BR><table border="0" width="100%" bgcolor="#EFEFEF">
  <tr>
    <td width="100%"><font face="Courier" size="2">Content-Type: image/jpeg <br>
    Content-Disposition: <strong>inline</strong> <br>
    Content-Description: example image with inline<br>
    <jpeg data> </td>
  </tr>
</table><BR><BR>Now, here is an example of data sent in order to apply
the <strong>attachment</strong> method to a JPEG image : <BR><BR><table border="0" width="100%" bgcolor="#EFEFEF">
  <tr>
    <td width="100%"><font face="Courier" size="2">Content-Type: image/jpeg <br>
    Content-Disposition: <strong>attachment</strong> <br>
    Content-Description: example image with attachment<br>
    <jpeg data> </td>
  </tr>
</table><BR><BR><b>Name of the downloaded file</b><br>
The Content-Disposition header provides a parameter which
can be used to suggest to the browser a file name for the file which will be saved on the
client disk ; This is the one that will appears by default in the dialog box. '<strong>filename</strong>:'
is the name of such a parameter.   <BR><BR>If you want to force the browser to display by default
the name 'example.jpg' to the downloaded file, the script will have to generate the
following headers :  <font face="Verdana">      
       <BR><BR><table border="0" width="100%" bgcolor="#EFEFEF">
  <tr>
    <td width="100%"><font face="Courier" size="2">Content-Type: image/jpeg <br>
    Content-Disposition: <strong>attachment</strong>; <strong>filename</strong>=example.jpg <br>
    Content-Description: example image with attachment<br>
    <jpeg data> </td>
  </tr>
</table><BR><BR><b>Example with jspSmartUpload</b><br>
In order to perform these operations, the download script
can use a component such as jspSmartUpload. The download is simply done by calling the
downloadFile method with the name of the file as parameter, followed by possible options
specifying which headers must be generated : <BR><BR><table border="0" width="100%" bgcolor="#EFEFEF">
  <tr>
    <td width="100%"><font face="Courier" size="2" color="#888888">// Importation of the
    JavaBean class<font face="Courier" size="2"><br>
    <%@ page language="java" import="com.jspsmart.upload.*"%><br>
    <font face="Courier" size="2" color="#888888">// Bean instantiation <font
    face="Courier" size="2"><br>
    <jsp:useBean id="mySmartUpload" scope="page"
    class="com.jspsmart.upload.SmartUpload" /><br>
    <%<br>
    <font color="#888888">// <font face="Courier" size="2" color="#888888">Initialization
    with the pageContext implicit object<font face="Courier" size="2"><br>
        mySmartUpload.initialize(pageContext);<br>
    <font face="Courier" size="2" color="#888888">// File download<font
    face="Courier" size="2"><br>
        mySmartUpload.<strong>downloadFile</strong>("c:	empsample.zip")<br>
    <font face="Courier" size="2" color="#888888">// with options : file type and
    proposition of name for the downloaded file <font face="Courier" size="2"><br>
        mySmartUpload.<strong>downloadFile</strong>("/upload/sample.zip","application/x-zip-compressed","downloaded.zip")<br>
    %></td>
  </tr>
</table><BR><BR><hr noshade size="1"><BR><BR><b>UPLOAD AND DOWNLOAD IN DATABASE</b><BR><BR>This part presents the transfer of files into and from a
database when:  <BR><BR><ul>
  <li>Files to upload or download are often large.</li>
  <li>Retreive data from a database requires a connection.<strong>
     </strong></li>
</ul><BR><BR><b>The BLOB type</b><br>
The size of the data often exceeds several hundreds of
kilobytes. It is necessary for the database to have a particular type to store such
information. Most of databases (MySQL or SQL Server for example) provide the <strong>BLOB</strong>
type <strong>(Binary Large OBjects)</strong> which can store up to several Go of data. <BR><BR>      The data sent during
an upload or a download are binaries. The use of a database providing the BLOB type allows
to directly write the file in the field of the specified table. Type BLOB is a binary
type, so that no conversion is necessary.    <BR><BR><b>Connection to the database</b><br>
In order to proceed with the operations of writing and
reading files in a database, the upload or download script will have first to create a
connection with your Database Management System. For example, you could use the JDBC
driver of your Database to retreive data: <BR><BR><table border="0" width="100%" bgcolor="#EFEFEF">
  <tr>
    <td width="100%"><font face="Courier" size="2" color="#888888">// Connection to the
    "test" MySQL database thanks to a jdbc driver<font face="Courier"
    size="2">  <br>
       Class.forName("org.gjt.mm.mysql.Driver").newInstance();<br>
       Connection con = DriverManager.<strong>getConnection</strong>("jdbc:mysql:///test");<br>
    <font face="Courier" size="2" color="#888888">// SQL request<font
    face="Courier" size="2"><br>
       Statement stmt = con.<strong>createStatement</strong>();<br>
       ResultSet rs = stmt.<strong>executeQuery</strong>("SELECT * FROM
    FILES");</td>
  </tr>
</table><BR><BR><b>Upload and download in a database</b><br>
The techniques of upload and download are the same as those
presented in the previous paragraphes. The only difference : <BR><BR><ul>
  <li>for the upload, the data returned by the web server are
    written in the database rather than a file,</li>
  <li>for the download, the file you want to download is
    directly read bytes by bytes from the database and sent to the client.  </li>
</ul>
</font>
