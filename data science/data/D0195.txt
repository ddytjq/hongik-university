I never expected Sun to create JVM's for all platforms but they should work with other vendors so that JVM betas and final releases occur at roughly the same time.  Otherwise, it puts them at a disadvantage.<BR><BR>I used to program in Java on the mac and I watch the current status closely.  I also used PPC Linux on a mac b/c the blackdown ports were coming faster than Apple's.  The situation has not changed so I'm still on a PC with windows and Linux.<BR><BR>I like Apple's approach to integrating Java with OSX but the truth is, I won't use it any time soon.  My work needs to run on Solaris, Linux, and Windows and therefore can't leverage native OS features.  If I were to target OSX specifically then I would use Object C which is the preferred language and will be the fastest solution.<BR><BR>I have seen the FREE development tools on OSX and I'm not very interested in them for Java development.  Almost all of my tools for Java are FREE anyway and written in Java so I'd use them over Apple's tools.<BR><BR>I too would spend the money on a nice Apple machine...definitely not the new iMac with its small screen... but a dual G4 with the latest display... but for Java development?  No way.  Not b/c of the cost.  That's not really the issue.  It's b/c Apple isn't keeping pace and never has in terms of Java support.<BR><BR>I wish them the best though.<BR>
