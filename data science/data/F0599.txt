<center><h2>Learn to Program with Visual Basic 6</h1></center>

<FONT FACE="Arial" SIZE=5><P ALIGN="CENTER">Chapter 4</P></font><p>

<B><FONT FACE="Arial" SIZE=3><P><A NAME="_Toc412024744"><A NAME="_Toc413993241">Exercise</P>
<P>Saving the China Shop Project</A></A></P></DIR>

<OL>

</B></FONT><FONT FACE="Palatino" SIZE=2><LI>The easiest way to save your project and form is by clicking on the </FONT><FONT FACE="Helvetica" SIZE=2>Save</FONT><FONT FACE="Palatino" SIZE=2> button on the Toolbar.</LI>
<LI>The first time you save a </FONT><B><FONT FACE="Times" SIZE=3>new</B></FONT><FONT FACE="Palatino" SIZE=2> project the following dialog box will appear:</LI>
</FONT>
<TABLE CELLSPACING=0 BORDER=0 CELLPADDING=7>
<TR><TD VALIGN="TOP"><DIR>

<FONT FACE="Palatino" SIZE=2><P ALIGN="CENTER"><IMG SRC="images/Image35.gif" WIDTH=385 HEIGHT=263></DIR>
</FONT></TD>
</TR>
</TABLE>

<FONT FACE="Palatino" SIZE=2>
<P>&nbsp;</P>
<P>Notice that the dialog box is prompting you to save the form file first. </FONT><B><FONT FACE="Times" SIZE=3>This is not a mistake</B></FONT><FONT FACE="Palatino" SIZE=2>. Visual Basic will first save the form or forms in your project, before the project itself. Also notice that the dialog box is asking you for a disk file name, and the location where it will be saved on your hard disk. The disk file name is located in the text box labeled </FONT><FONT FACE="Helvetica" SIZE=2>File <U>n</U>ame</FONT><FONT FACE="Palatino" SIZE=2>, and the location appears in the text box labeled </FONT><FONT FACE="Helvetica" SIZE=2>Save <U>i</U>n</FONT><FONT FACE="Palatino" SIZE=2>.</P>
<P>By default, Visual Basic will save your form with the same name as the form's </FONT><FONT FACE="Helvetica" SIZE=2>Name</FONT><FONT FACE="Palatino" SIZE=2> property, and save it in the directory where Visual Basic is installed on your PC. We want to name the external form file </FONT><B><FONT FACE="Courier New" SIZE=1>Main</B></FONT><FONT FACE="Palatino" SIZE=2> and change the location where the form will be saved to the </FONT><B><FONT FACE="Courier New" SIZE=1>\VBFiles\China</B></FONT><FONT FACE="Palatino" SIZE=2> subdirectory.</P>
<LI>Change </FONT><FONT FACE="Helvetica" SIZE=2>frmMain</FONT><FONT FACE="Palatino" SIZE=2> to </FONT><FONT FACE="Helvetica" SIZE=2>Main</FONT><FONT FACE="Palatino" SIZE=2> in the </FONT><FONT FACE="Helvetica" SIZE=2>File <U>n</U>ame</FONT><FONT FACE="Palatino" SIZE=2> Text Box and change the directory to find our file repository directory: </LI>
</FONT>
<TABLE CELLSPACING=0 BORDER=0 CELLPADDING=7>
<TR><TD VALIGN="TOP"><DIR>

<FONT FACE="Palatino" SIZE=2><P ALIGN="CENTER"><IMG SRC="images/Image36.gif" WIDTH=424 HEIGHT=292></DIR>
</FONT></TD>
</TR>
</TABLE>

<FONT FACE="Palatino" SIZE=2><P>&nbsp;</P>
<LI>Click on the </FONT><U><FONT FACE="Helvetica" SIZE=2>S</U>ave</FONT><FONT FACE="Palatino" SIZE=2> button to save the form in that directory. There is no need to change anything in the </FONT><FONT FACE="Helvetica" SIZE=2>Save as type</FONT><FONT FACE="Palatino" SIZE=2> list box. By default, the form file will be saved with a file extension of </FONT><B><FONT FACE="Courier New" SIZE=1>.frm</B></FONT><FONT FACE="Palatino" SIZE=2>.</LI>
<LI>When you have saved the form, Visual Basic will display another dialog box, prompting you for a project name and location. Notice how Visual Basic presumes that you want to save your project in the same location as your form.</LI>
<LI></LI>
<LI>A common mistake that beginners make here is to click on the </FONT><U><FONT FACE="Helvetica" SIZE=2>S</U>ave</FONT><FONT FACE="Palatino" SIZE=2> button now, resulting in the project file being saved as </FONT><FONT FACE="Helvetica" SIZE=2>Project1</FONT><FONT FACE="Palatino" SIZE=2>. </FONT><B><FONT FACE="Times" SIZE=3>Don’t click on the Save button yet!</FONT><FONT FACE="Palatino" SIZE=2> </B>You need to change the name of the project from </FONT><FONT FACE="Helvetica" SIZE=2>Project1</FONT><FONT FACE="Palatino" SIZE=2> to </FONT><FONT FACE="Helvetica" SIZE=2>China</FONT><FONT FACE="Palatino" SIZE=2>. Do that now, by typing the replacement into the </FONT><FONT FACE="Helvetica" SIZE=2>File <U>n</U>ame</FONT><FONT FACE="Palatino" SIZE=2> text box. As was the case when you saved the form, don't change anything in the </FONT><FONT FACE="Helvetica" SIZE=2>Save as <U>t</U>ype</FONT><FONT FACE="Palatino" SIZE=2> list box. By default, the project file will be saved with a file extension of </FONT><B><FONT FACE="Courier New" SIZE=1>.vbp</B></FONT><FONT FACE="Palatino" SIZE=2>:</LI>
</FONT>
<TABLE CELLSPACING=0 BORDER=0 CELLPADDING=7>
<TR><TD VALIGN="TOP"><DIR>

<FONT FACE="Palatino" SIZE=2><P ALIGN="CENTER"><IMG SRC="images/Image37.gif" WIDTH=426 HEIGHT=292></DIR>
</FONT></TD>
</TR>
</TABLE>

<FONT FACE="Palatino" SIZE=2><P>&nbsp;</P>
<LI>Now click on the </FONT><U><FONT FACE="Helvetica" SIZE=2>S</U>ave</FONT><FONT FACE="Palatino" SIZE=2> button to save the project to your directory.</LI>
<LI>The Project Explorer window looks a little different than it did before:</LI>
</FONT>
<TABLE CELLSPACING=0 BORDER=0 CELLPADDING=7>
<TR><TD VALIGN="TOP">
<FONT FACE="Palatino" SIZE=2><P ALIGN="CENTER"><IMG SRC="images/Image38.gif" WIDTH=197 HEIGHT=118></FONT></TD>
</TR>
</TABLE>

<FONT FACE="Palatino" SIZE=2>
<P>&nbsp;</P>
<P>The Project Explorer now shows that the project's </FONT><FONT FACE="Helvetica" SIZE=2>Name</FONT><FONT FACE="Palatino" SIZE=2> property is </FONT><FONT FACE="Helvetica" SIZE=2>Project1</FONT><FONT FACE="Palatino" SIZE=2>, and its external disk file name is </FONT><B><FONT FACE="Courier New" SIZE=1>China.vbp</B></FONT><FONT FACE="Palatino" SIZE=2>. The project contains a single form whose </FONT><FONT FACE="Helvetica" SIZE=2>Name</FONT><FONT FACE="Palatino" SIZE=2> property is </FONT><FONT FACE="Helvetica" SIZE=2>frmMain</FONT><FONT FACE="Palatino" SIZE=2>, and whose external disk file name is </FONT><B><FONT FACE="Courier New" SIZE=1>Main.frm</B></FONT><FONT FACE="Palatino" SIZE=2>.</P>
<LI>The final change to the project is to change the project's name to </FONT><FONT FACE="Helvetica" SIZE=2>China</FONT><FONT FACE="Palatino" SIZE=2>. Select </FONT><U><FONT FACE="Helvetica" SIZE=2>P</U>roject-Project1 Prop<U>e</U>rties</FONT><FONT FACE="Palatino" SIZE=2> from the Visual Basic main menu. Select the </FONT><FONT FACE="Helvetica" SIZE=2>General</FONT><FONT FACE="Palatino" SIZE=2> tab and change the </FONT><FONT FACE="Helvetica" SIZE=2>Project <U>N</U>ame</FONT><FONT FACE="Palatino" SIZE=2> to </FONT><FONT FACE="Helvetica" SIZE=2>China</FONT><FONT FACE="Palatino" SIZE=2>:</LI></FONT>
<TABLE CELLSPACING=0 BORDER=0 CELLPADDING=7>
<TR><TD VALIGN="TOP"><DIR>

<FONT FACE="Palatino" SIZE=2><P ALIGN="CENTER"><IMG SRC="images/Image39.gif" WIDTH=307 HEIGHT=142></DIR>
</FONT></TD>
</TR>
</TABLE>

<FONT FACE="Palatino" SIZE=2><P>&nbsp;</P>
<LI>Your Project Explorer window should now look like this:</LI>
</FONT>
<TABLE CELLSPACING=0 BORDER=0 CELLPADDING=7>
<TR><TD VALIGN="TOP"><DIR>

<FONT FACE="Palatino" SIZE=2><P>&nbsp;</P>
<LI>Having changed your project's </FONT><FONT FACE="Helvetica" SIZE=2>Name</FONT><FONT FACE="Palatino" SIZE=2> property, you now need to save this information to your project file. Click on the </FONT><FONT FACE="Helvetica" SIZE=2>Save</FONT><FONT FACE="Palatino" SIZE=2> button on the Toolbar as you did before. This time, because you have already gone through the process of naming and saving both your form and project, Visual Basic saves the changes to your project without prompting you for any information.</LI></OL>
