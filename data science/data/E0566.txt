<CENTER><H3><FONT COLOR="#a0a099">
    Tree Class
</FONT></H3></CENTER><HR>

<!-- Author and contact details -->
This article was contributed by 
<A href="mailto:akovache@san.rr.com"  >Alexander Kovachev</A>.

<p><u>Environment:</u> all

<P>Class for presenting tree structures in C++.</P>
<P>
1. Data objects:<BR>
- The purpose of the Data class is to keep the actual data. It keeps the data until all actual references to it are dead. This is done with reference counting (COM like). I know somebody may find this worthless and useless, but it makes the data more capsulated and secure. The implementation is divided in two classes Data and Ref (reference). All the data is stored in the Data class.All the job is done by Ref class, which manipulates the data class. <BR>

<PRE>
template &lt;class Type&gt; class Data {
private:
...<span class="codeComment">// all methods  vars are private</span>
   friend class Ref&lt;Type&gt;; <span class="codeComment">// Ref class will manipulate us</span>
};
template &lt;class Type&gt; class Ref {
    Ref(Type *p);
    Ref();
    Ref(const Ref&lt;Type&gt; rhs);
    <span class="codeComment">//destructor</span>
    virtual ~Ref();

<span class="codeComment">//operators =</span>
    Ref operator=(const Ref&lt;Type&gt; rhs);
    Ref operator=(const Type t);
    Ref operator=(Type *p); 
<span class="codeComment">// operator ==</span>
    bool operator==(const Ref&lt;Type&gt; rhs) 
<span class="codeComment">// misc</span>

Ref Clone();
    void Release();
    bool IsNull();

<span class="codeComment">// accessors to the actual data 
// (check if it's null first)</span>
    Type *operator-&gt;();
    Type get_Data();
    operator Type();
}
</PRE>

<p>2. Tree object:<BR>
-  Tree class represents a node in the tree, so it has parent and it has children. in this way i use a single class for representing a node and a tree, so the the tree is a node without praent. It also has reference (Ref) to the data. It's implementation is divided between two classes NodeData and Tree. NodeData objects holds the actual data for the parent and the children, but Tree object manipualtes this data and gives access to them. Actually you'll always use only Tree class directly.<BR>
<br>
    - definitions<BR>
        - node - a simple node or leaf in the tree<BR>
        - leaf - a node without children<BR>
        - root (tree) -  a node without parent (i.e. prent is Null)<BR>
        - Null - null node<BR>
<BR>


<PRE>template &lt;class Type&gt; class NodeData {
private:
    .... <span class="codeComment">// all methods  vars are private</span>
    friend class Tree&lt;Type&gt;;
}

template &lt;class Type&gt; class Tree {
public:
    <span class="codeComment">// we'll call the tree node</span>
    typedef Tree&lt;Type&gt; Node;
    
    <span class="codeComment">// parent</span>
    Node Parent();
    
    <span class="codeComment">// children - Nodes</span>
    __declspec( property( get=get_Nodes) ) Node Nodes[];
    __declspec( property( get=get_Count) ) int Count;

    Node get_Nodes(int nIndex);
    int get_Count();
    
    <span class="codeComment">// data acess</span>
    __declspec( property( get=get_Data) ) Ref&lt;Type&gt; Data;

    operator Type();
    Type *operator-&gt;();
    Ref&lt;Type&gt; get_Data();

    <span class="codeComment">// node functions (isLeaf and so on</span>
    bool IsLeaf();
    bool IsNode();
    bool IsRoot();

    <span class="codeComment">// comparision</span>
    bool operator == (const Node rhs)
          { return *(NodeBase*)this==(NodeBase )rhs;};

    <span class="codeComment">// node operations</span>
    Node AddNode(const Type t); 
    void Delete();
    void DeleteNode(int nIndex);
    void DeleteNode(Node node);

    <span class="codeComment">// constructors</span>
    Tree(const Type Data);
    Tree(Type *Data);
    
    Tree();
    Tree(const Tree&lt;Type&gt; rhs);
    Tree(const NodeBase rhs);
    
    <span class="codeComment">// destructor</span>
    virtual ~Tree();
}</PRE>

<p>3. Simple usage:</p>

<PRE>
<span class="codeComment">// function for printing nodes</span>
void PrintNode(StringNode node,int nLevel) {
    <span class="codeComment">// print new line</span>
    fputs("\n",stdout);
    <span class="codeComment">// print spaces for indent (2 for every level)</span>
    string s;
    s.resize(nLevel*2,' ');
    fputs(s.c_str(),stdout);

    <span class="codeComment">// print the value</span>
    fputs(node.get_Data()-&gt;c_str(),stdout);

    <span class="codeComment">// iterate through children</span>
    nLevel++;
    for(int n=0;n&lt;node.Count;n++) {
        PrintNode(node.Nodes[n],nLevel);
    }
}

int main(int argc, char* argv[])
{
    <span class="codeComment">// define the tree</span>
    StringTree tree; 
    ((string)tree)="Root";

    <span class="codeComment">//add few nodes</span>
    StringNode node=tree;

    node.AddNode("1").AddNode("11");
    StringNode node2=node.AddNode("2");

    node2.AddNode("21").AddNode("211");
    node2.AddNode("22");
    node2.AddNode("23");
    
    node.AddNode("3");

    <span class="codeComment">// print the tree</span>
    PrintNode(tree,0);

    <span class="codeComment">// wait for enter</span>
    char buf[3];
    fputs("\nPress ENTER :",stdout);
    fgets(buf,3,stdin);
    return 0;
}
</PRE>
