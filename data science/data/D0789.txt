<center><div class="menubar">Drawing Polygons        </div></center>
</td>

<tr>
<td bgcolor="#d2cec4">
<SPAN Class="smbl">
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;This lesson is probably going to be
the most confusing in the entire section. However, it is the most useful of all
for drawing weird shapes. Basically what is does is draw a simple polygon.
The programmer specifies the coordinates of the points, and the number of points,
and the computer connects those points in order, then fills in the shape. Here
are the basic formats. It will confuse you at first, but I will explain<br><br><i>
g.drawPolygon(x,y,points);<br>
g.fillPolygon(x,y,points);<br><br></i>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;First of all, the only difference between the
first and the second is that the second draws a solid shape, and the first draws an
outline. Okay, so here is what that mean. You are probably asking yourself,"Why are
there only one x and one y?" Yes, it is true that there are only two variables for
the coordinates, BUT, they can hold more than one value. For example:<br><br><i>
int x[]={1,5,3};<br><br></i>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;What this does is store more than one x value in
a variable. This is known as an array. 1 is the x value of the first coordinate, 5 is
the coordinate of the second coordinate, and 3 is the value of the third coordinate.
Then I might write the next line of code in:<br><br><i>
int y[]={24,93,53};<br><br></i>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;So what I did above would make the y value of the
coordinate be 24, the y value of the second coordinate be 93, and the y value of the
third coordinate be 53. Using the two lines above that declare the variables and
a g.fillPolygon(x,y,3); would draw a shape with 3 points. The first point would be (1,24),
the second point would be (5,93), and the third coordinate would be (3,53). Lines would connect
the first coordinate to the second, the second to the third, and the third to the first. Then,
the area in between the lines would be shaded. Check out the applet:<br><br>

<applet code=poly.class width=250 height=250></applet><br><br><pre><small>
//-------------------------------//
//Copyright 2000 FreeJavaHelp.com//
//-------------------------------//
//the next line imports the applet class
import java.applet.Applet;
//the next line imports the GUI components
import java.awt.*;
//the next line begins the program
public class poly extends Applet{
    //the next line creates an array that will hold x values
    int x[]={19,234,9,192,62};
    //the next line creates an array that will hold y values
    int y[]={35,135,241,141,71};
    public void init(){
        //the next line sets the bg to blue
        setBackground(Color.blue);
    }
    public void paint(Graphics g){
        //the next line sets the color to red
        g.setColor(Color.red);
        //the next line draws a polygon using the coordinates
        //defined above
        g.fillPolygon(x,y,5);
    }
}
</pre></small><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;This may have confused the heck out of you,
so I will explain it a little more. In the applet above, I define 5 x values
and 5 y values. The first x value goes with the first y value, the second with
the second, and so on. These coordinates are plotted, then connected in order
with lines, and then shaded in because I used fillPolygon and not drawPolygon.
If you have any more questions, just post in the forums!
