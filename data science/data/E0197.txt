<CENTER><H3><FONT color=#a0a099>
Switch on Strings in C++ 
</FONT></H3></CENTER>
<HR>

<!-- Author and contact details -->
This article was contributed by <A href="mailto:str@WudelWeb.de">Stefan Ruck</A>.<br>
<P>
<U>Environment: This is standard C++. So you do not need any special environment.</U>

 <!-- Text / source code -->
<P>
<H3>Preface</H3>
<p>How many times have you written code like this: 
<PRE><TT>if (!strcmp(pszValue, "Value X"))
  DoThis();
else if (!strcmp(pszValue, "Value Y"))
  DoThat();
else if (!strcmp(pszValue, "Value Z"))
  DoSomethingElse();
else
  DontKnowWhatToDo();</TT></PRE>

<p>Too many! And those of you who had the chance to take a look 
at C# might think 'Why can't I code something similar in C++ like this: 
<PRE><TT>switch(strValue)
{
  case "Value X":
    DoThis();
    break;
  case "Value Y":
    DoThat();
    break;
  case "Value Z";
    DoSomethingElse();
    break;
  default:
    DontKnowWhatToDo();
    break;
}</TT></PRE>
<p>(This code is legal C#. You can switch on strings in C#.) 
<P>In this article I will show you a way to implement a <TT>switch</TT> on 
strings using pure standard C++. </p>

<H3>The Bad News</H3>
<p>I can't give you a solution to do exactly what you can do in C#. </p>

<H3>The Good News</H3>
<p>The Standard Template Library (STL), part of the ANSI/ISO 
C++ Standard, offers everything needed to get really close to the C# sample. 
The solution is very simple. You need an enumeration and a <TT>std::map</TT>, 
and that's it. The enumeration defines the numeric values use in the 
<tt>switch</tt> statement. The <TT>std::map</TT> contains the link between the valid 
string values you want to compare some runtime data against, and the numeric enum 
values you can make a <TT>switch</TT> on. The string is the key of the map, the 
enumerator the value. 
<H3>Here We Go</H3>
<p>After thinking about the problem a little bit, there is only 
one place where to put the <TT>enum</TT> definition and the <TT>std::map</TT> 
definition: The .cpp file. Both, the <TT>enum</TT> and the <TT>std::map</TT>, 
should be declared as static to make them visible only to the code of this 
particular .cpp file, which helps to avoid global namespace pollution (for more 
on this see John Lakos, Large Scale C++ Software Design, Addison-Wesley). There 
is no need to declare them as members of any class since normally the connection 
between a string value and the action to be taken is very specific to the 
respective code. </p>
<P>And here comes a sample Switch On String implementation I will use for later 
discussion: 

<PRE><TT>#include &lt;map&gt;
#include &lt;string&gt;
#include &lt;iostream.h&gt;

<span class="codeComment">// Value-Defintions of the different String values</span>
static enum StringValue { evNotDefined, 
                          evStringValue1, 
                          evStringValue2, 
                          evStringValue3, 
                          evEnd };

<span class="codeComment">// Map to associate the strings with the enum values</span>
static std::map&lt;std::string, StringValue&gt; s_mapStringValues;

<span class="codeComment">// User input</span>
static char szInput[_MAX_PATH];

<span class="codeComment">// Intialization</span>
static void Initialize();

int main(int argc, char* argv[])
{
<span class="codeComment">  // Init the string map</span>
  Initialize();

<span class="codeComment">  // Loop until the user stops the program</span>
  while(1)
  {
<span class="codeComment">    // Get the user's input</span>
    cout &lt;&lt; "Please enter a string (end to terminate): ";
    cout.flush();
    cin.getline(szInput, _MAX_PATH);
<span class="codeComment">    // Switch on the value</span>
    switch(s_mapStringValues[szInput])
    {
      case evStringValue1:
        cout &lt;&lt; "Detected the first valid string." &lt;&lt; endl;
        break;
      case evStringValue2:
        cout &lt;&lt; "Detected the second valid string." &lt;&lt; endl;
        break;
      case evStringValue3:
        cout &lt;&lt; "Detected the third valid string." &lt;&lt; endl;
        break;
      case evEnd:
        cout &lt;&lt; "Detected program end command. "
             &lt;&lt; "Programm will be stopped." &lt;&lt; endl;
        return(0);
      default:
        cout &lt;&lt; "'" &lt;&lt; szInput 
  &lt;&lt; "' is an invalid string. s_mapStringValues now contains "
             &lt;&lt; s_mapStringValues.size() 
             &lt;&lt; " entries." &lt;&lt; endl;
        break;
    }
  }

  return 0;
}

void Initialize()
{
  s_mapStringValues["First Value"] = evStringValue1;
  s_mapStringValues["Second Value"] = evStringValue2;
  s_mapStringValues["Third Value"] = evStringValue3;
  s_mapStringValues["end"] = evEnd;

  cout &lt;&lt; "s_mapStringValues contains " 
       &lt;&lt; s_mapStringValues.size() 
       &lt;&lt; " entries." &lt;&lt; endl;
}</TT></PRE>

<p>In looking at the <TT>enum</TT> definition, you see that the first 
value is <TT>evNotDefined</TT> (ev means 'enum value'). Why this? 
<TT>std::map::operator[]</TT> can be used for two things: To set a value of a 
key (as you can see in <TT>Initialize()</TT>) and to retrieve the value 
associated with it (look at the <TT>switch</TT> statement in the <TT>main()</TT> 
function). The most important sentence of its description is: <B>'If this 
element</B> (key the value [author]) <B>does not exist, it is inserted.'</B>. In 
relation to our sample above this means, any time we go thru the <TT>switch</TT> 
and the value of <TT>szInput</TT> is new to <TT>s_mapStringValues</TT>, it is 
inserted (this is why I added the print out of the map's size) and the value of 
the new element is set to the initial value of an integral type, which is 0 by 
default. And as long as there is no special value assigned to the first 
enumerator of an enumeration, its value is zero too (please refer to the C++ 
Standard for more on this). That means, if you start the enumeration with a valid 
value (here: <TT>evStringValue1</TT>), any unknown (and so unexpected) string 
value would lead to a valid <TT>case</TT> of the <TT>switch</TT>, but to an 
invalid program behaviour. That's why <TT>evNotDefined</TT> is the first 
enumerator. <BR>In case you have to use a given enumeration where an enumerator 
with value zero is defined, you should call <TT>std::map::find()</TT> before the 
<TT>switch</TT> statement to check if the string value is valid. 
<P>The rest of the sample is quite simple. The <TT>switch</TT> is not 
made on the string itself but on the numeric value associated to it by the 
<TT>std::map</TT>. So there's not really magic in there. 
<H3>When to Initialize</H3>In this sample, the answer is easy: When the program 
starts. But unfortunately real life applications aren't as simple as all these 
samples. So when to initialize in real life? </p>

<P>There are two scenarios we have to cover: The switch is part of a non-static 
member function, and the switch is part of a static method or global function. </p>

<P>If it is part of a non-static member function, the class' constructor should 
do initialization, as long as you do not have to care about the runtime 
behaviour of the ctor. (See comment on 'lazy' init below.) To avoid useless 
multiple initialization, one should check the size of the map before setting the 
value. If it is zero, no initialization has been made before, otherwise there is 
no more need for it. </p>

<P>In case of a static method or global function, you would prefer something 
similar to C#'s static constructor. But there is nothing like this in C++ (as 
far as I know). So you do have two choices: 'Lazy' or 'As-soon-as-possible' 
initialization. </p>
<ul>
<li>'Lazy' initialization means just before the map is used, a 
check is done to see if it is filled correctly (again, check its size), and, if not, you will do 
so. </li>
<li>'As-soon-as-possible' init needs an additional Init 
method the (class) user has to call in order make sure the switch will work 
correctly. </li>
</ul>
<p>Both solutions have their pros and cons. The 'lazy' way leads to 
additional code before every <TT>switch</TT> that is using the map and so has an 
impact on the runtime behaviour. (You can't tell exactly how long the call will 
take since you do not know whether the string map will be initialized or not.) 
But the user can't forget to initialize the map. The 'As-soon-as-possible' 
implies the risk the user has forgotten to call the Init method (but this will 
be noticed very soon as long as you do some testing on you app), but saves some 
code when using the map and guaranties runtime behaviour. So it's up to you 
which way to go. I do prefer the latter one. </p>

<H3>Case In-Sensitive</H3>
<p>As long as you do not have to use special characters 
like umlaute (ä, ü, ...) or whatever, you just have to fill the map with upper- 
or lower-case-only strings and use _strupr or _strlwr in the <TT>switch</TT> 
statement. </p>
<P>A solution for special characters might be a good exercise for the ambitioned 
reader. </p>

<H3>Conclusion</H3>
<p>I think this solution is a good sample to show the power of 
the STL. Hopefully it was able to convince you of the simplicity and elegance of 
<TT>switch</TT> statements on string values. Use the STL as much as you can. </p>

<H3>Remarks</H3>
<p>I hope you do believe me when I say that I worked this out 
without knowing about the discussions on this topic at comp.lang.c++.moderated 
or any other discussion group. I have to confess that I did not read thru this 
discussions since I'm really happy with the solution demonstrated, which has 
proven its usability in real life. But I do not want to give the impression that 
I was the first and only one who had the idea of using a map to implement 
<TT>switch</TT> on strings in C++. </p>
