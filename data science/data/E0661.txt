<CENTER><H3><FONT COLOR="#AOAO99">
Extended Array Template Like CArray
</FONT></H3></CENTER><HR>

<!-- Author and contact details -->

<p>This article was contributed by 
<A HREF="mailto:cdjindia@yahoo.com">Chetan Joshi</A>.</p>


<p><u>Environment:</u> Generic. Tested on Windows 2000 and Windows Me

<p>

<H3>CExtendedArray&lt;TYPE, NUM_SORTFLDS, FLDTYPE&gt; </H3>

<p>This template implements an abstract MFC style class and provides  additional functionality of sorting the data internally over NUM_SORTFLDS simultaneously. Also, class provides ability to search for a given value over these fields quickly, (using binary search) implemented over the sorted fields. It depends on MFC template classes (afxtempl.h) to provide global functions like ConstructElement etc.... If you do not wish to use MFC at all, uncomment all of the functions from top of the file.</p>

<p>It is an abstract class and client must two CompareElem functions. Both of this functions compare two supplied elements and return negative, zero or positive number depending upon whether first element is "less than", "equal to" or "greater than" second element respectively. Only difference between two functions is that in one version, it provides references to two TYPE elements, whereas the second version, passes reference to type and FLDTYPE value.</p>

<p><B>TYPE</B></p>
<p>is a data type of object that the CExtArray template will store a pointer to I have done away with ARG_TYPE present in MFC class and all references to object are either converted to a pointer or reference Also, although you can use any field in TYPE, <u>class really becomes useful, when
TYPE is a pointer to structure or a class </u></p>

<p><B>NUM_SORTFLDS</B><p>
<p>is a integer value that defines number of fields within the structure, for which you want ability to search and sort data.</p>

<p><B>FLTDTYPE</B></p>

<p>is type of field that can be sorted. Only possible value that can make sense here is "void *" or LPARAM. I deliberated a lot on this and cannot think of any other type to use. Send me your comments if you come up with something better. It is basically a place holder that is used while calling CompareElem function.


<p>Sample usage of CExtendedArray&lt;&gt; is as follows:

<PRE>class something
{
  public:
      char    ApplId[4];
      char    szApplName[81];
};

typedef something* LPSOMESTR;

class MyArray : public CExtendedArray&lt;LPSOMESTR, 2, LPARAM&gt;
{
  int CompareElem(int nSortFld, 
                  LPSOMESTR&amp; first, 
                  LPSOMESTR&amp; second, 
                  LPARAM lHint)
  {
    int r;
    if (nSortFld == 0)
    {
      r = lstrcmpi(first-&gt;szApplName, second-&gt;szApplName);
      <span class="codeComment">//std::cerr &lt;&lt; "Comparing " &lt;&lt; first-&gt;szApplName 
      //   &lt;&lt; " :with: " &lt;&lt; second-&gt;szApplName 
      //   &lt;&lt; "&lt;-&gt;" &lt;&lt; r &lt;&lt; std::endl;</span>
      return r;
    }
    else if (nSortFld == 1)
    {
      return lstrcmpi(first-&gt;ApplId , second-&gt;ApplId);
      <span class="codeComment">//std::cerr &lt;&lt; "Comparing " &lt;&lt; first-&gt;ApplId 
      // &lt;&lt; " :with: " &lt;&lt; second-&gt;ApplId 
      // &lt;&lt; "&lt;-&gt;" &lt;&lt; r &lt;&lt; std::endl;</span>
      return r;
    }

    return 0;
  };

  int CompareElem(int nSortFld, LPSOMESTR& first, 
                  LPARAM valLookup, LPARAM lHint)
  {
    int r;
    if (nSortFld == 0)
    {
      r = lstrcmpi(first-&gt;szApplName, (LPCTSTR)valLookup);
      std::cerr &lt;&lt; "Comparing " &lt;&lt; first-&gt;szApplName 
                &lt;&lt; " :with: " &lt;&lt; (LPCTSTR)valLookup 
                &lt;&lt; "&lt;-&gt;" &lt;&lt; r &lt;&lt; std::endl;
      return r;
    }
    else if (nSortFld == 1)
    {
      r = lstrcmpi(first->ApplId , (LPCTSTR)valLookup);
      std::cerr &lt;&lt; "Comparing " &lt;&lt; first-&gt;ApplId 
                &lt;&lt; " :with: " &lt;&lt; (LPCTSTR)valLookup 
                &lt;&lt; "&lt;-&gt;" &lt;&lt; r &lt;&lt; std::endl;
      return r;
    }

    return 0;
  };
};

void testinit()
{
    MyArray wordArray;
    MyArray wordArray2;
    MyArray wordArray3;

    LPSOMESTR w;

    wordArray.SetSize(3,3);

    for (int i = 0; i &lt; 3; i++)
    {
        w = new something;
<span class="codeComment">        // fill in w</span>
        wordArray[i] = w;
    }

    wordArray.SortData(0, 0); <span class="codeComment">// szApplName</span>

    wordArray.SortData(1, 0); <span class="codeComment">// APPL</span>
    LPSOMESTR retVal = NULL;

    if (wordArray.Lookup(1, (LPARAM)"EXT", retVal, 0))
        std::cout &lt;&lt; retVal-&gt;szApplName &lt;&lt; std::endl;

    retVal = NULL;

    if (wordArray.Lookup(1, (LPARAM)"OTH", retVal, 0))
        std::cout &lt;&lt; retVal-&gt;szApplName &lt;&lt; std::endl;
    else
        std::cout &lt;&lt; "Not found" &lt;&lt; std::endl;
