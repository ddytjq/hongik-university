<CENTER><H3><FONT COLOR="#AOAO99">
Win32 Resource API C++ Wrapper
</FONT></H3></CENTER><HR>

<!-- Author and contact details -->

This article was contributed by 
<A HREF="mailto:oknp@hotmail.com">Peter Chiu</A>.

<p><a href="rsrc-simple.zip">Download Source Code and Example</a></p>

<!-- Sample image - gif or jpg -->
<p><a href="rsrc-simple-view.gif" target=newFrame>

<!-- For which environment is this code specific??? -->
<p><u>Environment:</u> VC6 95/98/Me/NT/2000/XP

<h2>Overview</h2>
<p> I personally find the Win32 Resource API to be overly complex for performing 
  simple operations such as enumerating and copying resources. Some operations 
  like deleting resources is even missing from the standard Win32 API set. The 
  classes presented in this article provides a simple interface for enumerating, 
  adding and removing resources from Win32 image files (.dll/.exe).</p>
<h2>Understanding Win32 Resources</h2>
<p>
  This section gives you a conceptual overview of the Win32 resource layout. We 
  will not go into details about the format of the .rsrc section here. </p>
<p>The diagram above shows an example of a conceptual view of the Win32 resource 
  tree. Each resource instance (leaf-level) contains the following attributes:</p>
<table width="100%">
  <tr> 
    <td bgcolor="#7F90AF" width="8%" align="left" valign="top"><b><font color="#FFFFFF" face="Verdana, Arial, Helvetica, sans-serif" size="2">Type</font></b></td>
    <td bgcolor="#F2F5FD" width="92%"><font face="Verdana, Arial, Helvetica, sans-serif" size="2">cursors, 
      dialogs, bitmaps, etc</font></td>
  </tr>
  <tr> 
    <td bgcolor="#7F90AF" width="8%" align="left" valign="top"><b><font color="#FFFFFF" face="Verdana, Arial, Helvetica, sans-serif" size="2">Name</font></b></td>
    <td bgcolor="#F2F5FD" width="92%"><font face="Verdana, Arial, Helvetica, sans-serif" size="2">a 
      Unicode string or a 2-byte numeric identifier</font></td>
  </tr>
  <tr> 
    <td bgcolor="#7F90AF" width="8%" align="left" valign="top"><b><font color="#FFFFFF" face="Verdana, Arial, Helvetica, sans-serif" size="2">Language</font></b></td>
    <td bgcolor="#F2F5FD" width="92%"><font face="Verdana, Arial, Helvetica, sans-serif" size="2">a 
      2-byte numeric value indicating the language of the resource instance. E.g., 
      English (U.S.) = 1033, Chinese (Traditional) = 1028</font></td>
  </tr>
  <tr> 
    <td bgcolor="#7F90AF" width="8%" align="left" valign="top"><b><font color="#FFFFFF" face="Verdana, Arial, Helvetica, sans-serif" size="2">Data</font></b></td>
    <td bgcolor="#F2F5FD" width="92%"><font face="Verdana, Arial, Helvetica, sans-serif" size="2">the 
      actual data of the resource (e.g. a bitmap image)</font></td>
  </tr>
</table>
<p>These resource instances live in a Win32 image file as a tree structure. Top 
  level of the tree hosts the resource types. For each resource type, there can 
  be multiple resources with the same name, but different language IDs. Those 
  are organized at the 2nd and 3rd levels of the resource tree. At the leaf-level 
  is the actual resource data itself.</p>
<h2>The Win32 Resource API</h2>
<p>Windows NT (Yes, and I mean only NT/2000/XP. Windows 95/98/Me do not support 
  these APIs. To get cross-platform functionality, check out the <a href="http://geeworks.com/rsrc/" target="new">Geeworks 
  Win32 Resource Library</a>) provides some useful APIs for modifying resources 
  in Win32 image files. For instance, to copy resources from one image file to 
  another involves the following step:</p>
<table width="100%">
  <tr> 
    <td width="3%" bgcolor="#7F90AF" align="left" valign="top"><b><font color="#FFFFFF" size="2" face="Verdana, Arial, Helvetica, sans-serif">1.</font></b></td>
    <td width="97%" bgcolor="#F2F5FD"><font size="2" face="Verdana, Arial, Helvetica, sans-serif">Set 
      up a recursive loop (using EnumResourceTypes, EnumResourceNames, EnumResourceLanguages) 
      to navigate the resource tree of the source image file and collect the resource 
      attributes.</font></td>
  </tr>
  <tr> 
    <td width="3%" bgcolor="#7F90AF" align="left" valign="top"><b><font color="#FFFFFF" size="2" face="Verdana, Arial, Helvetica, sans-serif">2.</font></b></td>
    <td width="97%" bgcolor="#F2F5FD"><font size="2" face="Verdana, Arial, Helvetica, sans-serif">Use 
      BeginUpdateResource to open the destination iamge file (not supported on 
      Win9x).</font></td>
  </tr>
  <tr> 
    <td width="3%" bgcolor="#7F90AF" align="left" valign="top"><b><font color="#FFFFFF" size="2" face="Verdana, Arial, Helvetica, sans-serif">3.</font></b></td>
    <td width="97%" bgcolor="#F2F5FD"><font size="2" face="Verdana, Arial, Helvetica, sans-serif">Load 
      resource data into memory using FindResource and LoadResource (partially 
      supported on Win9x).</font></td>
  </tr>
  <tr> 
    <td width="3%" bgcolor="#7F90AF" align="left" valign="top"><b><font color="#FFFFFF" size="2" face="Verdana, Arial, Helvetica, sans-serif">4.</font></b></td>
    <td width="97%" bgcolor="#F2F5FD"><font size="2" face="Verdana, Arial, Helvetica, sans-serif">Copy 
      the resource data using UpdateResource (again, not supported on Win9x).</font></td>
  </tr>
  <tr> 
    <td width="3%" bgcolor="#7F90AF" align="left" valign="top"><b><font color="#FFFFFF" size="2" face="Verdana, Arial, Helvetica, sans-serif">5.</font></b></td>
    <td width="97%" bgcolor="#F2F5FD"><font size="2" face="Verdana, Arial, Helvetica, sans-serif">Close 
      the destination image file to write the changes to the destination image 
      file.</font></td>
  </tr>
</table>
<p>Not only are they overly complicated, they are not even supported on Win9x 
  platforms. Details see Microsoft KB article Q271442 and documentation for BeginUpdateResource(), 
  UpdateResource(), or EditUpdateResource().</p>
<h2>The Win32 Resource API C++ Wrapper</h2>
<p>Two C++ classes are presented here for manipulating Win32 resources. CWin32ResourceSrc 
  encapsulates a read-only source for resource copying. It provides a simple, 
  non-recursive interface for resource enumeration. CWin32ResourceDest provides 
  functions for adding and removing resources from a Win32 image file.</p>
<table width="100%" cellpadding="3" cellspacing="3">
  <tr>
    <td>
      <table width="100%">
        <tr> 
          <td bgcolor="#7F90AF"> 
            <p><font face="Verdana, Arial, Helvetica, sans-serif"><b><font color="#FFFFFF">CWin32ResourceSrc</font></b></font></p>
          </td>
        </tr>
        <tr> 
          <td bgcolor="#F2F5FD"> 
            <table width="100%">
              <tr> 
                <td> 
                  <p><font face="Courier New, Courier, mono" size="2">BOOL Open(LPCTSTR 
                    Image);</font></p>
                </td>
              </tr>
              <tr> 
                <td> 
                  <p><font size="2" face="Verdana, Arial, Helvetica, sans-serif">Open 
                    a Win32 image file for resource enumeration.</font></p>
                </td>
              </tr>
            </table>
          </td>
        </tr>
        <tr> 
          <td bgcolor="#F2F5FD"> 
            <table width="100%">
              <tr> 
                <td> 
                  <p><font face="Courier New, Courier, mono" size="2">BOOL Close();</font></p>
                </td>
              </tr>
              <tr> 
                <td> 
                  <p><font size="2" face="Verdana, Arial, Helvetica, sans-serif">Close 
                    an opened image file.</font></p>
                </td>
              </tr>
            </table>
          </td>
        </tr>
        <tr> 
          <td bgcolor="#F2F5FD"> 
            <table width="100%">
              <tr> 
                <td> 
                  <p><font face="Courier New, Courier, mono" size="2">BOOL EnumTypes(RSRC_TYPE_ARRAY 
                    &amp; Array);</font></p>
                </td>
              </tr>
              <tr> 
                <td> 
                  <p><font size="2" face="Verdana, Arial, Helvetica, sans-serif">Returns 
                    all the types in a RSRC_TYPE_ARRAY which is defined as a LPCTSTR 
                    vector. The caller of the function passes in a RSRC_TYPE_ARRAY 
                    and is also responsible for cleaning up the array with EmptyIdArray 
                    when done using the array.</font></p>
                </td>
              </tr>
            </table>
          </td>
        </tr>
        <tr> 
          <td bgcolor="#F2F5FD"> 
            <table width="100%">
              <tr> 
                <td> 
                  <p><font face="Courier New, Courier, mono" size="2">BOOL EnumNames(LPCTSTR 
                    Type, RSRC_NAME_ARRAY &amp; Array);</font></p>
                </td>
              </tr>
              <tr> 
                <td> 
                  <p><font face="Verdana, Arial, Helvetica, sans-serif" size="2">Returns 
                    all the names for a particular type in a RSRC_NAME_ARRAY which 
                    is defined as a LPCTSTR vector. The caller of the function 
                    passes in a RSRC_NAME_ARRAY and is also responsible for cleaning 
                    up the array with EmptyIdArray when done using the array.</font></p>
