<CENTER><H3><FONT COLOR="#AOAO99">

A Custom Block Allocator for Speeding Up VC++ STL

</FONT></H3></CENTER>

<hr>

<p>This article was contributed by
   <a href="mailto:joaquin@tid.es">Joaqu&iacute;n M L&oacute;pez Mu&ntilde;oz</a>.</p>

<p><u>Environment:</u> Visual C++ 5.0</p>

<p>
block_allocator is a custom STL allocator for use with STL as
implemented in Microsoft VC++ 5.0. Rather than doing allocations
on a per-node basis, block_allocator allocates memory in fixed sized
chunks and delivers portions of these chunks as requested. Typical speed

improvements of 40% have been obtained with respect to the default
allocator. The size of the chunks, set by the user, should not be too
little (reduced speed improvemente) nor too large (memory wasted).
Experiment and see what sizes fit best to your application.
<p>
block_allocator can substitute for the default allocator in the
following
containers:
<ul>
  <li>list,
  <li>set,
  <li>multiset,
  <li>map,
  <li>multimap,
</ul>
and WON'T work with other containers such as vector or queue. Note
however that vector and queue already perform allocation in chunks.
For each of the containers supported, there's an associated block
allocated
container derived from it thru use of block_allocator. You have to
define an
activating macro for each container to be defined prior to the inclusion

of blockallocator.h:
<ul>
  <li>list     -&gt; block_allocated_list     (macro
DEFINE_BLOCK_ALLOCATED_LIST),
  <li>set      -&gt; block_allocated_set      (macro
DEFINE_BLOCK_ALLOCATED_SET),
  <li>multiset -&gt; block_allocated_multiset (macro
DEFINE_BLOCK_ALLOCATED_MULTISET),
  <li>map      -&gt; block_allocated_map      (macro
DEFINE_BLOCK_ALLOCATED_MAP),
  <li>multimap -&gt; block_allocated_multimap (macro
DEFINE_BLOCK_ALLOCATED_MULTIMAP),
</ul>
To use block allocation based STL in your application, define the
corresponding activating macro, include blockallocator.h and then change

your declarations as follows:
<ul>
  <li>list&lt;type&gt;         -&gt;
block_allocated_list&lt;type,chunk_size&gt;
  <li>set&lt;key&gt;           -&gt;
block_allocated_set&lt;key,chunk_size&gt;
  <li>multiset&lt;key&gt;      -&gt;
block_allocated_multiset&lt;key,chunk_size&gt;
  <li>map&lt;key,type&gt;      -&gt; block_allocated_map&lt;key,type&gt;

  <li>multimap&lt;key,type&gt; -&gt;
block_allocated_multimap&lt;key,type&gt;
</ul>
where chunk_size is the size of the chunks. You can enter too the
other optional template parameters (see MSVC++ STL docs for more info).
Normal containers and block allocated containers can coexist without
problems.
<p>
This code is written for MSVC++ 5.0. Chances are that it will work
with MSVC++ 6.0, but this has not been tested.
<p>
This code won't work with other STL implementations different from
Microsoft's. This is due to the fact that Microsoft STL relies on
a non-standard extension to the allocator interface to cope with
the lack of template member function support of Microsoft C++ compiler.
</p>
