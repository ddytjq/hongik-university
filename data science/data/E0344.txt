		<td VALIGN="TOP" width=600><p>
<b>Canonical Class Template</b><br>
We can modify the listing above to turn it into a canonical template class definition. Just like function templates, this means we have to use the &lt;T&gt; template syntax in a few places, and sometimes in more than a few.
Luckily, it's not that hard, and the result can be seen in the following listing:
<p>
<font size=3><pre>template &lt;class T&gt; class TBase
{
  public:
    // Constructors & Destructors
    TBase(void);
    TBase(const TBase&lt;T&gt;& copy);
    virtual ~TBase(void);

    // Operator overloading
    TBase&lt;T&gt;& operator = (const TBase&lt;T&gt;& other);
    int operator == (const TBase&lt;T&gt;& other) const;

    // Output
    friend ostream& operator &lt;&lt; (ostream& os, const TBase&lt;T&gt;& other);
};</font></pre>
<p>
Just to let you know what the implementation looks like (the empty skeletons, that is), take a look at the following listing:
<p>
<font size=3><pre>// Constructors & Destructors
template &lt;class T&gt; TBase&lt;T&gt;::TBase(void) {}
template &lt;class T&gt; TBase&lt;T&gt;::TBase(const TBase&lt;T&gt;& copy) {}
template &lt;class T&gt; TBase&lt;T&gt;::~TBase(void) {}
<br>
// Operator overloading
template &lt;class T&gt; TBase&lt;T&gt;& TBase&lt;T&gt;::operator = (const TBase&lt;T&gt;& other) {}
template &lt;class T&gt; int TBase&lt;T&gt;::operator == (const TBase&lt;T&gt;& other) const {}
<br>
// Output
template &lt;class T&gt; ostream& operator &lt;&lt; (ostream& os, const TBase&lt;T&gt;& other) {}</font></pre>
<p>
This is usually the place where I could do with a little help or support to get the class template syntax right.
<p>
<b>Derived Templates</b><br>
If you've been able to keep up with me so far, then let's get to the final round: templates derived from other templates. Sometimes you just have to derive your own custom class template TDerived from a base template class TBase (sound familiar?).
And just for your amusement (and mine), I've included the header listing for the derived canonical class template definition below:
<p>
<font size=3><pre>template &lt;class T&gt; class TDerived: public TBase&lt;T&gt;
{
  public:
    // Constructors & Destructors
    TDerived(void);
    TDerived(const TDerived&lt;T&gt;& copy);
    virtual ~TDerived(void);

    // Operator overloading
    TDerived&lt;T&gt;& operator = (const TDerived&lt;T&gt;& other);
    int operator == (const TDerived&lt;T&gt;& other) const;

    // Output
    friend ostream& operator &lt;&lt; (ostream& os, const TDerived&lt;T&gt;& other);
};</font></pre>
<p>
Certainly this TDerived class template definition needs a list of empty implementation skeletons, which are defined as follows (empty because they're skeletons, but they still need to be implemented by the programmer, of course).
<p>
<font size=3><pre>// Constructors & Destructors
template &lt;class T&gt; TDerived&lt;T&gt;::TDerived(void): TBase&lt;T&gt;() {}
template &lt;class T&gt; TDerived&lt;T&gt;::TDerived(const TDerived&lt;T&gt;& copy): TBase&lt;T&gt;(copy) {}
template &lt;class T&gt; TDerived&lt;T&gt;::~TDerived(void) {}
<br>
// Operator overloading
template &lt;class T&gt; TDerived&lt;T&gt;& TDerived&lt;T&gt;::operator = (const TDerived&lt;T&gt;& other) {}
template &lt;class T&gt; int TDerived&lt;T&gt;::operator == (const TDerived&lt;T&gt;& other) const {}
<br>
// Output
template &lt;class T&gt; ostream& operator &lt;&lt; (ostream& os, const TDerived&lt;T&gt;& other) {}</font></pre>
<p>
OK, who could already produce the above listing without a second thought? If you could, then you probably didn't need to read this article, because the fun stuff is over. What remains is the description of a little tool that I made for myself to actually produce and generate the output listings that we've seen so far.
