<center><div class="menubar">Five Common Mistakes        </div></center>
</td>

<tr>
<td bgcolor="#d2cec4">
<SPAN Class="smbl">
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Every programmer has to debug code. Its a
fact of life. Some programmers, however, have it much easier than others
because they either have more debugging experience, or know what errors are
most common and therefore immediately look for them. The following list is my
opinion of the most common mistakes, with #1 being the most common.<br><br>
<big><big>#1. Capitalization and
Spelling</big></big><br> <br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Wowee! This is definitely a big one. Java
is a case sensitive language. Not only does everything have to be spelled
EXACTLY right, but the capitalization also has to be correct. For example,
some people might put the following into their code:<br><br>button main=new
button();<br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;You simply can't do this. You will get an
error if you don't capitalize the letter B in button (both times). Luckily for
us, most class names has the first letter capitalized, and that is it. For
example, List, Choice, Button, and Label, are classes in the AWT. The first
letters are capitalized. If you have a problem with capitalization,
you should probably try to notice capitalization patterns. Spelling is
something you just have to memorize.<br><br>
<big><big>#2. Forgetting ;</big></big><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Every programmer, including myself, is
guilty of forgetting a semicolon once in a while. Semicolons are used to end
Java statements. Luckily, compilers are pretty good when it comes to spotting
missing semicolons, and that makes life a lot easier.<br><br>
<big><big>#3.Missing Braces and Parentheses</big></big><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;() and {} are used a lot in Java. Braces
are used make a group of statements into a block of code. Every applet has
braces and parentheses. For each opening brace or parenthese, there should be a
closing one. A missing brace especially will pop up a huge list of errors when
you try to compile.<br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;One trick I use to avoid forgetting a
parenthese or brace is to create both the opening AND closing ones and THEN
fill in the code in between. For example, whenever I am writing an applet, I
create the opening and closing braces which will contain the code for the
applet, and THEN I fill in the code. This usually takes care of the
problem.<br><br>
<big><big>#4. Endless Loops</big></big><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Consider a scenario with me for just a
second. In your program, you have a variable named x. The goal of your program
is to draw all the numbers from 0 to 20 on your applet window. You also have a
variable named y, which controls the y value of the coordinate where you paint
your number. To accomplish this, you might put the following loop in public
void paint():<br><br><pre>
while(x<21){
	g.drawString(x,10,y);
	y+=20;
	x++;
}
</pre><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;The above loop would get the program to do
what you wanted. If we took out the line x++; , we would have what is known as
an endless loop. An endless loop is a loop that never ends because the
condition under which it runs is always true. In the example above, if we take
out x++; , the value of x will always be less than 21, so the loop will run
forever (while the computer is on.) Endless loops freeze programs.<br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;The way to avoid is an endless loop is to
make sure that it is possible for the condition under which the loop runs will
be false at some time or another. This usually means changing a variable inside
the loop, like how x is changed in the example above.<br><br>
<big><big>#5. = vs. ==</big></big><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;One very common mistake that people make is
they use a = when they should use ==, or vice versa. = and == are two
completely different operators. I will explain a little bit about each one. =
is used to assign values for variables. For example, if I wanted to make p have
the same value as q, I would write the following line:<br><br><pre>
p=q;</pre><br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;The line above says, assign the value of q
to p. == is used to test if the value of something is ALREADY equal to the
value of something else. If I had an if statement that used = instead of ==, I
would have a major problem. Instead of checking if they are equal, it would
assign the value of the number or variable after the = to the variable before
the =. <br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Here is a rule of thumb I use. Only use ==
in loops or conditional statements in between the parentheses. All other times,
use =. This works really well. There might be some cases where it doesn't hold
true, but it will help.
